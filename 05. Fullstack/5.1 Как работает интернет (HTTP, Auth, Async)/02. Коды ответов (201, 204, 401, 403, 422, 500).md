## 📊 Самые используемые HTTP статус-коды (2xx, 3xx, 4xx, 5xx)

### **✅ 2xx — Успешно (SUCCESS)**

| Код                | Описание                             | Когда использовать                           |
| :----------------- | :----------------------------------- | :------------------------------------------- |
| **200 OK**         | Успешно, данные в теле               | GET (данные получены), PUT/PATCH (обновлено) |
| **201 Created**    | Ресурс создан                        | POST (новый пользователь/пост)               |
| **204 No Content** | Успешно, тело пустое                 | DELETE (ресурс удалён)                       |
| **202 Accepted**   | Запрос принят, обработка асинхронная | Background jobs                              |

### **🔄 3xx — Перенаправление (REDIRECT)**

| Код                        | Описание                  | Пример                           |
| :------------------------- | :------------------------ | :------------------------------- |
| **301 Moved Permanently**  | Ресурс переехал навсегда  | `/old-url → /new-url` (SEO)      |
| **302 Found**              | Временное перенаправление | `/login → /profile`              |
| **307 Temporary Redirect** | Сохраняет метод запроса   | POST остаётся POST               |
| **304 Not Modified**       | Ресурс не изменился       | Кэш валиден (ETag/Cache-Control) |

### **❌ 4xx — Ошибка клиента (CLIENT ERROR)**

| Код                          | Описание                   | Frontend реакция         |
| :--------------------------- | :------------------------- | :----------------------- |
| **400 Bad Request**          | Неверный синтаксис запроса | Валидация формы          |
| **401 Unauthorized**         | Нет авторизации            | Показать login           |
| **403 Forbidden**            | Доступ запрещён            | "У вас нет прав"         |
| **404 Not Found**            | Ресурс не существует       | "Страница не найдена"    |
| **422 Unprocessable Entity** | Семантическая ошибка       | "Email уже занят"        |
| **429 Too Many Requests**    | Rate limit превышен        | "Слишком много запросов" |

### **💥 5xx — Ошибка сервера (SERVER ERROR)**

| Код                           | Описание                          | Frontend                     |
| :---------------------------- | :-------------------------------- | :--------------------------- |
| **500 Internal Server Error** | Общая серверная ошибка            | "Что-то сломалось"           |
| **502 Bad Gateway**           | Прокси получил неверный ответ     | "Сервер временно недоступен" |
| **503 Service Unavailable**   | Сервер перегружен/на обслуживании | "Попробуйте позже"           |
| **504 Gateway Timeout**       | Таймаут прокси                    | "Превышено время ожидания"   |

## 🎯 **REST API практика (CRUD)**

```
📥 GET /api/users           → 200 OK + [users]
📥 GET /api/users/123       → 200 OK + user / 404 Not Found
📤 POST /api/users          → 201 Created + {id: 456}
🔄 PUT /api/users/123       → 200 OK + full user
🔧 PATCH /api/users/123     → 200 OK + updated user
🗑️  DELETE /api/users/123   → 204 No Content
```

## 🚨 **Типичные собес-запросы**

**Q: 200 vs 201?**

```
200 = "всё ок, вот данные"
201 = "создал новый ресурс, вот данные (+ Location header)"
```

**Q: 401 vs 403?**

```
401 = "покажи логин/пароль"
403 = "логин ок, прав нет"
```

**Q: Когда 422, а не 400?**

```
400 = "неправильный JSON синтаксис"
422 = "JSON ок, но email уже занят"
```

## 💻 **Код обработки (React/TS)**

```typescript
const handleResponse = async (response: Response) => {
  switch (true) {
    case response.ok: // 200-299
      return response.json();

    case response.status === 401:
      redirectToLogin();
      break;

    case response.status === 403:
      showError("Нет доступа");
      break;

    case response.status === 404:
      showNotFound();
      break;

    default:
      throw new Error(`HTTP ${response.status}`);
  }
};
```

## 📏 **Частота использования (статистика)**

```
95% трафика: 200, 304, 404
API: 200, 201, 400, 422, 500
Auth: 401, 403
CDN: 301, 304
```

**Запомни формулу**: **2xx=успех, 3xx=иди туда, 4xx=твоя ошибка, 5xx=моя ошибка** 🚀
