## 🎯 Самые важные HTTP заголовки (Frontend/Backend)

### **🔐 CORS (Cross-Origin Resource Sharing)**

**CORS (Cross-Origin Resource Sharing)** — механизм, реализованный в браузерах, который применяет «политику одного происхождения» и расширяет её через набор заголовков, возвращаемых сервером. Браузер проверяет ответ сервера: если в нём есть такие заголовки, как Access-Control-Allow-Origin, Access-Control-Allow-Methods, Access-Control-Allow-Headers, и они допускают конкретный origin и тип запроса, тогда JavaScript-код сможет прочитать ответ. Если же заголовки не соответствуют правилам, браузер заблокирует доступ к ответу, хотя сам HTTP‑запрос технически может быть отправлен. Важно понимать, что CORS — это защита именно на стороне браузера; backend‑to‑backend запросы CORS не ограничивает.

| Заголовок                                    | Где                | Значение                      | Эффект                          |
| :------------------------------------------- | :----------------- | :---------------------------- | :------------------------------ |
| `Access-Control-Allow-Origin: *`             | **Ответ сервера**  | `*` или `https://example.com` | Какие домены могут читать ответ |
| `Access-Control-Allow-Methods: GET, POST`    | Ответ              | Разрешённые методы            | Какие методы доступны           |
| `Access-Control-Allow-Headers: Content-Type` | Ответ              | Разрешённые заголовки         | Какие headers можно слать       |
| `Origin: https://client.com`                 | **Запрос клиента** | Текущий домен                 | Откуда запрос                   |

**Проблема preflight** (OPTIONS):

Preflight-запрос в CORS — это механизм безопасности, который позволяет браузеру заранее уточнить у сервера, разрешён ли данный кросс-доменный запрос с точки зрения политики CORS. Браузер отправляет OPTIONS с указанием origin и параметров будущего запроса. Сервер отвечает заголовками Access-Control-Allow-Methods, Access-Control-Allow-Headers, Access-Control-Allow-Origin и, возможно, Access-Control-Max-Age. Если параметры разрешены, браузер уже затем выполняет «настоящий» GET, POST, PUT и т.д. Такой подход защищает ресурсы от неожиданных кросс-доменных вызовов и даёт серверу возможность чётко объявить, какие комбинации методов, заголовков и origin‑ов допустимы.

```
POST https://api.com/users (CORS)
1. OPTIONS api.com → проверка прав
2. GET 200 + заголовки CORS → основной запрос
```

**Код Express**:

```javascript
app.use((req, res, next) => {
  res.header("Access-Control-Allow-Origin", "https://client.com");
  res.header("Access-Control-Allow-Methods", "GET, POST, PUT");
  next();
});
```

### **💾 Cache-Control (Кэширование)**

| Директива      | Значение                    | Эффект                 |
| :------------- | :-------------------------- | :--------------------- |
| `max-age=3600` | 1 час                       | Кэшировать 1 час       |
| `no-cache`     | Проверять у сервера         | `If-None-Match` запрос |
| `no-store`     | **Не кэшировать**           | Всегда полный запрос   |
| `public`       | CDN/прокси могут кэшировать | Для статики            |
| `private`      | Только браузер              | Для авторизованных     |

**Примеры**:

```
Cache-Control: public, max-age=31536000, immutable  // JS/CSS на год
Cache-Control: no-cache                          // API (проверять ETag)
Cache-Control: no-store                          // Пароли/формы
```

### **🔑 Authorization (Авторизация)**

| Тип                          | Пример       | Где             |
| :--------------------------- | :----------- | :-------------- |
| `Basic base64(login:pass)`   | Логин:пароль | Legacy          |
| `Bearer eyJ...`              | JWT токен    | Современные API |
| `Authorization: Bearer null` | Logout       | Отправить null  |

### **📝 Content-Type / Accept**

| Заголовок                           | Значение   | Назначение      |
| :---------------------------------- | :--------- | :-------------- |
| `Content-Type: application/json`    | **Запрос** | "Я шлю JSON"    |
| `Accept: application/json`          | **Запрос** | "Хочу JSON"     |
| `Content-Type: multipart/form-data` | **Запрос** | Загрузка файлов |

### **🛡️ Безопасность**

| Заголовок                                     | Значение  | Защита от       |
| :-------------------------------------------- | :-------- | :-------------- |
| `X-Content-Type-Options: nosniff`             | **Ответ** | MIME sniffing   |
| `X-Frame-Options: DENY`                       | Ответ     | Clickjacking    |
| `Strict-Transport-Security: max-age=31536000` | Ответ     | HTTP → HTTPS    |
| `Referrer-Policy: strict-origin`              | Ответ     | Скрыть referrer |

**Express Helmet** (всё сразу):

```javascript
const helmet = require("helmet");
app.use(helmet());
```

### **📊 API Headers**

| Заголовок                         | Пример     | Использование      |
| :-------------------------------- | :--------- | :----------------- |
| `ETag: "abc123"`                  | **Ответ**  | Кэш валидация      |
| `If-None-Match: "abc123"`         | **Запрос** | "Есть обновления?" |
| `Last-Modified: Wed, 21 Oct 2025` | Ответ      | Дата изменения     |
| `If-Modified-Since`               | Запрос     | Проверка даты      |

### **🎯 Практические комбинации**

**1. Статические файлы (CDN)**:

```
Cache-Control: public, max-age=31536000, immutable
Content-Type: application/javascript
```

**2. API (данные пользователей)**:

```
Cache-Control: private, max-age=300
ETag: "user-123-v2"
```

**3. Форма логина**:

```
Cache-Control: no-store
Content-Type: application/json
```

**4. CORS + Auth API**:

```
Access-Control-Allow-Origin: https://app.com
Authorization: Bearer eyJ...
Content-Type: application/json
```

## 🚨 **Типичные собес-запросы**

**Q: Зачем `Access-Control-Allow-Origin: *` опасно?**
**Ответ**: Любой сайт может читать ваши данные (CSRF/XSS).

**Q: `Cache-Control: no-cache` vs `no-store`?**

```
no-cache → проверять у сервера (ETag)
no-store → никогда не кэшировать
```

**Q: Какой Content-Type для JSON?**

```
Content-Type: application/json
Accept: application/json
```

## 💻 **Проверка в DevTools**

```
Network → Headers → Response Headers
→ Cache-Control, Access-Control-Allow-Origin
→ ETag, Content-Type
```

**Правило**: **CORS=между доменами, Cache-Control=производительность, Content-Type=формат данных** 🚀
