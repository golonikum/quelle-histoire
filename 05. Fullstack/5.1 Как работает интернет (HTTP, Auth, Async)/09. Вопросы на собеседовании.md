# Какой ответ сервера с заголовком `Cache-Control` наиболее корректно настроен для статического ресурса, который редко меняется и может кэшироваться браузером и CDN до 1 года?

**Ответ:**

Для статических ресурсов вроде версионированных `.js` или `.css` файлов, которые меняются только при изменении имени файла (через хэш в имени), подходящая политика кэширования — долгий срок хранения и метка о неизменности. Заголовок `Cache-Control: public, max-age=31536000, immutable` говорит браузеру и промежуточным прокси, что ресурс можно кэшировать всеми участниками до 31536000 секунд (примерно 1 год) и он не изменится в течение этого времени. Флаг `immutable` дополнительно подсказывает браузеру не делать условных запросов для проверки свежести. Такая стратегия существенно снижает нагрузку на сервер и ускоряет загрузку, если используется грамотная схема версионирования ресурсов, при которой новый билд всегда получает новое имя файла.
