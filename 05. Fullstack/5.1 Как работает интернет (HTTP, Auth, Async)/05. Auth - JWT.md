# Auth JWT: Как устроен, где хранить (HttpOnly Cookie vs LocalStorage)

JWT (JSON Web Token) — стандартный способ stateless-аутентификации. Разберём структуру, принцип работы и где хранить токен безопасно.

## 🧩 **Структура JWT**

```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6Ik5pa2l0YSIsImlhdCI6MTUxNjIzOTAyMn0.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
       ↓              ↓                    ↓
   Header        Payload              Signature
```

### **1. Header (заголовок)**

```json
{
  "alg": "HS256", // Алгоритм подписи
  "typ": "JWT" // Тип токена
}
```

**Base64Url** → `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9`

### **2. Payload (данные)**

```json
{
  "sub": "user123", // ID пользователя
  "name": "Никита", // Данные
  "iat": 1730784000, // issuedAt (время создания)
  "exp": 1730797600, // expiration (время истечения)
  "role": "admin" // Роли/права
}
```

**⚠️ НЕ шифруется, только Base64!**

### **3. Signature (подпись)**

```
HMACSHA256(
  base64Url(header) + "." +
  base64Url(payload),
  secret_key
)
```

**Полная проверка**: `signature === HMACSHA256(header+payload, secret)`

## 🔄 **Как работает авторизация**

```
1. POST /login {email: "user@test.ru", password: "123"}
2. ✅ Сервер проверяет пароль → генерирует JWT
3. ← 200 {token: "eyJhbGciOiJIUzI1NiIs..."}
4. GET /api/profile → Authorization: Bearer <token>
5. ✅ Сервер: decode(token) → проверяет подпись + exp
6. ← 200 {name: "Никита", role: "admin"}
```

## 💾 **Где хранить JWT: HttpOnly Cookie vs LocalStorage**

| Хранение            | **✅ Плюсы**                                                                | **❌ Минусы**                                         | **XSS**         | **CSRF**        |
| :------------------ | :-------------------------------------------------------------------------- | :---------------------------------------------------- | :-------------- | :-------------- |
| **LocalStorage**    | ✅ Легко читать JS<br>✅ Работает с SPA                                     | ❌ **XSS = кража токена**<br>❌ Нужно вручную чистить | 🔴 **КРИТИЧНО** | ✅ Безопасно    |
| **HttpOnly Cookie** | ✅ **XSS безопасно**<br>✅ Автоматическая отправка<br>✅ Работает с refresh | ❌ CSRF уязвимость<br>❌ Сложнее SPA                  | ✅ Безопасно    | 🔴 **КРИТИЧНО** |
| **Secure Cookie**   | Лучшее из двух                                                              | Сложная настройка                                     | ✅              | ⚠️ + CSRF токен |

### **🎯 Рекомендация 2026: HttpOnly + Refresh Token**

```http
# 1. Login → Set-Cookie (HttpOnly)
Set-Cookie: access_token=eyJ...; HttpOnly; Secure; SameSite=Strict
Set-Cookie: refresh_token=eyJ...; HttpOnly; Secure; SameSite=Strict; Path=/refresh

# 2. Каждый запрос → Cookie автоматом
GET /api/profile
Cookie: access_token=eyJ...; refresh_token=eyJ...
```

**React код** (никаких localStorage):

```typescript
// Только fetch/axios — куки автоматом
const { data } = await axios.get("/api/profile");
// Cookie уходит сам!
```

## 🛡️ **Двойная защита: Access + Refresh**

```
Access Token: 15 минут  → /api/profile (короткий)
Refresh Token: 30 дней → /api/refresh (длинный, только для обновления)
```

**Flow**:

```
1. Access истёк (401) → /api/refresh (с refresh cookie)
2. ← Новый access_token (HttpOnly cookie)
3. Повторный запрос с новым access
```

## 🚨 **Критические уязвимости**

| Уязвимость | LocalStorage                            | HttpOnly Cookie                    |
| :--------- | :-------------------------------------- | :--------------------------------- |
| **XSS**    | `localStorage.getItem('token')` → кража | ✅ **Недоступно JS**               |
| **CSRF**   | ✅ Безопасно                            | `POST /transfer` от злоумышленника |

**Итоговая защита**:

```
HttpOnly Cookie + SameSite=Strict + CSRF-токен
```

## 💻 **Express.js реализация**

```javascript
// 1. Login
app.post("/login", (req, res) => {
  const user = verifyUser(req.body);
  const accessToken = jwt.sign({ userId: user.id }, ACCESS_SECRET, {
    expiresIn: "15m",
  });
  const refreshToken = jwt.sign({ userId: user.id }, REFRESH_SECRET, {
    expiresIn: "30d",
  });

  res.cookie("access_token", accessToken, {
    httpOnly: true,
    secure: true,
    sameSite: "strict",
  });
  res.cookie("refresh_token", refreshToken, {
    httpOnly: true,
    secure: true,
    sameSite: "strict",
  });
  res.json({ message: "ok" });
});

// 2. Middleware проверки
const auth = (req, res, next) => {
  const token = req.cookies.access_token;
  if (!token) return res.status(401).send();
  jwt.verify(token, ACCESS_SECRET, (err, user) => {
    if (err) return res.status(401).send();
    req.user = user;
    next();
  });
};
```

## 🎯 **Собес-формула**

**Middle**: "JWT = header.payload.signature, stateless, Base64 не шифрование"
**Senior**: "HttpOnly Cookie + Refresh Token + SameSite=Strict + CSRF защита" 🚀

**Правило**: **Никогда localStorage для JWT в продакшене**
