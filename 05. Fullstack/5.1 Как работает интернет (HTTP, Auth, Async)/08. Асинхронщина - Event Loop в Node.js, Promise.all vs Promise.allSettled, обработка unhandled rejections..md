## ⚡ **Асинхронщина Node.js — Event Loop + Promises (Middle+/Senior)**

### **🔄 Event Loop Node.js — 6 фаз**[^1]

```
1. ✅ timers: setTimeout(), setInterval()
2. ✅ pending callbacks: I/O завершения (файлы, DNS)
3. ✅ idle, prepare: внутренние Node.js
4. ✅ poll: новые I/O события (fs.readFile, fetch)
5. ✅ check: setImmediate()
6. ✅ close callbacks: сокеты, файлы закрыты
```

**Код-пример**:

```javascript
setTimeout(() => console.log("1"), 0);
setImmediate(() => console.log("2"));
console.log("0"); // Выполнится ПЕРВЫМ!

// 0 → 1 → 2 (timers → check)
```

### **🎯 Promise.all vs Promise.allSettled**

| **Promise.all**                  | **Promise.allSettled**             |
| :------------------------------- | :--------------------------------- |
| ✅ **Все** промисы `resolve`     | Все промисы **завершены**          |
| ❌ **Один** `reject` → `reject`  | Все `fulfilled`/`rejected`         |
| `[{status: 'fulfilled', value}]` | `[{status: 'fulfilled/rejected'}]` |

**Код**:

```javascript
const p1 = Promise.resolve(1);
const p2 = Promise.reject("Ошибка");
const p3 = Promise.resolve(3);

Promise.all([p1, p2, p3]).catch((err) => console.log("all:", err)); // "Ошибка" ❌

Promise.allSettled([p1, p2, p3]).then((results) => console.log(results));
// [
//   {status: 'fulfilled', value: 1},
//   {status: 'rejected', reason: 'Ошибка'},
//   {status: 'fulfilled', value: 3}
// ] ✅
```

### **🚨 Unhandled Promise Rejections**

**Node.js 15+**: `unhandledRejection` **крашит процесс**!

#### **✅ Правильная обработка**

```javascript
// 1. .catch()
fetch('/api').catch(err => logError(err)); ✅

// 2. process.on('unhandledRejection')
process.on('unhandledRejection', (reason, promise) => {
  console.error('🚨 Unhandled Rejection:', reason);
  process.exit(1); // Или graceful shutdown
}); ✅

// 3. В utils
const safeAsync = fn => async (...args) => {
  try {
    return await fn(...args);
  } catch (error) {
    logError(error);
  }
};

app.get('/users', safeAsync(async (req, res) => {
  const users = await db.users.findAll();
  res.json(users);
}));
```

### **🔥 Собес-вопросы (Middle+/Senior)**

**1. Порядок выполнения:**

```javascript
setTimeout(() => console.log("timeout"), 0);
Promise.resolve().then(() => console.log("promise"));
setImmediate(() => console.log("immediate"));
console.log("sync");
// sync → promise → timeout → immediate
```

**2. "Зачем `allSettled` вместо `all`?"**

```
✅ allSettled: логирование всех ошибок API
✅ all: "fire and forget" (все должны успех)
```
