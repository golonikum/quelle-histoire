# 🏗️ **NestJS Архитектура — 5 ключевых концепций**

## **1. 🧩 Модули (@Module()) — Контейнеры логики**

Модуль — класс с декоратором `@Module()`, который группирует связанные компоненты (контроллеры, сервисы, другие модули) в логический блок. Каждый модуль — контейнер зависимостей с собственным `IoC` scope.

**Назначение**: **Логическое разделение** приложения на независимые блоки (Users, Auth, Payments).

```typescript
@Module({
  imports: [AuthModule, DatabaseModule], // Зависимости (другие модули)
  controllers: [UsersController], // HTTP роуты
  providers: [UsersService, PrismaService], // IoC сервисы
  exports: [UsersService], // Публичный API модуля
})
export class UsersModule {}
```

**Зачем**: Feature slicing, lazy loading, переиспользование.

### **🔄 Жизненный цикл модуля**

```
1. NestJS создаёт ApplicationContext
2. Загружает Root Module (AppModule)
3. Рекурсивно загружает imports → imports → ...
4. Создаёт провайдеры (singleton по умолчанию)
5. Регистрирует контроллеры (HTTP роуты)
```

### **💎 5 типов модулей**

#### **1. Root Module (AppModule)**

```typescript
// src/app.module.ts
@Module({
  imports: [UsersModule, AuthModule],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {} // ✅ Корневой модуль
```

#### **2. Feature Module (UsersModule)**

```typescript
// src/users/users.module.ts
@Module({
  controllers: [UsersController],
  providers: [UsersService],
  exports: [UsersService], // ✅ Доступен импортерам
})
export class UsersModule {}
```

#### **3. Shared Module (CommonModule)**

```typescript
// src/common/common.module.ts
@Module({
  providers: [LoggerService, ValidationPipe],
  exports: [LoggerService, ValidationPipe], // ✅ Общие утилиты
})
export class CommonModule {}
```

#### **4. Global Module (@Global())**

```typescript
@Global() // Доступен ВЕЗДЕ без импорта!
@Module({
  providers: [ConfigService],
  exports: [ConfigService],
})
export class ConfigModule {}
```

#### **5. Dynamic Module (forRoot/forFeature)**

```typescript
// database/database.module.ts
@Module({})
export class DatabaseModule {
  static forRoot(entities: any[]) {
    return {
      module: DatabaseModule,
      providers: [
        /* Prisma config */
      ],
      exports: [PrismaService],
      global: true,
    };
  }
}

// app.module.ts
@Module({
  imports: [DatabaseModule.forRoot([User])],
})
export class AppModule {}
```

## **2. 🎯 Провайдеры (@Injectable()) — Бизнес-логика**

**Провайдер** — **класс, управляемый IoC-контейнером NestJS**. Декоратор `@Injectable()` **регистрирует** класс в контейнере и делает его **доступным для инъекции** в другие классы.

**Аналогия**: **Service в Angular**, **Bean в Spring** — единый контейнер управляет жизненным циклом.

**Назначение**: **Сервисы** с **Dependency Injection** (IoC контейнер).

```typescript
@Injectable() // Автоматически регистрируется
export class UsersService {
  constructor(private prisma: PrismaService) {} // ✅ DI

  async createUser(email: string) {
    return this.prisma.user.create({ data: { email } });
  }
}
```

**Что происходит**:

1. `@Injectable()` → NestJS создаёт **singleton** экземпляр
2. **Конструктор** → IoC контейнер **автоматически** внедряет зависимости
3. **Scope**: singleton **по умолчанию** (один экземпляр на приложение)

**IoC в действии**:

```typescript
@Controller("users")
export class UsersController {
  constructor(private usersService: UsersService) {} // Авто внедрение!
}
```

### **💉 Dependency Injection — 4 способа**

#### **1. Constructor Injection (рекомендуемый)**

```typescript
@Injectable()
export class UsersService {
  constructor(
    private prisma: PrismaService, // ✅ Тип определяет провайдер
    private emailService: EmailService, // ✅ Авто-инъекция
  ) {}
}
```

#### **2. Property Injection (@Inject())**

```typescript
@Injectable()
export class UsersService {
  @Inject(PrismaService) // ✅ Явное указание токена
  private prisma: PrismaService;
}
```

#### **3. Method Injection (параметры методов)**

```typescript
@Injectable()
export class UsersService {
  @Inject("DATABASE_CONNECTION")
  createUser(@Inject(PrismaService) prisma: PrismaService) {
    // Редко используется
  }
}
```

#### **4. Custom Token Injection (@Inject('TOKEN'))**

```typescript
// Модуль
@Module({
  providers: [
    {
      provide: "DATABASE_CONNECTION", // Кастомный токен
      useValue: prismaClient,
    },
  ],
  exports: ["DATABASE_CONNECTION"],
})
// Сервис
@Injectable()
export class UsersService {
  constructor(@Inject("DATABASE_CONNECTION") private db: any) {}
}
```

## **3. 🌐 Контроллеры (@Controller()) — HTTP слой**

**Контроллер** — **класс с декоратором `@Controller()`**, который **обрабатывает HTTP-запросы** и **возвращает ответы**. Связывает **внешний HTTP мир** с **внутренней бизнес-логикой** (сервисами).

**Задача**: **Маршрутизация + валидация → вызов сервиса → JSON ответ**.

```typescript
@Controller("users") // База /users
export class UsersController {
  @Get() // GET /users
  findAll() {}

  @Post()
  @Body(ValidationPipe) // Авто валидация DTO
  create(@Body() dto: CreateUserDto) {}

  @Get(":id")
  findOne(@Param("id") id: string) {}
}
```

### **🔧 Декораторы маршрутизации (7 штук)**

| Декоратор        | HTTP метод | Пример                                        |
| :--------------- | :--------- | :-------------------------------------------- |
| `@Get()`         | `GET`      | `@Get() findAll()` → `GET /users`             |
| `@Post()`        | `POST`     | `@Post() create()` → `POST /users`            |
| `@Put(':id')`    | `PUT`      | `@Put(':id') update()` → `PUT /users/1`       |
| `@Patch(':id')`  | `PATCH`    | `@Patch(':id') patch()` → `PATCH /users/1`    |
| `@Delete(':id')` | `DELETE`   | `@Delete(':id') remove()` → `DELETE /users/1` |
| `@All()`         | `ALL`      | `@All() catchAll()` → все методы              |
| `@HttpCode(201)` | Статус     | `@Post() @HttpCode(201) create()`             |

### **📥 Декораторы параметров (Request data)**

| Декоратор      | Доступ к        | Пример                                 |
| :------------- | :-------------- | :------------------------------------- |
| `@Body()`      | `req.body`      | `@Body() dto: CreateUserDto`           |
| `@Query()`     | `req.query`     | `@Query('page') page: number`          |
| `@Param('id')` | `req.params.id` | `@Param('id') id: string`              |
| `@Headers()`   | `req.headers`   | `@Headers('authorization') token`      |
| `@Req()`       | `req`           | `@Req() request: Request` (⚠️ редко)   |
| `@Res()`       | `res`           | `@Res() response: Response` (⚠️ редко) |

**Полный пример**:

```typescript
@Post(':id/photo')
uploadPhoto(
  @Param('id') userId: string,
  @Body('filename') filename: string,
  @Query('size') size: string,
  @Headers('x-api-key') apiKey: string,
) {
  return this.usersService.uploadPhoto(+userId, filename, size, apiKey);
}
// POST /users/123/photo?size=large
```

### **🔥 Swagger документация (автоматически)**

```typescript
@ApiTags('users')
@ApiOperation({ summary: 'Create user' })
@ApiResponse({ status: 201, description: 'User created' })
@Post()
create(@Body() dto: CreateUserDto) {
  return this.usersService.create(dto);
}
```

### **🚨 Антипаттерны**

```

❌ SQL в контроллере
❌ try/catch в контроллере
❌ res.json(), res.status()
```

## **4. 🔐 Guards (@UseGuards()) — Безопасность**

**Guards** — **функция-предохранитель**, которая **выполняется ДО контроллера** и **решает**: разрешить или **заблокировать** запрос. Реализует интерфейс `CanActivate`.

**Задача**: **Аутентификация** (есть токен?), **авторизация** (есть права?), **ограничения** (rate limit?).

**Аналогия**: **Дверной замок** — проверяет ключ ДО входа в дом.

### **🔧 Базовая реализация**

```typescript
import { Injectable, CanActivate, ExecutionContext } from "@nestjs/common";

@Injectable()
export class JwtAuthGuard implements CanActivate {
  constructor(private jwtService: JwtService) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const request = context.switchToHttp().getRequest();
    const token = this.extractToken(request);

    if (!token) return false;

    try {
      const payload = await this.jwtService.verifyAsync(token);
      request.user = payload;
      return true;
    } catch {
      return false;
    }
  }
}
```

### **📍 3 уровня применения Guards**

| Уровень        | Код                                    | Защищает            |
| :------------- | :------------------------------------- | :------------------ |
| **Method**     | `@UseGuards(AuthGuard)`                | **1 роут**          |
| **Controller** | `@UseGuards(AuthGuard)` (на классе)    | **Весь контроллер** |
| **Global**     | `app.useGlobalGuards(new AuthGuard())` | **Всё приложение**  |

#### **1. Method Guard (1 роут)**

```typescript
@Controller("users")
export class UsersController {
  @Get("profile")
  @UseGuards(AuthGuard) // Только этот роут
  getProfile(@Request() req) {
    return req.user; // user из AuthGuard
  }
}
```

#### **2. Controller Guard (весь контроллер)**

```typescript
@Controller("admin")
@UseGuards(AuthGuard) // Все роуты контроллера
export class AdminController {
  @Get("dashboard")
  getDashboard() {}

  @Post("settings")
  updateSettings() {}
}
```

#### **3. Global Guard (всё приложение)**

```typescript
// main.ts
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.useGlobalGuards(new AuthGuard()); // ✅ ВЕЗДЕ токен нужен
  await app.listen(3000);
}
```

### **⚙️ ExecutionContext — типы запросов**

| Тип           | Доступ                  | Пример       |
| :------------ | :---------------------- | :----------- |
| **HTTP**      | `switchToHttp()`        | REST API     |
| **WebSocket** | `switchToWs()`          | Сокеты       |
| **RPC/gRPC**  | `switchToRpc()`         | Микросервисы |
| **GraphQL**   | `getType() === 'query'` | GQL          |

```typescript
canActivate(context: ExecutionContext): boolean {
  const type = context.getType();

  if (type === 'http') {
    const request = context.switchToHttp().getRequest();
    // HTTP логика
  }

  if (type === 'ws') {
    const client = context.switchToWs().getClient();
    // WebSocket логика
  }
}
```

## **5. 🔧 Pipes — Валидация/трансформация**

**Pipes** — **функции-преобразователи**, которые **обрабатывают данные ДО контроллера**. Выполняют **валидацию**, **типизацию**, **трансформацию** входных параметров (`@Body()`, `@Param()`, `@Query()`).

**Аналогия**: **Фильтр для воды** — очищает/преобразует данные перед использованием.

**Задача**: `string → number`, `invalid JSON → 400 Bad Request`, `raw data → DTO`.

### **🔄 3 уровня применения Pipes**

| Уровень       | Код                                        | Применение         |
| :------------ | :----------------------------------------- | :----------------- |
| **Parameter** | `@Body(new ValidationPipe())`              | **1 аргумент**     |
| **Method**    | `@UsePipes(ValidationPipe)`                | **Весь метод**     |
| **Global**    | `app.useGlobalPipes(new ValidationPipe())` | **Всё приложение** |

### **📦 Встроенные Pipes (готовые)**

| Pipe                 | Что делает                | Пример                                            |
| :------------------- | :------------------------ | :------------------------------------------------ |
| **`ParseIntPipe`**   | `string → number`         | `@Param('id', ParseIntPipe) id: number`           |
| **`ParseBoolPipe`**  | `'true' → true`           | `@Query('active', ParseBoolPipe) active: boolean` |
| **`ParseUUIDPipe`**  | UUID валидация            | `@Param('uuid', ParseUUIDPipe) id: string`        |
| **`ValidationPipe`** | **DTO + class-validator** | `@Body(ValidationPipe) dto: CreateUserDto`        |

#### **ParseIntPipe пример**

```typescript
@Get(':id')
findOne(@Param('id', ParseIntPipe) id: number) {
  // /users/abc → 400 Bad Request (не число)
  // /users/123 → id = 123 ✅
  return this.usersService.findOne(id);
}
```

### **🛠️ ValidationPipe + DTO (самый популярный)**

#### **1. DTO с декораторами**

```typescript
// dto/create-user.dto.ts
import { IsEmail, IsString, MinLength, IsOptional } from "class-validator";

export class CreateUserDto {
  @IsEmail({}, { message: "Неверный email" })
  email: string;

  @IsString()
  @MinLength(3)
  name: string;

  @IsOptional()
  @IsString()
  avatar?: string;
}
```

#### **2. Controller с ValidationPipe**

```typescript
@Post()
async create(@Body(ValidationPipe) dto: CreateUserDto) {
  // dto.email уже валидный!
  return this.usersService.create(dto);
}
```

**Результат**:

```bash
# ❌ Неверные данные
curl -X POST /users -d '{"email":"invalid","name":"ab"}'
→ 400 { errors: [{field:'email', message:'Неверный email'}] }

# ✅ Корректные данные
curl -X POST /users -d '{"email":"test@test.ru","name":"Никита"}'
→ 201 {id:1, email:"test@test.ru"}
```

### **🌍 Global ValidationPipe (рекомендуется)**

```typescript
// main.ts
async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.useGlobalPipes(
    new ValidationPipe({
      whitelist: true, // Удаляет лишние поля
      forbidNonWhitelisted: true, // Запрещает лишние поля
      transform: true, // Авто string → number/boolean
      transformOptions: { enableImplicitConversion: true },
    }),
  );

  await app.listen(3000);
}
```

**Теперь везде работает**:

```typescript
// Больше не нужно @UsePipes!
@Post()
create(@Body() dto: CreateUserDto) {
  return this.usersService.create(dto);
}
```

### **🔨 Custom Pipes (пользовательские)**

```typescript
import { PipeTransform, Injectable, BadRequestException } from "@nestjs/common";

@Injectable()
export class PhonePipe implements PipeTransform<string> {
  transform(value: string): string {
    const cleaned = value.replace(/\D/g, ""); // Только цифры

    if (cleaned.length !== 11) {
      throw new BadRequestException("Неверный формат телефона");
    }

    return cleaned;
  }
}
```

**Использование**:

```typescript
@Post()
create(
  @Body('phone', PhonePipe) phone: string,  // +79991234567
  @Body() dto: CreateUserDto
) {
  return this.usersService.create({ ...dto, phone });
}
```

```typescript
export class CreateUserDto {
```

## **🔥 Типичные ошибки новичков**

```
❌ Service импортирует Controller
❌ Controller содержит бизнес-логику
❌ 1 AppModule со всем проектом
✅ Feature Modules + exports
```

**Формула**: **`@Module` = контейнер, `@Injectable` = DI, `@Controller` = HTTP, `@Guard` = security, `@Pipe` = validation** 🚀
