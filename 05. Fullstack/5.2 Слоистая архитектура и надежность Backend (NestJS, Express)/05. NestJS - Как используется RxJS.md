# Как используется RxJS в NestJS?

**RxJS в NestJS** используется везде, где нужна **реактивная обработка асинхронных данных**: Interceptors, Pipes, Guards, Observables в контроллерах, WebSocket'ы, EventEmitter.

## 🎯 **Где используется RxJS**

### **1. Interceptors (основное применение)**

```ts
@Injectable()
export class TransformInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(
      // ✅ next.handle() возвращает Observable!
      map((data) => ({
        // map — RxJS оператор
        data,
        statusCode: 200,
      })),
      tap((response) => Logger.log("Response sent")), // tap — побочные эффекты
    );
  }
}
```

### **2. Guards (реже, но можно)**

```ts
@Injectable()
export class RolesGuard implements CanActivate {
  canActivate(context: ExecutionContext): Observable<boolean> | Promise<boolean> | boolean {
    return from(this.reflector.getAllAndOverride('roles', [...])).pipe(
      map(roles => this.rolesService.hasRole(user, roles))
    );
  }
}
```

### **3. Pipes (трансформация данных)**

```ts
@Injectable()
export class PaginationPipe implements PipeTransform {
  transform(value: any, ...args: any[]): Observable<number[]> {
    return from([1, 2, 3]).pipe(map((x) => x * 2));
  }
}
```

## 🔥 **Реактивные контроллеры**

### **Stream ответы**

```ts
@Controller("events")
export class EventsController {
  @Get("stream")
  getEvents(): Observable<{ timestamp: number }> {
    return interval(1000).pipe(
      // Каждую секунду
      map(() => ({ timestamp: Date.now() })),
    );
  }
}
```

**Результат**: бесконечный SSE поток данных!

### **SSE (Server-Sent Events)**

```ts
@Get('sse')
sse(): Observable<MessageEvent> {
  return interval(2000).pipe(
    map(i => ({ data: { message: `Event #${i}` } } as MessageEvent))
  );
}
```

## 🛠️ **EventEmitter + RxJS**

```ts
@Injectable()
export class OrderService {
  constructor(
    private eventEmitter: EventEmitter2  // @nestjs/event-emitter
  ) {}

  async processOrder(order: OrderDto) {
    this.eventEmitter.emit('order.created', order);
  }
}

// Listener
@OnEvent('order.created')
async handleOrderCreated(payload: OrderDto) {
  await this.emailService.sendOrderConfirmation(payload);
}
```

## 🎯 **HttpService (встроенный RxJS)**

```ts
@Injectable()
export class ApiService {
  constructor(private http: HttpService) {}

  async getUsers(): Promise<User[]> {
    return this.http
      .get<User[]>("https://api.example.com/users")
      .pipe(
        map((response) => response.data),
        catchError((err) => throwError(() => new Error("API Error"))),
      )
      .toPromise();
  }
}
```

## 🚀 **Продвинутые примеры (мидл+)**

### **Retry + Timeout**

```ts
this.http.get("/api/data").pipe(
  retry(3), // 3 попытки
  timeout(5000), // 5 сек таймаут
  catchError((err) => of(null)), // Fallback
);
```

### **Debounce в реальном времени**

```ts
@Get('search')
search(query: string): Observable<SearchResult[]> {
  return fromEvent(document, 'input').pipe(
    debounceTime(300),        // Ждём 300мс паузы
    distinctUntilChanged(),   // Только новые значения
    switchMap(q => this.searchService.search(q))
  );
}
```

### **WebSocket (SocketGateway)**

```ts
@SubscribeMessage('events')
handleEvents(client: Socket, data: any): Observable<any> {
  return interval(1000).pipe(
    map(i => ({ eventId: i, data }))
  );
}
```

## 📊 **Последовательность RxJS в lifecycle запроса**

```
Запрос
  ↓
Middleware (Express, НЕ RxJS)
  ↓
Guard (boolean/Promise/Observable)
  ↓
Pipe (DTO → Observable)
  ↓
**Interceptor BEFORE** (Observable)
  ↓
Controller (Promise/Observable)
  ↓
**Interceptor AFTER** (Observable)
  ↓
Ответ
```

## 🧠 **Собеседование**

**Вопрос**: "Зачем RxJS в NestJS?"

```
Ответ:
1. ✅ Interceptors работают с Observable (next.handle())
2. ✅ Реактивные контроллеры (SSE, стримы)
3. ✅ HttpService возвращает Observable
4. ✅ WebSocket/Gateway — чисто RxJS
5. ✅ EventEmitter2 на RxJS
6. ✅ Pipes/Guards поддерживают Observable
```

**Ключ**: NestJS контроллеры могут возвращать **`Promise \| Observable \| значение`**, Nest автоматически подписывается и ждёт завершения!

**RxJS = реактивная основа асинхронности NestJS**!
