# **NestJS: Дополнительные концепции**

**Interceptors, Exception Filters, Custom Decorators, Middleware** — ключевые механизмы для создания **production-ready** приложений.

## 🎯 **1. Interceptors (@UseInterceptors)**

**Зачем**: Перехват **до и после** контроллера. RxJS Observable.

```ts
@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const now = Date.now();
    return next.handle().pipe(
      tap(() => console.log(`Request: ${Date.now() - now}ms`))
    );
  }
}

// Использование
@UseInterceptors(LoggingInterceptor)
@Get()
findAll() { }
```

**Типичные задачи**:

- Логирование времени выполнения
- Трансформация `{ data, statusCode }`
- Кеширование Redis
- SSE стримы

## 🔥 **2. Exception Filters (@UseFilters / @Catch)**

**Зачем**: Централизованная **обработка ошибок** с кастомным HTTP статусом.

```ts
@Injectable()
export class HttpExceptionFilter implements ExceptionFilter {
  catch(exception: any, host: ArgumentsHost): void {
    const ctx = host.switchToHttp();
    const response = ctx.getResponse();
    const status = exception.getStatus ? exception.getStatus() : 500;

    response.status(status).json({
      statusCode: status,
      timestamp: new Date().toISOString(),
      message: exception.message,
    });
  }
}

// Глобально (main.ts)
app.useGlobalFilters(new HttpExceptionFilter());
```

**Пример с валидацией**:

```ts
catch(exception: ValidationErrorException, host: ArgumentsHost) {
  const errors = exception.errors.map(err => ({
    field: err.property,
    message: Object.values(err.constraints).join(', ')
  }));

  response.status(400).json({ errors });
}
```

## 🛠️ **3. Custom Decorators**

**Зачем**: **Переиспользуемые декораторы** с метаданными (Reflector API).

```ts
// decorators/current-user.decorator.ts
export const CurrentUser = createParamDecorator(
  (data: unknown, ctx: ExecutionContext) => {
    const request = ctx.switchToHttp().getRequest();
    return request.user;  // Из JWT Guard
  }
);

// Использование
@Get(':id')
getUser(@CurrentUser() user: User, @Param('id') id: string) {
  return this.usersService.findOne(id, user.id);
}
```

**Бизнес-декоратор**:

```ts
export const Roles = (...roles: Role[]) =>
  SetMetadata('roles', roles);

@Roles('admin')
@Put(':id')
update(@Param('id') id: string) { }
```

## 📡 **4. Middleware (configure() → NestModule)**

**Зачем**: **Низкоуровневый** доступ к `req/res` (CORS, helmet, rate-limit).

```ts
@Module({})
export class AppModule implements NestModule {
  configure(consumer: MiddlewareConsumer) {
    consumer
      .apply(helmet(), cors(), rateLimit())
      .exclude("health")
      .forRoutes(UsersController);
  }
}
```

## 🎯 **Последовательность выполнения**

```
Запрос
  ↓
**1. Global Middleware** (helmet, CORS)
  ↓
**2. Route Middleware** (app.module.ts)
  ↓
**3. Guards** (@UseGuards)
  ↓
**4. Pipes** (@Body(ValidationPipe))
  ↓
**5. Interceptor BEFORE** (@UseInterceptors)
  ↓
**Controller Method**
  ↓
**6. Interceptor AFTER**
  ↓
**7. Exception Filter** (если ошибка)
  ↓
Ответ
```

## 🏗️ **Практический пример: Production App**

```ts
// 1. Custom Decorator
export const ApiResponse = createParamDecorator((data, ctx) => {
  return { data, statusCode: 200, timestamp: new Date() };
});

// 2. Interceptor (трансформация)
@UseInterceptors(TransformInterceptor)
@UseGuards(JwtAuthGuard)
@Controller("users")
export class UsersController {
  @Get()
  @UseFilters(HttpExceptionFilter)
  findAll(@CurrentUser() user: User) {
    return this.usersService.findAll(user.id);
  }
}

// 3. Exception Filter (ошибки)
@Catch(HttpException)
export class AllExceptionsFilter implements ExceptionFilter {}

// 4. Middleware (CORS + RateLimit)
export class AppModule implements NestModule {
  configure(consumer: MiddlewareConsumer) {
    consumer.apply(cors(), helmet()).forRoutes("*");
  }
}
```

## 📊 **Сравнение механизмов**

| Механизм             | Выполняется     | Доступ к           | RxJS | Типичное применение |
| :------------------- | :-------------- | :----------------- | :--- | :------------------ |
| **Middleware**       | **До роутинга** | `req/res`          | ❌   | CORS, helmet        |
| **Guard**            | До контроллера  | `ExecutionContext` | ✅   | Аутентификация      |
| **Pipe**             | **DTO**         | `value/метаданные` | ✅   | Валидация           |
| **Interceptor**      | **До+После**    | `Observable`       | ✅   | Логирование, кеш    |
| **Exception Filter** | **Ошибка**      | `exception/host`   | ❌   | Обработка ошибок    |

## 🚀 **Глобальная настройка (main.ts)**

```ts
async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  // Глобальные механизмы
  app.useGlobalPipes(new ValidationPipe());
  app.useGlobalInterceptors(new TransformInterceptor());
  app.useGlobalFilters(new AllExceptionsFilter());

  await app.listen(3000);
}
```

## 🧠 **Собеседование: ключевые отличия**

**Вопрос**: "Interceptor vs Middleware?"

```
Ответ:
- **Middleware**: До роутинга, Express `req/res/next()`
- **Interceptor**: До+После контроллера, RxJS Observable
```

**Вопрос**: "Зачем Exception Filters?"

```
Ответ: Централизованная обработка ВСЕХ ошибок с единым HTTP форматом
```
