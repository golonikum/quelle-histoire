**IoC (Inversion of Control)** — принцип проектирования, при котором управление созданием, конфигурацией и жизненным циклом объектов передаётся от приложения контейнеру или фреймворку, а не разработчику. `IoC` позволяет избавиться от жёсткой связи и сосредоточиться на бизнес-логике, а не на создании объектов.

**DI (Dependency Injection)** — паттерн проектирования, который реализует `IoC`. Он позволяет контейнеру внедрять зависимости (объекты, которые нужны классу для работы) в класс, вместо того чтобы класс сам управлял этими зависимостями.

## 🚀 NestJS — Основы

### 🎯 Что такое NestJS

**NestJS** — **Angular для Node.js**. Фреймворк **TypeScript-first** для создания **масштабируемых** серверных приложений. Использует **декораторы**, **IoC контейнер**, **модульную архитектуру**.

```
Express.js: минималистичный → самописный код
NestJS: Angular-like → enterprise-ready
```

### 🏗️ Архитектура (5 ключевых концепций)

| Компонент         | Назначение                | Аналог Angular |
| :---------------- | :------------------------ | :------------- |
| **@Module()**     | Контейнер логики          | NgModule       |
| **@Controller()** | HTTP роуты                | @Component     |
| **@Injectable()** | Бизнес-логика             | @Service       |
| **@Pipe()**       | Валидация/трансформация   | @Pipe          |
| **@Guard()**      | Middleware/аутентификация | @CanActivate   |

### 💻 Минимальный пример (CRUD Users)

#### **1. app.module.ts** (корневой модуль)

```typescript
import { Module } from "@nestjs/common";
import { AppController } from "./app.controller";
import { AppService } from "./app.service";
import { UsersModule } from "./users/users.module";

@Module({
  imports: [UsersModule], // Подключаем модули
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

#### **2. users/users.module.ts**

```typescript
import { Module } from "@nestjs/common";
import { UsersController } from "./users.controller";
import { UsersService } from "./users.service";

@Module({
  controllers: [UsersController],
  providers: [UsersService],
})
export class UsersModule {}
```

#### **3. users/users.service.ts** (бизнес-логика)

```typescript
import { Injectable } from "@nestjs/common";

@Injectable()
export class UsersService {
  private users = [{ id: 1, name: "Никита", email: "nikita@test.ru" }];

  findAll() {
    return this.users;
  }

  findOne(id: number) {
    return this.users.find((user) => user.id === id);
  }
}
```

#### **4. users/users.controller.ts** (HTTP)

```typescript
import {
  Controller,
  Get,
  Post,
  Body,
  Param,
  ParseIntPipe,
} from "@nestjs/common";
import { UsersService } from "./users.service";

@Controller("users") // /api/users
export class UsersController {
  constructor(private usersService: UsersService) {} // ✅ IoC

  @Get() // GET /users
  findAll() {
    return this.usersService.findAll();
  }

  @Get(":id") // GET /users/1
  findOne(@Param("id", ParseIntPipe) id: number) {
    return this.usersService.findOne(id);
  }
}
```

**Результат**:

```
GET  http://localhost:3000/users        → [{id:1, name:"Никита"...}]
GET  http://localhost:3000/users/1      → {id:1, name:"Никита"...}
```

### 🔥 Продвинутые возможности

#### **1. DTO + Валидация (class-validator)**

```typescript
// dto/create-user.dto.ts
import { IsEmail, IsString, IsNotEmpty } from 'class-validator';

export class CreateUserDto {
  @IsEmail()
  email: string;

  @IsString()
  @IsNotEmpty()
  name: string;
}

// controller
@Post()
create(@Body() createUserDto: CreateUserDto) {
  return this.usersService.create(createUserDto);
}
```

#### **2. Guards (аутентификация)**

```typescript
// guards/auth.guard.ts
import { Injectable, CanActivate, ExecutionContext } from '@nestjs/common';

@Injectable()
export class AuthGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest();
    return !!request.headers.authorization; // JWT проверка
  }
}

// controller
@Get('profile')
@UseGuards(AuthGuard)
getProfile() { /* */ }
```

#### **3. Pipes (валидация параметров)**

```typescript
@Get(':id')
findOne(@Param('id', ParseIntPipe) id: number) {
  // id всегда number!
}
```

#### **4. Prisma/TypeORM интеграция**

```typescript
// users/users.service.ts
import { Injectable } from "@nestjs/common";
import { PrismaService } from "../prisma/prisma.service";

@Injectable()
export class UsersService {
  constructor(private prisma: PrismaService) {} // IoC!

  async findAll() {
    return this.prisma.user.findMany();
  }
}
```
