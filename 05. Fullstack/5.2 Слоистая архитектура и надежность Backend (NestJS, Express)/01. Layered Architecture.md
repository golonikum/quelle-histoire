## 🏗️ **Layered Architecture — Четкое разделение слоев**

**Принцип**: Каждый слой имеет **строгую ответственность**, общается **только с соседними** слоями, **зависимости только вниз**.

```
HTTP → Controllers → Services → Repositories → DB/PostgreSQL/Redis
           ↑            ↓            ↓              ↓
        валидация     бизнес       домен           I/O
        логика        логика
```

### **📊 Слои — роли и ответственность**

| Слой             | Назначение        | Что делает                                | ✅ Примеры                       | ❌ НЕ делает               |
| :--------------- | :---------------- | :---------------------------------------- | :------------------------------- | :------------------------- |
| **Controllers**  | **Входная точка** | Приём HTTP, **валидация**, ответы         | `req.body`, `zod`, `status(400)` | Бизнес-логика, SQL запросы |
| **Services**     | **Оркестрация**   | Сборка данных из репозиториев, транзакции | `userService.create()`           | SQL, валидация             |
| **Repositories** | **I/O адаптер**   | CRUD операции, **SQL/ORM**                | `prisma.user.findUnique()`       | Бизнес-логика              |

### **💻 Полная реализация Express + Prisma**

#### **📁 Структура проекта**

```
src/
├── controllers/   # HTTP handlers
├── services/      # Бизнес-логика
├── repositories/  # БД запросы
└── types/         # DTOs, схемы
```

#### **1. Controllers — только HTTP + валидация**

```javascript
// controllers/userController.js
const { createUserSchema } = require("../types/userSchema");
const userService = require("../services/userService");

class UserController {
  async createUser(req, res, next) {
    try {
      // ✅ ТОЛЬКО валидация + вызов сервиса
      const validatedData = createUserSchema.parse(req.body);

      const user = await userService.createUser(validatedData);

      res.status(201).json({
        success: true,
        data: { id: user.id, email: user.email },
      });
    } catch (error) {
      if (error.name === "ZodError") {
        return res.status(400).json({ errors: error.errors });
      }
      next(error);
    }
  }

  async getUser(req, res) {
    // ✅ Только id + вызов сервиса
    const { id } = req.params;
    const user = await userService.getUserById(Number(id));

    if (!user) {
      return res.status(404).json({ error: "User not found" });
    }

    res.json({ data: user });
  }
}

module.exports = new UserController();
```

#### **2. Services — бизнес-логика + оркестрация**

```javascript
// services/userService.js
const userRepository = require("../repositories/userRepository");
const emailService = require("./emailService");

class UserService {
  async createUser({ email, password, name }) {
    // ✅ БИЗНЕС-ЛОГИКА
    if (await userRepository.emailExists(email)) {
      throw new Error("Email already exists");
    }

    const hashedPassword = await hashPassword(password);

    // ✅ Оркестрация нескольких репозиториев
    const user = await userRepository.create({
      email,
      password: hashedPassword,
      name,
    });

    // ✅ Бизнес-события
    await emailService.sendWelcomeEmail(user);

    return user;
  }

  async getUserById(id) {
    // ✅ Композиция данных
    const user = await userRepository.findById(id);
    const profile = await userRepository.getUserProfile(id);

    return { ...user, profile };
  }
}

module.exports = new UserService();
```

#### **3. Repositories — только БД**

```javascript
// repositories/userRepository.js
const prisma = require("../db/prisma");

class UserRepository {
  async create(userData) {
    // ✅ ТОЛЬКО SQL/Prisma
    return prisma.user.create({ data: userData });
  }

  async findById(id) {
    return prisma.user.findUnique({
      where: { id: Number(id) },
      select: { id: true, email: true, name: true },
    });
  }

  async emailExists(email) {
    const user = await prisma.user.findUnique({ where: { email } });
    return !!user;
  }
}

module.exports = new UserRepository();
```

### **🔄 Поток запроса (Dependency Rule)**

```
POST /users → Controller → Service → Repository → Prisma → PostgreSQL
                           ↑________________________|
                           Сборка ответа ↑
```

```
Controllers ← НЕ ЗНАЮТ → Repositories (нарушает архитектуру!)
Controllers → ЗНАЮТ → Services → ЗНАЮТ → Repositories ✅
```

### **✅ Преимущества архитектуры**

| Проблема          | Без слоёв           | Слои                             |
| :---------------- | :------------------ | :------------------------------- |
| **Тестирование**  | Мокать HTTP + БД    | Тестировать Service изолированно |
| **Смена БД**      | Rewrite Controllers | Меняем только Repository         |
| **Бизнес-логика** | Размазана по файлам | Централизованно в Services       |
| **Валидация**     | Везде дублируется   | Только в Controllers             |

### **🚨 Типичные собес-вопросы**

**Q: "Зачем Service если Repository может всё?"**

```
Repository: "SELECT * FROM users"
Service: "проверить права → email уникален → хэш пароля → отправить письмо"
```

**Q: "Где транзакции?"**

````
✅ Service (оркестрация нескольких Repository)
```javascript
async createOrderWithPayment(orderData, paymentData) {
  return prisma.$transaction(async (tx) => {
    const order = await orderRepository.create(tx, orderData);
    const payment = await paymentRepository.create(tx, paymentData);
    return { order, payment };
  });
}
````

**Q: "Controller 200 строк — нормально?"**

```
❌ НЕТ! >50 строк = в Service
```

### **🎯 Dependency Injection (DI)**

```javascript
// services/userService.js
class UserService {
  constructor(userRepo, emailService) {
    this.userRepo = userRepo;
    this.emailService = emailService;
  }
}

// app.js
const userRepo = new UserRepository(prisma);
const emailSvc = new EmailService();
const userService = new UserService(userRepo, emailSvc);
```

### **💎 Senior-level признаки**

```
✅ Controllers < 50 строк
✅ Services содержат бизнес-логику
✅ Repositories — чистый SQL/ORM
✅ Нет прямых импортов Repository в Controller
✅ Unit тесты для каждого слоя
```

**Правило**: **Controller=валидация, Service=бизнес, Repository=БД** 🚀
