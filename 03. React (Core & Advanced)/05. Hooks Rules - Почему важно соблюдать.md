# Hooks Rules: Почему важно соблюдать

Правила хуков (Hooks Rules) обязательны, потому что React **не использует имена хуков** для хранения их состояния между ререндерами. Вместо этого он полагается на **порядок вызовов** — хуки хранятся в связанном списке (linked list) в `fiber node`.

## Почему порядок критичен

React ассоциирует состояние так:
**1-й вызов `useState` = первый слот**
**2-й вызов `useEffect` = второй слот**
При каждом рендере React воспроизводит **тот же порядок**.

**Неправильно** (порядок меняется):

```jsx
function BadComponent({ show }) {
  if (show) {
    const [count, setCount] = useState(0); // ❌ 1-й слот
    return <div>{count}</div>;
  }
  const [name, setName] = useState(""); // ❌ Становится 1-м слотом!
  return <div>{name}</div>;
}
```

**Результат**: `count` получает значение `name`, `name` — undefined. Баги, краши, stale state.

**Правильно** (всегда тот же порядок):

```jsx
function GoodComponent({ show }) {
  const [count, setCount] = useState(0); // ✅ Всегда 1-й
  const [name, setName] = useState(""); // ✅ Всегда 2-й

  return <div>{show ? <div>{count}</div> : <div>{name}</div>}</div>;
}
```

## Два правила и их последствия

| Правило                    | ❌ Нарушение             | ✅ Соблюдение             |
| :------------------------- | :----------------------- | :------------------------ |
| **Только верхний уровень** | `if`, циклы, `try/catch` | Всегда до `return`        |
| **Только React-функции**   | Обычные JS-функции       | Компоненты + custom hooks |

**В обычной функции**:

```jsx
// ❌ НЕ работает
function validateForm(data) {
  const [errors, setErrors] = useState({}); // React не знает эту функцию!
}
```

**В custom hook** (работает):

```jsx
// ✅ Правильно
function useValidation(data) {
  const [errors, setErrors] = useState({});
  // Логика
  return [errors, validate];
}
```

## ESLint и диагностика

```json
// .eslintrc
{
  "plugins": ["react-hooks"],
  "rules": {
    "react-hooks/rules-of-hooks": "error",
    "react-hooks/exhaustive-deps": "warn"
  }
}
```

Автоматически ловит 99% нарушений.

## Что происходит при нарушении

```
Рендер 1: useState(A) → queue[^0] = A, useEffect(B) → queue[^1] = B
Рендер 2: if(false) → пропуск useState(A), useEffect(B) → queue[^0] = B ❌
Рендер 3: if(true) → useState(A) → queue[^0] = A (было B!), useEffect(B) → queue[^1] = B
```

**Состояния "путешествуют"** между хуками → непредсказуемые баги.

**Правило как мантра**: _"Только сверху вниз, только в React-функциях"_. Нарушение = гарантированные production-баги.
