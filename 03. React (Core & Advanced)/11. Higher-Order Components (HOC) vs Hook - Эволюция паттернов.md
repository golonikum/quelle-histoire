# Higher-Order Components (HOC) vs Hooks: Эволюция паттернов.

**HOC vs Hooks** — это **эволюция паттернов переиспользования логики** в React. HOC появились первыми (2015), Hooks (2018) их **заменили**, но не полностью.

## Что такое HOC (Higher-Order Component)

**HOC** — функция, которая **берёт компонент → возвращает улучшенный компонент**:

```jsx
// withData.js — HOC для загрузки данных
const withData = (WrappedComponent) => {
  return class extends React.Component {
    state = { data: null, loading: true };

    componentDidMount() {
      fetch("/api/data").then((res) =>
        this.setState({ data: res, loading: false }),
      );
    }

    render() {
      return <WrappedComponent {...this.props} data={this.state.data} />;
    }
  };
};

// Использование
const PostsList = ({ data }) => (
  <ul>
    {data.map((d) => (
      <li>{d.title}</li>
    ))}
  </ul>
);
const PostsListWithData = withData(PostsList);
```

## Проблемы HOC ("HOC Hell")

```jsx
// Плохо: вложенность растёт
const PostsListWithAuth = withAuth(PostsListWithData);
const PostsListWithLoading = withLoading(PostsListWithAuth);
const FinalPostsList = withErrorHandling(PostsListWithLoading); // 😵‍💫
```

**Минусы HOC**:

- **Prop drilling** через HOC → компонент
- **Сложно комбинировать** (wrapper hell)
- **Static анализ** ломается
- **Тестирование** — сложно изолировать логику

## Hooks — революция (React 16.8)

**Hooks** — **функции**, которые **"цепляются"** к состоянию React прямо в компоненте:

```jsx
// useData.js — кастомный хук
function useData(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    setLoading(true);
    fetch(url)
      .then((res) => res.json())
      .then(setData)
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading };
}

// Использование — просто!
function PostsList() {
  const { data, loading } = useData("/api/posts");

  if (loading) return <div>⏳ Загрузка...</div>;
  return (
    <ul>
      {data.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
```

## Сравнение на реальном примере

| Аспект             | HOC                       | Hooks                                |
| :----------------- | :------------------------ | :----------------------------------- |
| **Код**            | `<PostsWithData />`       | `const { data } = useData()`         |
| **Комбинирование** | `withLoading(withAuth())` | `useAuth(); useData(); useLoading()` |
| **Читабельность**  | 🤮 Wrapper hell           | ✅ Линейный                          |
| **Тестирование**   | Сложно                    | Просто                               |
| **Props**          | Автоматически             | Явно                                 |

```jsx
// HOC — 20 строк обёрток
const FinalPosts = withError(withLoading(withAuth(PostsList)));

// Hooks — 5 строк
function FinalPosts() {
  const auth = useAuth();
  const { data, loading } = useData("/posts");
  const error = useError();

  if (!auth.user) return <Login />;
  if (loading) return <Spinner />;
  if (error) return <ErrorMsg />;

  return <PostsList data={data} />;
}
```

## Эволюция паттернов

```
React 15: Class Components + HOC + Render Props
        ↓ проблемы: wrapper hell, prop confusion
React 16.8: Hooks (useState, useEffect, useCallback...)
         ↓ проще, чище, мощнее
React 18+: Hooks + Concurrent (useTransition, Suspense)
```

## Когда HOC ещё живы?

**✅ Редкие кейсы**:

```jsx
// 1. Error Boundaries (только классы!)
class ErrorBoundary extends React.Component {
  componentDidCatch() { ... }
}

// 2. Legacy код миграция
const withRouter = (Component) => (props) => (
  <Router.Consumer>{ctx => <Component {...props} router={ctx} />}</Router.Consumer>
);
```

## Современная архитектура (2026)

```
Hooks (95%)       → useAuth, useData, useForm
Context API       → глобальное состояние
Custom Render Props → редкие случаи JSX
HOC               → legacy + Error Boundaries
```

## Миграция HOC → Hooks (шаблон)

```jsx
// Было HOC
const withAuth = (Component) => (props) => (
  <AuthConsumer>{(auth) => <Component {...props} auth={auth} />}</AuthConsumer>
);

// Стало Hook
function useAuth() {
  const auth = useContext(AuthContext);
  return auth;
}
```

**Hooks победили**, потому что **ближе к данным**, **проще комбинировать**, **легче тестировать**. HOC — **исторический артефакт**.
