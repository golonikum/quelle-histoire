# Keys - Зачем они нужны при рендере списков

Передача индекса как `key` в списках React кажется удобной, но приводит к проблемам с reconciliation (сопоставлением DOM), состоянием и производительностью при любых изменениях списка. React использует ключи для идентификации элементов, а индекс нестабилен при мутациях.[^1][^2]

## Как работает Reconciliation

React во время обновления сравнивает Virtual DOM по дереву:

1. **Ключи определяют тип**: один и тот же key = один и тот же компонент (с сохранением state/focus).
2. **Индексы ломаются при сдвиге**: React думает, что элемент "переместился", а не изменился.[^2]

## Проблемы с примерами

### 1. Перетаскивание/сортировка

```jsx
// Данные: [{id:1, text:'A'}, {id:2, text:'B'}]
items.sort().map((item, index) => <Item key={index} value={item.text} />);
```

**До**: A(0), B(1)
**После сортировки** (B перед A): B(0), A(1)
React: "Элемент 0 был A, стал B → полная перерисовка + state A переходит к B". Фокус/инпуты перепрыгивают.[^1]

### 2. Добавление в начало

```jsx
// Добавляем {id:0, text:'New'}
items.unshift(newItem); // индексы сдвигаются
```

React перерендерит **все** элементы после вставки, т.к. каждый key изменился.[^2]

### 3. Удаление

```jsx
items.splice(1, 1); // удаляем второй
```

Оставшиеся элементы получают новые индексы → React думает, что они "другие".[^6]

## Сравнение key

| Сценарий            | `key={index}`       | `key={item.id}`      |
| :------------------ | :------------------ | :------------------- |
| Статичный список    | ✅ Работает         | ✅ Работает          |
| Сортировка          | ❌ State мигрирует  | ✅ State остаётся    |
| Добавление/удаление | ❌ Перерендер всего | ✅ Только изменённые |
| Фокус в input       | ❌ Перепрыгивает    | ✅ Сохраняется       |

## Доказательство на коде

```jsx
// Плохо
const BadList = ({ items }) => (
  <ul>
    {items.map((item, index) => (
      <li key={index}>
        <input defaultValue={item.text} />
      </li>
    ))}
  </ul>
);

// Хорошо
const GoodList = ({ items }) => (
  <ul>
    {items.map((item) => (
      <li key={item.id}>
        <input defaultValue={item.text} />
      </li>
    ))}
  </ul>
);
```

При перетаскивании во втором input фокус не теряется, в первом — да.

## Когда индекс допустим

```
Только для **статичных** списков без сортировки/фильтрации/добавлений: `['a','b','c'].map((v,i) => <li key={i}>{v}</li>)`. Иначе всегда `item.id` или `uuid`.
```
