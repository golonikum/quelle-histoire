# Memoization: React.memo, useMemo, useCallback — когда и зачем использовать

Мемоизация в React предотвращает **ненужные ререндеры** и **тяжелые вычисления**, сохраняя стабильные ссылки на значения/функции между рендерами. Используйте **осознанно** — по профайлеру, а не "профилактически".

## React.memo — компоненты

Предотвращает ререндер, если **props не изменились** (shallow comparison).

```jsx
const Child = React.memo(({ count, onClick }) => {
  console.log("Child render");
  return <button onClick={onClick}>{count}</button>;
});
```

**Когда использовать**: Частые ререндеры от родителей, props примитивы или стабильно мемоизированы.

**⚠️ Ловушка**: Объекты/функции в props создают **новую ссылку каждый рендер** → `memo` бесполезен без `useMemo/useCallback`.

## useMemo — значения

Мемоизирует **результат** дорогих вычислений или объекты/массивы.

```jsx
// ❌ Тяжелый фильтр каждый рендер
const visibleTodos = todos.filter((todo) => todo.active);

// ✅ Считается только при изменении зависимостей
const visibleTodos = useMemo(
  () => todos.filter((todo) => todo.active),
  [todos],
);
```

**Когда использовать**:

- Фильтрация/сортировка больших массивов
- Создание объектов `{ filtered: true }` для props
- Регулярки, парсинг JSON

## useCallback — функции

`useCallback(fn, deps)` = `useMemo(() => fn, deps)` для функций.

```jsx
// ❌ Новая функция → Child ререндерится
const handleClick = () => setCount((c) => c + 1);

// ✅ Стабильная ссылка
const handleClick = useCallback(() => setCount((c) => c + 1), []);
```

**Когда использовать**:

- Функции в `dependencies` других хуков: `useEffect(() => {}, [handleClick])`
- Функции-пропсы для `React.memo` компонентов

## Правило "тройного применения"

```
React.memo(Child)  +  useMemo(props)  +  useCallback(onClick)
```

```jsx
const Parent = () => {
  const [count, setCount] = useState(0);

  // Стабильная функция
  const increment = useCallback(() => setCount((c) => c + 1), []);

  // Стабильный объект props
  const childProps = useMemo(() => ({ count, increment }), [count, increment]);

  return <ChildMemo {...childProps} />;
};

const ChildMemo = React.memo(({ count, increment }) => {
  console.log("Child render"); // 🔥 Только при изменении count!
  return <button onClick={increment}>{count}</button>;
});
```

## Когда НЕ использовать

| ❌ Пустые deps `[ ]` везде | ✅ Только по необходимости          |
| :------------------------- | :---------------------------------- |
| `useCallback(noop, [])`    | `React.memo(ExpensiveChild)`        |
| `useMemo(примитив, [x])`   | `useMemo(фильтр1000items, [items])` |

**Метрика**: В хорошем приложении **<5% компонентов** в `React.memo`, **<10% хуков** — мемо.[^1]

**Профайлер**: React DevTools → найди ререндеры → примени memo **точечно**.
