# Concurrent Mode: useTransition, useDeferredValue, Suspense

**React Fiber** — это **фундамент**, на котором строятся **Concurrent Mode**, **Suspense** и **useTransition**. Вместе они решают главную проблему: **тяжёлые обновления не должны блокировать UI**.

## Concurrent Mode (режим конкурентности)

**Fiber** позволяет React **делать несколько вещей одновременно**:

```
1. Показать "Кнопка нажата" (высокий приоритет)
2. Фильтровать 10 000 товаров (низкий приоритет)
3. Сохранить черновик (ещё ниже)
```

**До React 18**: `setFilter('expensive')` → UI зависает на 300мс
**С Concurrent**: React сначала покажет изменения кнопки, потом медленно отфильтрует.

## useTransition — "Не торопись с тяжёлым"

```jsx
function App() {
  const [query, setQuery] = useState("");
  const [filteredItems, setFilteredItems] = useState([]); // ← Отдельное состояние!
  const [items, setItems] = useState([]);
  const [isPending, startTransition] = useTransition();

  useEffect(() => {
    // 10 000 товаров
    const bigList = Array.from({ length: 10000 }, (_, i) => ({
      id: i,
      name: `Товар #${i + 1} - Смартфон iPhone ${i % 10}`,
    }));
    setItems(bigList);
  }, []);

  const handleChange = (e) => {
    const value = e.target.value;

    // 1. Input обновляется СРАЗУ (высокий приоритет)
    setQuery(value);

    // 2. ТЯЖЁЛЫЙ ФИЛЬТР — низкий приоритет
    startTransition(() => {
      // 🔥 ВОТ ЗДЕСЬ тяжёлая работа!
      const filtered = items.filter((item) =>
        item.name.toLowerCase().includes(value.toLowerCase()),
      );
      setFilteredItems(filtered); // ← Обновление состояния!
    });
  };

  return (
    <div>
      <input onChange={handleChange} value={query} />
      {isPending && <div>⏳ Фильтруем...</div>}
      <ul>
        {filteredItems.slice(0, 20).map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
}
```

**Что происходит**:

1. Пользователь печатает → input обновляется **мгновенно**
2. Фильтр 10k элементов → показываем "⏳" → UI отзывчивый
3. Фильтр готов → список обновляется

### Что реально даёт useTransition

**НЕ убирает задержку полностью**, а **меняет её приоритет**:

```
Без useTransition:
👆 Печатаешь "iph" → React: "Подожди, сначала отфильтрую 10k → потом покажу 'iph' в input"
   ↓ 300мс полная блокировка

С useTransition:
👆 Печатаешь "iph" → React: "Срочно: покажи 'iph' в input → потом отфильтруй 10k"
   ↓ input мгновенно + "⏳" + фоновая работа
```

### Когда разница заметна

#### 1. **Очень тяжёлые вычисления** (реальный кейс)

```jsx
// Имитация супер-тяжёлого фильтра
const superSlowFilter = (items, query) => {
  let result = [];
  for (let i = 0; i < items.length; i++) {
    // Имитация CPU-интенсивной работы
    for (let j = 0; j < 1000; j++) {
      Math.sqrt(Math.random());
    }
    if (items[i].name.includes(query)) result.push(items[i]);
  }
  return result;
};
```

#### 2. **Быстрый ввод + одновременный клик**

```
👆 Пишешь "iphone" + кликаешь кнопку → без useTransition клик игнорируется
```

#### 3. **Мобильные устройства** (там разница огромная)

### Итог для мидла

```
useTransition = "Сделай input/кнопки срочно, остальное потом"
Разница заметна при:
• >50k элементов
• Сложные вычисления (sort+filter+reduce)
• Мобильные устройства
• setState + клик одновременно
```

## Suspense — "Загрузи потом"

**Пока данные грузятся — покажи placeholder**:

```jsx
<Suspense fallback={<BigSpinner />}>
  <SlowTab />
</Suspense>
```

```
App ──┐
      ├─ Tab1 (готов) ✓
      ├─ Tab2 (загружается) ⏳ ← Suspense ловит
      └─ Tab3 (готов) ✓
```

**Suspense** показывает **заглушку**, пока **ленивый компонент загружается** по сети. Идеально для **разбиения бандла** на чанки.

### Полный рабочий код (создай 3 файла)

#### 1. `App.jsx` (главный компонент)

```jsx
import React, { Suspense, lazy } from "react";

// Ленивая загрузка — НЕ включается в основной бандл!
const SlowPosts = lazy(() => {
  // Имитация задержки загрузки по сети (2 секунды)
  return Promise.all([
    import("./SlowPosts.jsx"),
    new Promise((resolve) => setTimeout(resolve, 2000)),
  ]).then(([module]) => module);
});

function App() {
  return (
    <div style={{ padding: 20 }}>
      <h1>📱 Мобильное приложение</h1>

      <nav style={{ marginBottom: 20 }}>
        <button>Главная</button>
        <button>Профиль</button>
        <button>Посты (медленные)</button>
      </nav>

      {/* Пока SlowPosts грузится — показываем Skeleton */}
      <Suspense
        fallback={
          <div
            style={{
              padding: 20,
              background: "#f0f0f0",
              borderRadius: 8,
            }}
          >
            <div
              style={{
                width: 100,
                height: 20,
                background: "#ddd",
                marginBottom: 10,
                borderRadius: 4,
              }}
            />
            <div
              style={{
                width: 200,
                height: 20,
                background: "#ddd",
                borderRadius: 4,
              }}
            />
            <div
              style={{
                width: 150,
                height: 20,
                background: "#ddd",
                borderRadius: 4,
              }}
            />
          </div>
        }
      >
        <SlowPosts />
      </Suspense>
    </div>
  );
}

export default App;
```

#### 2. `SlowPosts.jsx` (ленивый компонент)

```jsx
import React, { useState, useEffect } from "react";

export default function SlowPosts() {
  const [posts, setPosts] = useState([]);

  useEffect(() => {
    // Имитация медленного API
    setTimeout(() => {
      setPosts([
        { id: 1, title: "Пост 1: React Fiber" },
        { id: 2, title: "Пост 2: useTransition" },
        { id: 3, title: "Пост 3: Suspense в действии" },
      ]);
    }, 1000);
  }, []);

  return (
    <div>
      <h2>📝 Посты (загружено!)</h2>
      <ul>
        {posts.map((post) => (
          <li
            key={post.id}
            style={{ padding: 10, borderBottom: "1px solid #eee" }}
          >
            {post.title}
          </li>
        ))}
      </ul>
    </div>
  );
}
```

#### 3. `index.js`

```jsx
import React from "react";
import { createRoot } from "react-dom/client";
import App from "./App.jsx";

const root = createRoot(document.getElementById("root"));
root.render(<App />);
```

### Что происходит?

```
1. Запускается App → React видит lazy(SlowPosts)
2. Создаётся network request: /src/SlowPosts.jsx (отдельный чанк)
3. Показывается Skeleton (серые полоски)
4. Через 2с: chunk загружен + API готов → рендер SlowPosts
```

### В Network вкладке браузера:

```
main.js (быстро)
└── SlowPosts.jsx (2с задержка) ← отдельный HTTP запрос!
```

### Результат UX:

```
⏱️ 0с: Главная страница + Skeleton постов
⏱️ 2с: Посты подгружаются (плавный переход)
```

## Для мидла достаточно знать:

| Фича              | Зачем                     | Как юзать                                |
| :---------------- | :------------------------ | :--------------------------------------- |
| **useTransition** | Тяжёлый фильтр/сортировка | `startTransition(() => setHeavyState())` |
| **Suspense**      | Lazy компоненты/API       | `<Suspense fallback={<Spinner />}>`      |
| **Concurrent**    | Всё выше                  | Включается автоматически в React 18+     |

**Практика**: добавь `useTransition` в любой поиск/фильтр — увидишь магию отзывчивости!

## useDeferredValue

**`useDeferredValue`** — хук React 18+, который **откладывает обновление** "тяжёлых" значений (списки, графики), пока UI остаётся отзывчивым. **"Сначала input, потом список"**.

### 🎯 Проблема без useDeferredValue

```jsx
function Search() {
  const [query, setQuery] = useState("");

  // ❌ Input + список = 500мс лаг каждый символ!
  return (
    <>
      <input value={query} onChange={(e) => setQuery(e.target.value)} />
      <SlowList query={query} /> {/* 5000 элементов */}
    </>
  );
}
```

**Пользователь печатает** → **весь UI зависает** → **плохой UX**.

### ✅ Решение с useDeferredValue

```jsx
function Search() {
  const [query, setQuery] = useState("");
  const deferredQuery = useDeferredValue(query); // 🔥 Отложенное значение

  return (
    <>
      {/* Input мгновенный! */}
      <input value={query} onChange={(e) => setQuery(e.target.value)} />

      {/* Список обновится ПОЗЖЕ */}
      <SlowList query={deferredQuery} />
    </>
  );
}
```

### 🎬 Как работает (шаг за шагом)

```
1. Пользователь печатает "a" → setQuery("a")
2. React рендерит:
   query = "a" ✅ (input мгновенно)
   deferredQuery = "a" (старое значение пока!)
3. Фоновый рендер: deferredQuery = "a" → SlowList фильтруется
4. Пользователь печатает "ab" → setQuery("ab")
5. React:
   query = "ab" ✅ (input мгновенно)
   deferredQuery = "a" (старое значение!)
6. SlowList покажет "ab" только когда успеет
```

### 🔄 useDeferredValue vs useTransition

| Хук                    | Что откладывает                         | Где использовать      |
| :--------------------- | :-------------------------------------- | :-------------------- |
| **`useTransition`**    | **setState внутри** `startTransition()` | Фильтры, сортировки   |
| **`useDeferredValue`** | **Передаваемое значение**               | Input → список/график |

```jsx
// useTransition — помечаем setState
const [filter, startTransition] = useTransition();
startTransition(() => setFilter(value));

// useDeferredValue — помечаем значение
const deferredQuery = useDeferredValue(query);
<SlowList query={deferredQuery} />;
```

### 🚀 Рабочий пример (5000 элементов)

```jsx
import { useState, useDeferredValue } from "react";
import { useEffect } from "react";

function SlowList({ query }) {
  const [items, setItems] = useState([]);

  useEffect(() => {
    // Имитация тяжёлого фильтра
    const heavyList = Array(5000)
      .fill()
      .map((_, i) => ({
        id: i,
        name: `Элемент #${i + 1}`,
      }))
      .filter((item) => item.name.includes(query));
    setItems(heavyList);
  }, [query]);

  return (
    <ul style={{ height: 300, overflow: "auto" }}>
      {items.map((item) => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}

function App() {
  const [query, setQuery] = useState("");
  const deferredQuery = useDeferredValue(query);

  return (
    <div style={{ padding: 20 }}>
      <input
        value={query}
        onChange={(e) => setQuery(e.target.value)}
        placeholder="Печатайте быстро!"
        style={{ padding: 10, fontSize: 16, width: 300 }}
      />
      <SlowList query={deferredQuery} />
    </div>
  );
}
```

### 🎯 Результат UX

```
❌ Без useDeferredValue:
"печатаю" → input лагает → список лагает (500мс)

✅ С useDeferredValue:
"печатаю" → input мгновенно → список через 200мс
```

### ⚠️ Когда использовать

```
✅ Поиск/фильтр больших списков (>1000 элементов)
✅ Графики/таблицы с данными
✅ Автокомплит с тяжёлым рендером
✅ Любое значение, которое вызывает долгий рендер

❌ Простые состояния (isOpen, count)
❌ useTransition уже используется
```

**TL;DR**: **`useDeferredValue` = "input срочно, список потом"**. React показывает **отложенную версию** значения для тяжёлых компонентов.[^1]

## 🎯 **useDeferredValue vs useTransition: ключевая разница**

**useTransition** и **useDeferredValue** решают **одну проблему** (тяжёлые рендеры), но **управляют разным**:

| Хук                | **Оборачивает**                 | **Где применять**            | **Пример**                                |
| :----------------- | :------------------------------ | :--------------------------- | :---------------------------------------- |
| `useTransition`    | **`setState`** (код обновления) | **Ты контролируешь state**   | `startTransition(() => setFilter(value))` |
| `useDeferredValue` | **`значение`** (props)          | **Значение приходит сверху** | `<SlowList query={deferredQuery} />`      |

## 🚨 **Почему нельзя заменить одно другим**

### **Сценарий 1: Ты контролируешь state → useTransition**

```jsx
// ✅ В твоём компоненте
function Search() {
  const [query, setQuery] = useState("");
  const [isPending, startTransition] = useTransition();

  const handleChange = (value) => {
    setQuery(value); // СРАЗУ!
    startTransition(() => {
      setFilteredItems(heavyFilter(value)); // ПОТОМ
    });
  };
}
```

### **Сценарий 2: Значение приходит props → useDeferredValue**

```jsx
// ❌ useTransition НЕ поможет!
function SlowList({ query }) {
  // query приходит сверху!
  // Нельзя обернуть setState — его нет!
  return <HeavyTable data={filter(query)} />;
}

// ✅ useDeferredValue оборачивает query
function SlowList({ query }) {
  const deferredQuery = useDeferredValue(query);
  return <HeavyTable data={filter(deferredQuery)} />;
}
```

## 🎯 **Правило выбора (собес-ответ)**

```
"Если я контролирую setState → useTransition
 Если значение приходит props → useDeferredValue

НЕ использовать вместе — дублируют задачу!"
```

## 🔬 **Под капотом Fiber**

```
React Fiber Scheduler:
1. Высокий приоритет: input, кнопки
2. Низкий приоритет: списки, графики

useTransition → помечает setState "низкий приоритет"
useDeferredValue → помечает значение "низкий приоритет"
```

**TL;DR**: **useTransition** = "отложи мой setState", **useDeferredValue** = "отложи это значение". Для **библиотечных компонентов** — только useDeferredValue! 🚀
