# Interface vs Type: Различия и когда использовать

| Аспект                     | `interface`                | `type`                             |
| :------------------------- | :------------------------- | :--------------------------------- |
| **Расширение**             | `extends`                  | `&` (intersection)                 |
| **Объединение деклараций** | ✅ Автоматически сливается | ❌ Ошибка                          |
| **Union типы**             | ❌ Нельзя                  | ✅ `string \| number`              |
| **Primitives**             | ❌ Нельзя                  | ✅ `type ID = string \| number`    |
| **Tuples**                 | ❌ Нельзя                  | ✅ `type Point = [number, number]` |
| **Мapped types**           | ❌ Нельзя                  | ✅ `type Readonly<T>`              |
| **Ошибки TS**              | Более читаемые             | Менее точные                       |

## Примеры различий

### 1. Расширение

```typescript
// Interface
interface User {
  name: string;
}
interface Admin extends User {
  role: "admin";
}

// Type
type UserType = { name: string };
type AdminType = UserType & { role: "admin" };
```

### 2. **Declaration Merging** (главное отличие!)

```typescript
// ✅ Interface — сливается
interface User {
  name: string;
}
interface User {
  age: number;
}
// User = { name: string; age: number }

// ❌ Type — ошибка!
type User = { name: string };
type User = { age: number }; // Duplicate identifier 'User'
```

### 3. Unions и сложные типы

```typescript
// Только type!
type ID = string | number;
type Status = "loading" | "success" | "error";
type Pair = [string, number];
```

## Правила использования (практика React/TS)

```
✅ interface — 90% случаев
- Props компонентов
- Объекты/Response API
- Модели данных

✅ type — 10% случаев
- Union/Intersection
- Primitives
- Utility types (Partial, Pick...)
```

### React Props — всегда `interface`

```typescript
interface ButtonProps {
  onClick: () => void;
  children: React.ReactNode;
  variant?: 'primary' | 'secondary';
}

const Button: React.FC<ButtonProps> = ({ onClick, children, variant }) => (
  <button onClick={onClick}>{children}</button>
);
```

### Сложная логика — `type`

```typescript
type ApiResponse<T> =
  | { success: true; data: T }
  | { success: false; error: string };

type Event =
  | { type: "click"; x: number; y: number }
  | { type: "keydown"; key: string };
```

## Когда **строго** `interface`

```typescript
// 1. Props React компонентов
interface TodoItemProps { ... }

// 2. Модели данных (declaration merging)
interface User { id: number; }
interface User { name: string; } // Автоматически { id, name }

// 3. Библиотеки (расширение)
interface Window {
  __MY_APP__: string;
}
```

## Когда **строго** `type`

```typescript
// 1. Union types
type Theme = "light" | "dark";

// 2. Utility types
type PartialUser = Partial<User>;
type UserId = User["id"];

// 3. Mapped types
type ReadOnlyUser = { readonly [K in keyof User]: User[K] };
```

## Итоговая мантра

```
Props/API/objects → interface
Unions/Primitives → type

90% React кода = interface UserProps { ... }
10% логика = type Status = 'ok' | 'error'
```

**Declaration merging** — **главное преимущество interface**. В остальном — вкусовщина.

## Вопросы по `interface` vs `type` (Middle/Middle+ уровень)

### 🔍 **Тест понимания основ**

**1. Покажите разницу на примере расширения:**

```typescript
// Напишите Admin на базе User
interface User {
  name: string;
}
type UserT = { name: string };
// Ожидаемый ответ: interface Admin extends User vs type AdminT = UserT & {...}
```

**2. Что произойдёт?**

```typescript
interface User {
  name: string;
}
interface User {
  age: number;
} // ?
type UserT = { name: string };
type UserT = { age: number }; // ?
```

**Правильный ответ**: Interface сливается, type — ошибка.

### 🛠️ **React-специфика (Middle+)**

**3. Типизировать React компонент — `interface` или `type`?**

```typescript
// Правильно
interface ButtonProps {
  onClick: () => void;
  children?: React.ReactNode;
}
const Button: React.FC<ButtonProps> = ({ onClick }) => ...
```

**Почему interface**: React DevTools читает `displayName`, declaration merging.

**4. Live-coding: Generic таблица**

```typescript
interface TableProps<T> {
  // ← ?
  data: T[];
  columns: (keyof T)[];
}
```

### 🎯 **Продвинутые вопросы (Middle+)**

**5. Расширить глобальный объект Window:**

```typescript
// Только interface работает!
declare global {
  interface Window {
    // Не type Window!
    __APP_VERSION__: string;
  }
}
```

**Вопрос**: Почему `type Window` выдаст ошибку?

**6. Mapped types — только `type`!**

```typescript
type ReadonlyUser<T> = {
  readonly [K in keyof T]: T[K];
};
// Почему interface здесь не работает?
```

**7. Index signature порядок:**

```typescript
interface User {
  name: string;
  [key: string]: any; // ✅ Работает
}
type UserT = {
  [key: string]: any;
  name: string; // ❌ Ошибка!
};
```

### 💻 **Задачи на код (Middle)**

**8. API Response с discriminated union:**

```typescript
type ApiResponse<T> = // type!
  { ok: true; data: T } | { ok: false; error: string };
```

**9. Расширить Props с утилитами:**

```typescript
interface BaseProps {
  className?: string;
}
type ButtonProps = BaseProps & {
  // &
  variant: "primary" | "secondary";
};
```

**10. Комбинировать с HOC:**

```typescript
interface WithLoadingProps {
  isLoading: boolean;
}
type FinalProps = ButtonProps & WithLoadingProps;
```

### 🚨 **Красные флаги (Middle уровень)**

```
❌ "type лучше во всём"
❌ Не знает declaration merging
❌ interface для union types
❌ Не типизирует React.FC<Props>
✅ Знает: "Props → interface, unions → type"
```

### **Типичная подстава (Middle+):**

```typescript
// Что выведет IntelliSense?
interface ExternalLib {
  getUser(id: string): User;
}
interface ExternalLib {
  getUsers(ids: string[]): User[];
}
// Ответ: оба метода доступны!
```

**Итоговая формула для Middle**:

```
React Props → interface ButtonProps { }
API модели → interface UserResponse { }
Unions → type Status = 'ok' | 'error'
Globals → declare global { interface Window }
```
