# Generic Types: Для создания переиспользуемых компонентов и функций

**Generic Types** позволяют создавать переиспользуемые компоненты и функции с **типобезопасностью** для любых типов данных.

## Базовый синтаксис

```
function identity<T>(value: T): T {
  return value;
}

// Автовывод типа
identity(42);        // T = number
identity("hello");   // T = string

// Явное указание
identity<string>("hi");
```

## Практические примеры для React/React-разработчика

### 1. **Переиспользуемый хук**

```ts
function useLocalStorage<T>(key: string, initial: T): [T, (val: T) => void] {
  const [value, setValue] = useState<T>(initial);

  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [value]);

  return [value, setValue];
}

// Использование
const [user, setUser] = useLocalStorage<User>("user", { name: "Никита" });
const [count, setCount] = useLocalStorage<number>("count", 0);
```

### 2. **Generic компонент**

```ts
interface ListProps<T> {
  items: T[];
  renderItem: (item: T) => ReactNode;
}

function List<T>({ items, renderItem }: ListProps<T>) {
  return (
    <ul>
      {items.map((item, i) => (
        <li key={i}>{renderItem(item)}</li>
      ))}
    </ul>
  );
}

// Использование
<List
  items={users}
  renderItem={(user) => <div>{user.name}</div>}
/>
<List
  items={[1, 2, 3]}
  renderItem={(num) => <div>{num * 2}</div>}
/>
```

## 🛠️ Продвинутые техники (мидл+)

### 1. **Constraints (ограничения)**

```ts
function merge<T extends object, U extends object>(obj1: T, obj2: U): T & U {
  return { ...obj1, ...obj2 };
}

type Config = { apiUrl: string; timeout: number };
merge({ apiUrl: "/api" }, { timeout: 5000 }); // ✅
merge(42, "hi"); // ❌
```

### 2. **Multiple generics**

```ts
function mapObject<K extends string | number, V, W>(
  obj: Record<K, V>,
  fn: (value: V, key: K) => W,
): Record<K, W> {
  return Object.fromEntries(
    Object.entries(obj).map(([key, value]) => [key, fn(value, key as K)]),
  ) as Record<K, W>;
}
```

### 3. **Default types**

```ts
function createStore<T = unknown>(
  initial: T,
): { state: T; setState: (val: T) => void } {
  return { state: initial, setState: () => {} };
}

const numberStore = createStore(0); // T = number
const stringStore = createStore(""); // T = string
const anyStore = createStore(); // T = unknown
```

## 🎯 Типовые паттерны для React/Next.js

### API Response

```ts
interface ApiResponse<T> {
  data: T;
  status: "success" | "error";
  meta?: { total: number };
}

async function fetchData<T>(url: string): Promise<ApiResponse<T>> {
  const res = await fetch(url);
  return res.json();
}

const users = await fetchData<User[]>("/api/users");
```

### Form Hook

```ts
function useForm<T extends Record<string, any>>(initial: T) {
  const [form, setForm] = useState<T>(initial);

  const update = <K extends keyof T>(key: K, value: T[K]) => {
    setForm((prev) => ({ ...prev, [key]: value }));
  };

  return { form, update };
}
```

## 🚨 Ошибки, которых избегать

```ts
// ❌ НЕ РАБОТАЕТ с union types
function getLength<T>(item: T): number {
  return item.length; // ❌ T может не иметь length!
}

// ✅ Правильно с constraint
function getLength<T extends { length: number }>(item: T): number {
  return item.length;
}
```

## Чем T[] отличается от Array<T>?

**`T[]` и `Array<T>` функционально идентичны**, но есть важные нюансы для мидл+.

### Основные отличия

| Аспект                        | `T[]`                                          | `Array<T>`                           |
| :---------------------------- | :--------------------------------------------- | :----------------------------------- |
| **Синтаксис**                 | Короткий, привычный                            | Generic-нотация                      |
| **TypeScript выводит**        | ✅ Всегда `number[]`                           | ✅ Показывает `Array<number>`        |
| **Читабельность вложенности** | `number[][]` (неоднозначно)                    | `Array<Array<number>>` (ясно)        |
| **keyof оператор**            | `keyof number[]` = `"length" \| "push" \| ...` | `keyof Array<number>` = те же методы |

### 🎯 Практические примеры

#### ✅ `T[]` выигрывает в простоте

```ts
type Users = User[]; // ✅ Коротко
type UserIds = number[]; // ✅ Понятно

const users: User[] = []; // TS покажет User[]
const users2: Array<User> = []; // TS покажет User[] ← предпочтение!
```

#### ❌ Проблема с `keyof`

```ts
type T = { id: number; name: string };
type Arr1 = T[];           // number[] ✅
type Arr2 = Array<T>;      // Array<T>

keyof Arr1;  // "length" | "pop" | "push" | ... ✅ Методы массива
keyof Arr2;  // То же самое ✅

// ❌ Ложное ожидание:
// keyof T[] НЕ даёт "id" | "name"!
```

### 🛠️ Когда что использовать (React/TSX)

#### ✅ `T[]` для простых случаев (90%)

```tsx
interface Props {
  items: string[]; // ✅
  users: User[]; // ✅
  onChange: (value: number[]) => void; // ✅
}
```

#### ✅ `Array<T>` для сложных типов

```tsx
// Многострочные типы
type ComplexProps = {
  matrix: Array<Array<Array<number>>>;
  callbacks: Array<(value: User) => void>;
};

// Или в TSX (jsx не любит [])
const Complex: React.FC<{ data: Array<User | null> }> = ({ data }) => { ... }
```

### 🚨 Важный нюанс для мидл+

```ts
// Разница видна в utility types
type MapUsers = Map<User, boolean>; // ✅
type MapUsers2 = Map<User, boolean>[]; // ✅ Массив Map'ов

type ArrayUsers = Array<User>[]; // ✅ Массив массивов
type ArrayUsers2 = User[][]; // ✅ То же самое, но менее читаемо
```

### 🎯 Рекомендация для React-разработчика

```ts
// ✅ В 95% случаев
interface TodoListProps {
  todos: Todo[];
  onToggle: (id: string) => void;
}

// ✅ Только для сложных случаев
type MatrixCell = Array<Array<{ value: number; state: "edit" | "view" }>>;

// ✅ В хуках и утилитах
function useItems<T>(): T[] {
  // ✅ Коротко
  return [];
}
```

### 🧠 Собеседование

**Вопрос**: "Что выведет `keyof string[]`?"
**Ответ**: Методы массива (`"length" | "pop" | ...`), **НЕ** `keyof string`!

**Правило**:

- `T[]` — короче, проще, **рекомендуется TS**
- `Array<T>` — для сложных типов или когда нужна консистентность с `Map<K,V>`

**Используй `T[]` везде, кроме JSX и очень сложных типов.**

## Как работает type inference в generics?

**Type Inference в generics** — это механизм, когда TypeScript **автоматически определяет** тип параметра `T` без явного указания.

### 🎯 Как TS определяет тип?

#### 1. **По аргументу функции** (самый частый случай)

```ts
function identity<T>(arg: T): T {
  return arg;
}

const num = identity(42); // T = number ✅
const str = identity("hi"); // T = string ✅
const user = identity({ name: "Никита" }); // T = {name: string} ✅
```

**TS смотрит на тип `arg` и подставляет его в `T`**.

#### 2. **По возвращаемому значению**

```ts
function createArray<T>(value: T): T[] {
  return [value];
}

const arr = createArray(123); // T = number → number[]
```

#### 3. **По контексту вызова** (React хуки)

```ts
function useState<T>(initial: T): [T, (val: T) => void] {
  // ...
}

const [count, setCount] = useState(0); // T = number ✅
const [user, setUser] = useState({ name: "Никита" }); // T = {name: string} ✅
```

### 🧠 Пошаговый алгоритм вывода

```
identity(42)
  ↓
1. Смотрим аргумент: 42 → number
2. arg: T → T = number
3. Возврат: T → number
4. Итог: identity<number>(42): number ✅
```

### 🚨 Когда inference НЕ работает

#### ❌ Несколько аргументов разных типов

```ts
function merge<T, U>(a: T, b: U): T & U {
  return { ...a, ...b };
}

merge({ x: 1 }, "hello"); // ❌ T=?, U=?
// Нужно явно указать: merge<{x:number}, string>(...)
```

#### ❌ Сложные зависимости

```ts
function wrapper<T>(fn: (x: T) => void) {
  return fn;
}

wrapper((x: User) => {}); // ❌ T не выводится
```

### ✅ Решения проблем

#### 1. **Первый аргумент определяет остальные**

```ts
function apiCall<T, U>(config: T, fn: (data: U) => void): U {
  return {} as U;
}

apiCall({ url: "/users" }, (users) => {}); // T = {url: string}, U = User[]
```

#### 2. **Default parameters**

```ts
function create<T = string>(value?: T): T {
  return value ?? ("" as T);
}
```

### 🎯 Продвинутые примеры (мидл+)

#### Higher-Order Functions

```ts
function pipe<T, R>(fn1: (x: T) => R): (x: T) => R {
  return fn1;
}

const doubled = pipe((x) => x * 2)(5); // T = number, R = number ✅
```

#### React: Props + Children inference

```ts
interface Props<T> {
  items: T[];
  render: (item: T) => ReactNode;
}

function List<T>({ items, render }: Props<T>) {
  return items.map(render);
}

// TS выведет T из items!
<List items={users} render={user => <div>{user.name}</div>} />
```

### 🧪 Собеседование: типовые вопросы

**1. Что выведет?**

```ts
function first<T>(arr: T[]): T {
  return arr[^0];
}

first([1, 2, 3]);  // number ✅
```

**2. Почему ошибка?**

```ts
function bad<T>(fn: T): T {
  return fn(123); // ❌ T не имеет .call()
}
```

**3. Напишите типобезопасный `map`**

```ts
function map<T, U>(arr: T[], fn: (item: T) => U): U[] {
  return arr.map(fn);
}
```

### 🎯 Правила для React-разработчика

1. **Доверяй inference** — пиши `<T>(arg: T)` без явных типов
2. **Первый аргумент** обычно определяет `T`
3. **В React хуках** `useState(initial)` выводит тип из `initial`
4. **Не работай с `unknown`** — TS не выведет тип из `unknown`

```
useState(0) → number
useState("") → string
useState(null) → null ❌ ← всегда пиши initial value!
```

**Inference = магия TS, которая делает generics удобными**. 95% случаев работает автоматически!

## Что такое `T extends U`?

**`T extends U`** — это **Conditional Type** (условный тип) в TypeScript. Это синтаксис `T extends U ? X : Y`, где проверяется **"T является подтипом U или равен U?"**.

### 🎯 Как работает

```
T extends U ? X : Y
```

- **Если T assignable to U** → возвращает `X`
- **Иначе** → возвращает `Y`

### 🧠 Пошаговые примеры

#### ✅ Базовый пример

```ts
type IsString<T> = T extends string ? "Да" : "Нет";

type T1 = IsString<string>; // "Да" ✅
type T2 = IsString<number>; // "Нет" ✅
type T3 = IsString<"hello">; // "Да" ✅ (literal extends string)
```

#### ✅ Практика: `Extract` utility

```ts
type Extract<T, U> = T extends U ? T : never;

type Colors = "red" | "green" | "blue";
type R = Extract<Colors, "red" | "green">; // "red" | "green" ✅
```

### 🔥 ДИСТРИБУТИВНОСТЬ (главная магия)

Когда `T` — это **union**, TS **распределяет** проверку по каждому члену:

```
type T = A | B | C;
T extends U ? X : Y
```

**Становится**:

```
(A extends U ? X : Y) | (B extends U ? X : Y) | (C extends U ? X : Y)
```

```ts
type MyExclude<T, U> = T extends U ? never : T;

type Colors = "red" | "green" | "blue";
type Result = MyExclude<Colors, "red">; // "green" | "blue" ✅
```

**Разбор**:

```
("red" extends "red" ? never : "red")   = never
("green" extends "red" ? never : "green") = "green"
("blue" extends "red" ? never : "blue")  = "blue"
Итог: never | "green" | "blue" = "green" | "blue"
```

### 🛠️ Практика для мидл+

#### 1. **Constraints в generics**

```ts
function fn<T extends { id: number }>(obj: T) {
  return obj.id; // ✅ TS знает, что id есть
}

fn({ id: 1, name: "Никита" }); // ✅
fn("hello"); // ❌
```

#### 2. **infer (вывод типов)**

```ts
type UnpackArray<T> = T extends (infer U)[] ? U : T;

type T1 = UnpackArray<string[]>; // string ✅
type T2 = UnpackArray<number>; // number ✅
```

#### 3. **Реализация `NonNullable`**

```ts
type MyNonNullable<T> = T extends null | undefined ? never : T;

type T = string | null | undefined;
type R = MyNonNullable<T>; // string ✅
```

### 🎯 Типовые паттерны

| Сценарий                 | Код                                      |
| :----------------------- | :--------------------------------------- |
| **Исключить тип**        | `T extends U ? never : T`                |
| **Извлечь тип**          | `T extends U ? T : never`                |
| **Проверить наличие**    | `T extends { prop: any } ? true : false` |
| **Вывести из структуры** | `T extends (infer U)[] ? U : T`          |

### 🚨 СОБЕСЕДОВАНИЕ

**Задача**: Реализуйте `ReturnType`

```ts
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : any;

type T1 = MyReturnType<() => string>; // string ✅
type T2 = MyReturnType<(x: number) => boolean>; // boolean ✅
```

**Вопрос**: Почему `'a' | 'b'` **распределяется**, а объект **нет**?

```ts
type T = "a" | "b";
type R1 = T extends string ? true : false; // true | true = true ✅

type Obj = { a: 1 } | { b: 2 };
type R2 = Obj extends { a: number } ? true : false; // НЕ распределяется!
```

### 💡 Быстрая шпаргалка

```
T extends string           // T — строка или её подтип?
T extends { id: number }   // T имеет свойство id?
T extends any              // T — любой тип (всегда true)
T extends never            // Никогда (всегда false)
```

**Запомни**: `extends` = **"может ли T быть использован как U?"**
