# Ключевые механизмы TypeScript для обеспечения неизменяемости (immutability)

`readonly` и `as const` — ключевые механизмы TypeScript для обеспечения неизменяемости (immutability), но они работают по-разному и дополняют друг друга.

## Readonly

`readonly` делает свойства объекта неизменяемыми на уровне типов. Это модификатор, применяемый к отдельным полям интерфейсов, типов или классов, либо утилита `Readonly<T>` для всего объекта.

- В интерфейсах: `interface User { readonly id: number; name: string; }` — `id` нельзя перезаписать.
- В классах: инициализация только при объявлении или в конструкторе: `class Person { constructor(readonly birthDate: Date) {} }`.
- Утилита: `type ReadonlyConfig = Readonly<{ port: number; dbName: string; }>;` — все поля становятся readonly, но только верхний уровень.

**Ограничение**: мутации вложенных объектов возможны, например, `config.nested.arr.push(1)` сработает.

## as const

`as const` создаёт **глубоко неизменяемый** литеральный тип, фиксируя точные значения всех примитивов и делая все свойства/элементы readonly рекурсивно.

```
const wallet = { key: "open_door_pls" } as const;
// Тип: { readonly key: "open_door_pls" }
// wallet.key = "new" — ошибка
```

- Работает на **любом уровне вложенности**: `{ equipment: { engine: "V8" } } as const` — `engine` тоже readonly.
- Для массивов: `const ids = [1, 2, 3] as const;` — `readonly [1, 2, 3]`, нельзя `push` или мутировать.
- В отличие от `Object.freeze()` (runtime), `as const` — чисто типобезопасность компиляции.

## Сравнение

| Аспект     | readonly                    | as const                |
| :--------- | :-------------------------- | :---------------------- |
| Применение | К полям/типам               | К литералам             |
| Глубина    | Только верхний уровень      | Рекурсивно              |
| Значения   | Любые типы                  | Только литеральные      |
| Runtime    | Нет эффекта                 | Нет эффекта             |
| Массивы    | `readonly T[]` (можно push) | `readonly [val1, val2]` |
| Классы     | Да                          | Нет                     |

## Практика для мидл+

**Комбинируйте**:

```
interface APIResponse {
  readonly data: ReadonlyArray<{ readonly id: number; }>;
}

const response = {
  data: [{ id: 1 }, { id: 2 }]
} as const satisfies APIResponse;  // TS 4.9+: проверка + литеральные типы
```

**Питфаллы**:

- `as const` не работает с вычисляемыми значениями (только литералы).
- `readonly` в классах ≠ `private readonly` (setter возможен).

Используйте `as const` для конфигов/API-констант, `readonly` — для интерфейсов/DTO.
