# Type Guards - Проверка типов в рантайме (typeof, instanceof, пользовательские)

**Type Guards** — это функции или выражения, которые **во время выполнения** проверяют тип значения и **сужают** (narrow) тип в глазах TypeScript.

## 🎯 Встроенные Type Guards

### 1. **`typeof` — примитивы**

```ts
function printLength(x: string | number | null) {
  if (typeof x === "string") {
    return x.length; // ✅ x: string
  }
  if (typeof x === "number") {
    return x.toFixed(2); // ✅ x: number
  }
}
```

### 2. **`instanceof` — классы**

```ts
class User {
  name: string = "";
}
class Admin {
  role: string = "";
}

function processUser(x: User | Admin) {
  if (x instanceof User) {
    return x.name; // ✅ x: User
  }
  return x.role; // ✅ x: Admin
}
```

### 3. **Оператор `in` — наличие свойств**

```ts
interface Fish {
  swimmable: true;
}
interface Bird {
  flyable: true;
}

function move(animal: Fish | Bird) {
  if ("swimmable" in animal) {
    return "плавает"; // ✅ animal: Fish
  }
  return "летает"; // ✅ animal: Bird
}
```

## 🔥 Пользовательские Type Guards (мидл+)

### ✅ **Type Predicate** (синтаксис `is`)

```ts
function isUser(obj: any): obj is User {
  return obj && typeof obj === "object" && "name" in obj;
}

function getName(x: User | Admin | null) {
  if (isUser(x)) {
    return x.name; // ✅ x: User
  }
  return null;
}
```

## 🛠️ Discriminated Unions + Type Guards

**Золотой стандарт для API**:

```ts
interface Loading {
  type: "loading";
}
interface Success<T> {
  type: "success";
  data: T;
}
interface Error {
  type: "error";
  error: string;
}

type ApiResult<T> = Loading | Success<T> | Error;

function handleResult<T>(result: ApiResult<T>): T | null {
  if (result.type === "success") {
    return result.data; // ✅ result: Success<T>, data: T
  }
  return null;
}
```

## 🎯 Продвинутые техники

### 1. **Array type guard**

```ts
function isUserArray(arr: any): arr is User[] {
  return Array.isArray(arr) && arr.every((item) => isUser(item));
}

const data = apiResponse.data;
if (isUserArray(data)) {
  data.map((u) => u.name); // ✅ data: User[]
}
```

### 2. **Assertion function** (TS 3.7+)

```ts
function assertUser(x: any): asserts x is User {
  if (!isUser(x)) {
    throw new Error("Not a User");
  }
}

const maybeUser = getData();
assertUser(maybeUser); // Больше никаких проверок!
console.log(maybeUser.name); // ✅ User
```

## 🚨 Типовые ошибки

```ts
// ❌ НЕ РАБОТАЕТ
if (typeof x === "string") {
  x.toUpperCase(); // ОШИБКА! x всё ещё string | number
}

// ✅ ПРАВИЛЬНО
if (typeof x === "string") {
  x.toUpperCase(); // ✅ x: string
}
```

## 🧠 Собеседование: задачи

**1. Напишите `hasProperty` guard**

```ts
function hasProperty<T, K extends PropertyKey>(
  obj: T,
  prop: K,
): obj is T & Record<K, unknown> {
  return prop in (obj as any);
}

const obj: unknown = {};

if (hasProperty(obj, "prop")) {
  console.log(obj.prop); // ✅
}
```

**2. Что выведет?**

```ts
type Shape = Circle | Square;
interface Circle {
  kind: "circle";
  radius: number;
}
interface Square {
  kind: "square";
  side: number;
}

function area(shape: Shape) {
  if (shape.kind === "circle") {
    return Math.PI * shape.radius ** 2; // ✅
  }
}
```

**3. Реализуйте `NonNullable` guard**

```ts
function isNonNullable<T>(x: T): x is NonNullable<T> {
  return x != null;
}
```

## 📋 Шпаргалка

| Guard          | Когда использовать       | Пример                     |
| :------------- | :----------------------- | :------------------------- |
| `typeof`       | Примитивы                | `typeof x === "string"`    |
| `instanceof`   | Классы                   | `x instanceof Date`        |
| `in`           | Интерфейсы/unions        | `"data" in response`       |
| `is` predicate | Custom логика            | `isUser(obj): obj is User` |
| `asserts`      | Гарантированная проверка | `assertUser(x)`            |

**Правило**: Type Guard всегда возвращает **более узкий тип** в `true`-ветке!
