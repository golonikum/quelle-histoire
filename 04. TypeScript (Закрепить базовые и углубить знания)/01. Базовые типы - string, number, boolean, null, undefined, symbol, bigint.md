## Базовые типы TypeScript

| Тип             | Описание          | Пример                                                              |
| :-------------- | :---------------- | :------------------------------------------------------------------ |
| **`string`**    | Текст             | `let name: string = "Иван"`                                         |
| **`number`**    | Число (int/float) | `let age: number = 30`<br>`let pi: number = 3.14`                   |
| **`boolean`**   | true/false        | `let isActive: boolean = true`                                      |
| **`null`**      | Намеренно пусто   | `let user: null = null`                                             |
| **`undefined`** | Не задано         | `let temp: undefined = undefined`<br>`let x;`                       |
| **`symbol`**    | Уникальный ID     | `const id = Symbol('user')`                                         |
| **`bigint`**    | Большие числа     | `let big: bigint = 123n`<br>`let bigger: bigint = 100_000_000_000n` |

## Особенности TS

```typescript
// ✅ Любое значение "угадывается"
let count = 42; // number
let name = "React"; // string

// ✅ Явная типизация
let age: number = 25;
let isDev: boolean = true;

// ❌ Ошибки на этапе разработки
let wrong: string = 123; // 🚫 Type 'number' is not assignable to type 'string'

// any — отключает проверку (не используй!)
let anything: any = "abc";
anything = 123; // ✅ Но лучше избегать

// unknown — безопасный any
let data: unknown = "test";
if (typeof data === "string") {
  data.toUpperCase(); // ✅ Теперь TS уверен
}
```

## Практические нюансы

```
null vs undefined:
- null    = "я пустой специально"
- undefined = "меня забыли задать"

number в JS/TS:
- 42      = целое
- 3.14    = дробное
- NaN     = Not a Number
- Infinity = бесконечность
```

**Запомни**: `string | number | boolean` — **примитивы**, `null/undefined` — **специальные**, `symbol/bigint` — **редкие**.

## Вопросы по базовым типам TS (Middle уровень)

### 🔍 **Простые вопросы (тест базы)**

**1. Какие примитивы есть в TypeScript?**
**Ответ**: `string`, `number`, `boolean`, `null`, `undefined`, `symbol`, `bigint`.[^4]

**2. В чём разница `null` и `undefined`?**

```
null    = "пусто специально"    let user: null = null
undefined = "не задано"       let x;
```

**3. Что будет типом этой переменной?**

```typescript
let value = 42;
let text = "hello";
let obj = {};
```

**Ответ**: `number`, `string`, `{}` (object).

### 🧠 **Подводные камни (Middle)**

**4. Что выведет TS?**

```typescript
let num: number = 42;
num = NaN; // ?
num = Infinity; // ?
```

**Ответ**: ✅ `NaN` и `Infinity` — это `number`.

**5. `any` vs `unknown` — в чём разница?**

```typescript
let a: any = "test";
a.foo(); // ✅ TS молчит (опасно!)

let u: unknown = "test";
u.foo(); // ❌ Ошибка! Нужно проверить
if (typeof u === "string") u.foo(); // ✅ Теперь ок
```

**6. Что выведет?**

```typescript
function fn(x: string | number) {
  return x.toFixed(2); // ?
}
```

**Ответ**: ❌ Ошибка! Только `number` имеет `toFixed()`.

### 💻 **Задачи на код (Middle)**

**7. Типизировать параметры функции:**

```typescript
function greet(name, age) {
  return `Привет, ${name}! Тебе ${age} лет.`;
}
// Напишите типы
function greet(name: string, age: number): string { ... }
```

**8. Что не так? Пофиксить:**

```typescript
let users: string[] = ["Иван", "Пётр"];
users.push(123); // ?
// ✅ let users: (string | number)[] или type guard
```

**9. Live-coding: типизировать объект**

```typescript
interface User {
  id: number;
  name: string;
  email?: string;  // ← ?
}
// Использовать в функции
function getUser(id: number): User | null { ... }
```

### 🎯 **React-специфика (Middle React/TS)**

**10. Типизировать React state:**

```typescript
const [count, setCount] = useState(0); // ✅ number
const [user, setUser] = useState(null); // ❌ null | undefined
const [user, setUser] = useState<User | null>(null); // ✅
```

**11. `children` в Props:**

```typescript
interface Props {
  children: string; // ❌
  children: React.ReactNode; // ✅
}
```

### 🚨 **Типичные ошибки Middle**

```
❌ let count = 0; count = "1"  // Забыл типизацию
❌ interface Props { count: any }  // any вместо number
❌ useState<string[]>([])  // Сложнее чем нужно
✅ const [users, setUsers] = useState<User[]>([])
```

### **Частые подставы:**

```typescript
// Что тип?
let arr = []; // any[]
let obj = {}; // {}
let fn = () => {}; // () => void

// Функция с rest-параметрами
function sum(a: number, b: number, ...rest: number[]) {}
```

**Итог для Middle**:
**Знает примитивы + `any/unknown`** → Junior+
**Типизирует React state/props** → Middle
**Понимает `null | undefined`** → Middle+ [^1].
