# Utility Types - Partial, Required, Readonly, Pick, Omit, Exclude, Extract, NonNullable и другие

**Полный список Utility Types TypeScript** (TS 5.x) — все 20+ встроенных утилит.

## 🛠️ **1. Модификаторы свойств (4)**

| Тип                | Описание                      | Пример                          |
| :----------------- | :---------------------------- | :------------------------------ |
| `Partial<T>`       | Все свойства `optional`       | `{name?: string; age?: number}` |
| `Required<T>`      | Все свойства **обязательные** | `{name: string; age: number}`   |
| `Readonly<T>`      | Все свойства `readonly`       | `{readonly name: string}`       |
| `ReadOnlyArray<T>` | **Не мутирующий** массив      | `ReadonlyArray<string>`         |

## 🛠️ **2. Извлечение/исключение свойств (4)**

| Тип                         | Описание                     | Пример                      |
| :-------------------------- | :--------------------------- | :-------------------------- |
| `Pick<T, K>`                | **Выбрать** ключи `K`        | `Pick<User, 'name'>`        |
| `Omit<T, K>`                | **Исключить** ключи `K`      | `Omit<User, 'password'>`    |
| `Record<K, T>`              | Объект `{ [P in K]: T }`     | `Record<string, User>`      |
| `PickByValue<T, ValueType>` | Выбрать по **значению** типа | `PickByValue<User, string>` |

## 🎯 **3. Работа с Unions (2)**

| Тип             | Описание                    | Пример                           |
| :-------------- | :-------------------------- | :------------------------------- |
| `Exclude<T, U>` | `T`, **исключая** `U`       | `Exclude<'a'\|'b', 'a'>` → `'b'` |
| `Extract<T, U>` | **Извлечь** из `T` типы `U` | `Extract<'a'\|'b', 'a'>` → `'a'` |

## 🛠️ **4. Null/Undefined безопасность (2)**

| Тип               | Описание                         | Пример                                   |
| :---------------- | :------------------------------- | :--------------------------------------- |
| `NonNullable<T>`  | Убрать `null \| undefined`       | `NonNullable<string \| null>` → `string` |
| `InstanceType<T>` | Тип **инстанса** из конструктора | `InstanceType<typeof Date>` → `Date`     |

## 🔄 **5. Функциональные типы (6)**

| Тип                        | Описание                     | Пример                                |
| :------------------------- | :--------------------------- | :------------------------------------ |
| `Parameters<T>`            | **Параметры** функции        | `Parameters<() => void>` → `never`    |
| `ReturnType<T>`            | **Возвращаемый тип** функции | `ReturnType<() => string>` → `string` |
| `ConstructorParameters<T>` | Параметры **конструктора**   | `ConstructorParameters<typeof Date>`  |
| `Awaited<T>`               | Разворачивает `Promise<T>`   | `Awaited<Promise<string>>` → `string` |
| `Uppercase<string>`        | `STRING`                     | `Uppercase<'hello'>` → `'HELLO'`      |
| `Lowercase<string>`        | `string`                     | `Lowercase<'HELLO'>` → `'hello'`      |

## 📝 **6. Остальные (6)**

| Тип                    | Описание               | Пример        |
| :--------------------- | :--------------------- | :------------ |
| `ThisParameterType<T>` | `this` тип функции     | Для bind/call |
| `OmitThisParameter<T>` | Функция **без** `this` |               |
| `ThisType<D>`          | Заменить `this` на `D` |               |
| `BuiltIn`              | **Встроенные** типы JS |               |

## 🔥 **Практика: Record (самый частый)**

```ts
// Словарь пользователей по ID
type UserDict = Record<string, User>;
// { "user-123": User, "user-456": User }

// Функция возвращает объект с ключами из массива
function createEnum<T extends string[]>(
  arr: [...T],
): Record<T[number], T[number]> {
  return arr.reduce((acc, val) => ({ ...acc, [val]: val }), {} as any);
}

const Status = createEnum(["loading", "success", "error"] as const);
// { loading: 'loading', success: 'success', error: 'error' }
```

## 🎯 **React/Next.js паттерны**

```ts
interface User {
  id: string;
  name: string;
  email: string;
  role: string;
  password: string;
}

// API
type UserDto = Omit<User, "password">; // БЕЗ пароля
type UserList = Record<string, UserDto>; // Словарь пользователей
type CreateUser = Partial<Omit<User, "id">>; // Создание
type UpdateUser = Partial<User>; // Обновление

// Компоненты
type UserCardProps = Pick<UserDto, "id" | "name">; // Карточка
type UserRowProps = Required<Pick<UserDto, "email">>; // Строка таблицы
```

## 🧠 **Собеседование: цепочки**

```ts
type User = { id: string; name: string; admin: boolean };

type UserPublic = Omit<User, "admin">; // 1. Убрать admin
type PartialPublic = Partial<UserPublic>; // 2. Сделать optional
type ReadOnlyList = Readonly<Required<PartialPublic>[]>; // 3. Полный readonly массив

// { id: string; name: string }[]
```

## 📋 **Полная шпаргалка по группам**

```
#1 СВОЙСТВА: Partial → Required → Readonly
#2 КЛЮЧИ:     Pick → Omit → Record
#3 UNIONS:    Exclude → Extract
#4 NULL:      NonNullable → InstanceType
#5 ФУНКЦИИ:   Parameters → ReturnType → Awaited
#6 STRING:    Uppercase → Lowercase → Capitalize → Uncapitalize
```

**Запомни**: `Record<K,T>` = универсальный словарь, `Omit/Pick` = точечная подрезка типов.
