## Сложные типы TypeScript

| Тип         | Описание                   | Пример                                                                                  |
| :---------- | :------------------------- | :-------------------------------------------------------------------------------------- |
| **`Array`** | Массив                     | `let nums: number[] = [1, 2, 3]`<br>`let arr: Array<string> = ["a", "b"]`               |
| **`Tuple`** | Фиксированная длина + типы | `let user: [string, number] = ["Иван", 30]`<br>`let point: [number, number] = [10, 20]` |
| **`Enum`**  | Перечисление констант      | `enum Status { Active, Inactive, Pending }`<br>`let status: Status = Status.Active`     |
| **`any`**   | ❌ Любой тип (отключи TS)  | `let data: any = "abc"; data.foo()`                                                     |
| **`void`**  | ❌ Ничего не возвращает    | `function log(): void { console.log("hi") }`                                            |
| **`never`** | ❌ Никогда не происходит   | `function fail(): never { throw new Error() }`                                          |

## Детали использования

### Array

```typescript
// ✅ Два синтаксиса
let ids: number[] = [1, 2, 3];
let names: Array<string> = ["a", "b"];

// ❌ Ошибки
let wrong: number[] = [1, "two", 3]; // string не number
```

### Tuple (кортеж)

```typescript
// ✅ Фиксированная длина + разные типы
let user: [string, number, boolean] = ["Иван", 30, true];

// ✅ Доступ по индексу
user[0].toUpperCase(); // "ИВАН"
user[1]++; // 31

// ❌ Нарушение структуры
user.push("admin"); // Работает, но TS ругается при присвоении
```

### Enum

```typescript
enum Role {
  Admin = "ADMIN",
  User = "USER",
  Guest,
}

let role: Role = Role.Admin; // "ADMIN"

// Numeric enum (по умолчанию)
enum Status {
  Off, // 0
  On, // 1
  Pending, // 2
}
```

### any (антипаттерн)

```typescript
// ❌ Отключает TS полностью
let data: any = apiResponse();
data.чтоУгодно(); // Безопасно для TS, опасно для runtime

// ✅ unknown (лучше)
let safe: unknown = apiResponse();
if (typeof safe === "string") {
  safe.toUpperCase(); // ✅ Теперь ок
}
```

### void vs never

```typescript
// void — нормальная функция
function greet(): void {
  console.log("Hi!");
}

// never — ошибка/бесконечный цикл
function crash(): never {
  throw new Error("💥");
}

function exhaustiveCheck(value: string): never {
  const _exhaustiveCheck: never = value; // Никогда не дойдёт
}
```

## Практика (React + TS)

```typescript
// Props с tuple
interface UserRowProps {
  user: [string, number]; // [name, age]
}

// Enum в компоненте
enum Theme {
  Light = "light",
  Dark = "dark",
}

interface Props {
  theme: Theme;
}
```

```
Array     → React state: state: string[]
Tuple     → API response: [id, title]
Enum      → константы: Status.Loading
any       → ❌ никогда!
never     → type guards: switch exhaustive
```

**Запомни**: **Tuple = массив с типами**, **Enum = красивые константы**, **never = "тут ошибка"** 🚀

## Типичные задачи по сложным типам TS (Middle/Middle+)

### 🧠 **Array + Tuple**

**1. Что выведет? Пофиксить:**

```typescript
let ids: number[] = [1, 2, 3];
ids.push("4"); // ?

let user: [string, number] = ["Иван", 25];
user.push("admin"); // Работает, но как типизировать правильно?
```

**Решение**: `readonly [string, number]` или `as const`

**2. Live-coding: типизировать матрицу**

```typescript
type Matrix = number[][]; // ❌ Может быть [[1,2],[^3]] или [[^1],[2,3]]
// Нужно: [[1,2],[3,4]] — фиксированные строки/столбцы
type Matrix<M extends number, N extends number> = [number, ...number[]][];
```

### ⚙️ **Enum**

**3. Что НЕ так с этим enum?**

```typescript
enum Status {
  Loading,
  Success,
  Error,
}
const status: Status = "loading"; // ?
```

**Задача**: Привести к строковому enum или union type:

```typescript
enum Status {
  Loading = "loading",
  Success = "success",
  Error = "error",
}
```

**4. Преобразовать enum в тип:**

```typescript
enum Role { Admin, User }
type RoleType = ? // "Admin" | "User"
```

**Ответ**: `Role[keyof typeof Role]`

### 🎯 **Array методы**

**5. Типизировать `map` с возвратом:**

```typescript
interface User {
  name: string;
  age: number;
}
const users: User[] = [];
const names = users.map((u) => u.name); // string[] ✅
const ages = users.map((u) => ({ name: u.name })); // ?
```

**Решение**: `Array<User>.map(): (User) => T[]`

### 🛠️ **Tuple продвинутые**

**6. Функция возвращает разное по длине:**

```typescript
function parse(input: string): [string, number] | [string] | [] {}
let result = parse(""); // Какой тип?
```

**Решение**: Union tuples: `[string, number] | [string] | []`

**7. Расширить tuple:**

```typescript
type Point2D = [number, number];
type Point3D = Point2D & [number]; // ❌
```

**Правильно**: `type Point3D = [...Point2D, number]`

### 🔥 **Реальные React задачи**

**8. Типизировать `useState` для формы:**

```typescript
type FormData = {
  name: string;
  email: string | null;
  tags: string[];
};
const [form, setForm] = useState<FormData>({ name: "", email: null, tags: [] });
setForm({ ...form, tags: ["new"] }); // tags остаётся string[]
```

**9. Generic таблица (Middle+):**

```typescript
interface TableProps<T> {
  data: T[];
  renderRow: (item: T, index: number) => JSX.Element;
  keyExtractor: (item: T) => string | number;
}
```

### 🚀 **Сложные live-coding (Middle+)**

**10. Функция `pick` из lodash:**

```typescript
function pick<T, K extends keyof T>(obj: T, keys: K[]): Pick<T, K> {
  // Реализовать тип возврата
}
pick({ name: "Иван", age: 30 }, ["name"]).age; // ❌ Ошибка!
```

**11. Функция `omit`:**

```typescript
function omit<T, K extends keyof T>(obj: T, keys: K[]): Omit<T, K> {}
```

**12. Типизировать `groupBy`:**

```typescript
function groupBy<T, K extends keyof T>(arr: T[], key: K): Record<T[K], T[]> {}
groupBy(users, "city"); // { "Москва": [...], "СПб": [...] }
```

### 💥 **Подводные камни**

**13. Что сломается?**

```typescript
type User = { name: string; age: number };
let users: User[] = [{ name: "Иван", age: 30 }];
users[^0] = { name: "Пётр" }; // ❌ age пропал!
```

**Решение**: `readonly User[]`

**14. Tuple spread:**

```typescript
type Point = [number, number];
type ColoredPoint = [...Point, string]; // [number, number, string]
```

### **Иерархия сложности (собес):**

```
Junior: number[] vs Array<number>
Middle: readonly, tuples, enum → type
Middle+: generics в React, Pick/Omit, mapped types
```

**Частая ошибка**: `let arr: any[]` вместо `unknown[]` или конкретных типов 🚨
