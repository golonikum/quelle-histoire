# Объединения (|) и пересечения (&) типов

Объединения (`|`) и пересечения (`&`) в TypeScript — это мощные инструменты работы с типами. Исправлю неточности и добавлю глубины для мидл+ уровня.

## Объединение (Union, `|`)

Объединение — это **тип, который может быть одним ИЛИ другим** (логическое OR).

```
type ID = number | string;
type EventType = 'click' | 'hover' | 'focus';

let id: ID = 123;        // ✅
id = 'user-123';         // ✅
id = true;              // ❌ Type 'boolean' is not assignable to type 'number | string'
```

**Ключевые особенности:**

- Доступны только **общие методы** всех типов: `string | number` → только `toString()`, `valueOf()`
- **Never возникает**, когда нет общих методов вообще

```
type NoCommon = string | number;
// Доступно: toString(), valueOf()
// НЕ доступно: toUpperCase(), toFixed()
```

## Пересечение (Intersection, `&`)

Пересечение — **все свойства сразу** (логическое AND).

```
interface Named { name: string }
interface Aged { age: number }
interface Role { role: string }

type Person = Named & Aged & Role;
// Эквивалентно: { name: string; age: number; role: string }

const person: Person = {
  name: 'Никита',
  age: 30,
  role: 'mid+'
};
```

## Исправленные неточности

### ✅ Правильно про Never

```
type Primitives = string & number;  // never — примитивы взаимоисключаемы
type Objects = {a: 1} & {b: 2};    // {a: number; b: number}
```

### ✅ String literals работают с intersection

```
type X = 'hello' & { length: number };  // "hello" (строка с фиксированной длиной)
```

## Продвинутые паттерны для мидл+

### 1. Discriminated Unions (рекомендуемый подход)

```
interface Loading { type: 'loading' }
interface Success { type: 'success'; data: User }
interface Error { type: 'error'; error: string }

type ApiResult = Loading | Success | Error;

function processResult(result: ApiResult) {
  switch (result.type) {
    case 'loading': return '⏳';
    case 'success': return result.data;  // ✅ TS знает тип
    case 'error': return result.error;   // ✅ TS знает тип
  }
}
```

### 2. Intersection с generics (собеседование)

```
type MergeProps<T, U> = T & Omit<U, keyof T>;

interface BaseButton { onClick: () => void }
interface IconButton { icon: string }

type IconBtnProps = MergeProps<BaseButton, IconButton>;
// Результат: { onClick: () => void; icon: string }
```

### 3. Function intersections (редко, но спрашивают)

```
type NumFn = (x: number) => number;
type StrFn = (x: string) => string;

type BothFn = NumFn & StrFn;
const fn: BothFn = (x: any) => {
  if (typeof x === 'number') return x * 2;
  return x.toUpperCase();
};
```

## Таблица: когда что использовать

| Сценарий               | Union | Intersection    |
| :--------------------- | :---- | :-------------- |
| API ответы             | ✅    | ❌              |
| Props merging          | ❌    | ✅              |
| Polymorphic компоненты | ✅    | ❌              |
| Utility types          | ❌    | ✅ `Omit<T, K>` |
| Builder pattern        | ❌    | ✅              |

## Собеседование: реальные задачи

**Задача 1: Deep Intersection**

```ts
type DeepIntersection<T, U> = {
  [K in keyof T & keyof U]: T[K] extends object
    ? U[K] extends object
      ? DeepIntersection<T[K], U[K]>
      : T[K] & U[K]
    : T[K] & U[K];
};

type A = { a: { c: number; d: string }; b: string };
type C = { a: { d: string } };

type D = DeepIntersection<A, C>;

const d: D = { a: { d: "str" } };
```

**Задача 2: Что выведет?**

```ts
type A = { a: number } | { b: string };
type B = { a: string } & { b: number };
type C = A & B; // never
```

**Задача 3: Функция с union-return**

```ts
function parseId(id: string): number | string | never {
  if (/^\d+$/.test(id)) return parseInt(id, 10);
  if (id.startsWith("user-")) return id;
  throw new Error("Invalid ID"); // TS выведет never
}
```

Используй **union для вариантов поведения**, **intersection для композиции**. Никогда не путай их порядок — он не влияет на результат.
