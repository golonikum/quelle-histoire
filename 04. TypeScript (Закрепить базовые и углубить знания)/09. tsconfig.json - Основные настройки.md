# tsconfig.json - Основные настройки

**`tsconfig.json`** — главный файл конфигурации TypeScript компилятора. Определяет, **как** и **что** компилировать.

## 🎯 **Структура файла**

```json
{
  "compilerOptions": {
    /* настройки компиляции */
  },
  "include": ["src/**/*"], // Включить файлы
  "exclude": ["node_modules"] // Исключить
}
```

## 🛠️ **Основные настройки `compilerOptions`**

### **1. Цель компиляции**

```json
{
  "target": "ES2022", // JS версия для браузера/Node
  "lib": ["ES2022", "DOM"] // Встроенные API (Array, fetch, DOM)
}
```

**Популярные комбинации**:

```
React/Frontend: "target": "ES2020", "lib": ["ES2020", "DOM", "DOM.Iterable"]
Node.js: "target": "ES2022", "lib": ["ES2022"]
```

### **2. Строгость типов (ОБЯЗАТЕЛЬНО!)**

```json
{
  "strict": true // Включает ВСЕ строгие проверки сразу
}
```

**Что включает** `strict: true`:

- `strictNullChecks` — `null/undefined` отдельные типы
- `noImplicitAny` — запрещает неявный `any`
- `strictFunctionTypes` — строгие функции
- `noImplicitThis` — проверка `this`

### **3. Модули (Next.js/React)**

```json
{
  "module": "ESNext", // ESM (import/export)
  "moduleResolution": "node", // Как искать модули
  "esModuleInterop": true, // import React from 'react'
  "allowSyntheticDefaultImports": true // Поддержка default import
}
```

### **4. Путь вывода**

```json
{
  "rootDir": "./src", // Где исходники
  "outDir": "./dist", // Куда компилировать
  "declaration": true, // Создавать .d.ts файлы
  "sourceMap": true // Для отладки в браузере
}
```

## 🔥 **Рекомендуемый `tsconfig.json` для React/Next.js**

```json
{
  "compilerOptions": {
    // 🎯 ЦЕЛЬ КОМПИЛЯЦИИ: в какую JS версию транслировать
    "target": "ES2020", // ES2020 для современных браузеров/Node.js
    "lib": ["ES2020", "DOM", "DOM.Iterable"], // Встроенные API: fetch, Array.flat(), DOM

    // ✅ СТРОГОСТЬ ТИПОВ (ОБЯЗАТЕЛЬНО!)
    "strict": true, // Включает ВСЕ строгие проверки
    "noUnusedLocals": true, // Неиспользуемые переменные = ошибка
    "noUnusedParameters": true, // Неиспользуемые параметры = ошибка

    // 📁 МОДУЛИ И ИМПОРТЫ
    "module": "ESNext", // ESM (import/export)
    "moduleResolution": "node", // Поиск модулей как в Node.js
    "esModuleInterop": true, // import React from 'react'
    "allowSyntheticDefaultImports": true, // Поддержка default import
    "resolveJsonModule": true, // Импорт JSON файлов

    // 📂 ПУТИ И ВЫВОД
    "rootDir": "./src", // Исходники
    "outDir": "./dist", // Куда складывать .js файлы
    "declaration": true, // Генерировать .d.ts файлы
    "sourceMap": true, // .map файлы для отладки

    // ⚛️ REACT/NEXT.JS
    "jsx": "react-jsx", // Новый JSX transform (React 17+)
    "jsxImportSource": "react", // Где искать jsx-runtime

    // 🚀 ОПТИМИЗАЦИЯ
    "skipLibCheck": true, // НЕ проверять .d.ts файлы (быстрее)
    "isolatedModules": true, // Каждый .ts файл компилируется отдельно
    "noEmit": true, // НЕ генерировать .js (Next.js/Vite сам сделает)

    // 🔒 БЕЗОПАСНОСТЬ
    "forceConsistentCasingInFileNames": true, // user.tsx ≠ User.tsx
    "noFallthroughCasesInSwitch": true // switch без break = ошибка
  },

  // 📂 КАКОЕ ВКЛЮЧИТЬ/ИСКЛЮЧИТЬ
  "include": [
    "src/**/*", // Все .ts/.tsx в src
    "next-env.d.ts" // Next.js типы (если используешь)
  ],
  "exclude": [
    "node_modules", // Библиотеки
    "dist", // Скомпилированные файлы
    "**/*.test.ts" // Тесты (опционально)
  ]
}
```

## 🎯 **Ключевые настройки для мидл+**

| Опция                        | Зачем                             | Значение |
| :--------------------------- | :-------------------------------- | :------- |
| `noUnusedLocals`             | Неиспользуемые переменные         | `true`   |
| `noUnusedParameters`         | Неиспользуемые параметры          | `true`   |
| `noFallthroughCasesInSwitch` | `switch` без `break`              | `true`   |
| `noUncheckedIndexedAccess`   | `obj[key]` может быть `undefined` | `true`   |

## 🚨 **Типовые ошибки новичков**

```json
// ❌ НЕПРАВИЛЬНО
{
  "strict": false,        // Потеряешь типобезопасность!
  "target": "ES5"         // Устарело для современных проектов
}

// ✅ ПРАВИЛЬНО
{
  "strict": true,
  "target": "ES2020"
}
```

## 🧠 **Собеседование**

**Вопрос**: Зачем `skipLibCheck: true`?
**Ответ**: Пропускает проверку `.d.ts` файлов (быстрее компиляция, `@types/react` иногда ломается).

**Вопрос**: `"noEmit": true`?
**Ответ**: TS проверяет типы, но **не генерирует** `.js` (использует Webpack/Vite/Next.js).

**Вопрос**: `isolatedModules: true`?
**Ответ**: Каждый `.ts` файл должен компилироваться отдельно (требует `export {}`).

## 📋 **Быстрый старт шаблон**

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "moduleResolution": "node",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx"
  },
  "include": ["src"]
}
```

**Правило**: `strict: true` + `target: "ES2020"` + `module: "ESNext"` = 95% современных проектов!
