# Error Boundaries

**Error Boundaries** — это **"подушка безопасности"** в React. Если в дочернем компоненте **упадёт JavaScript-ошибка**, Error Boundary **поймает её** и покажет **красивую страницу ошибки** вместо **белого экрана**.

## Проблема без Error Boundaries

```
function BuggyButton() {
  throw new Error('БАМ! 💥'); // Ошибка!
  return <button>Кнопка</button>;
}

function App() {
  return (
    <div>
      <Header />      // Работает ✅
      <BuggyButton /> // Ошибка! 💥
      <Footer />      // Никогда не покажется
    </div>
  );
}
```

**Результат**: **всё приложение** — белый экран.

## Рабочий пример Error Boundary

```jsx
import React from "react";

// 🔥 КЛАССОВЫЙ КОМПОНЕНТ (хуки НЕ работают!)
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = {
      hasError: false,
      error: null,
    };
  }

  // 1. ЛОВИМ ошибку и обновляем состояние
  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }

  // 2. ЛОГИРУЕМ для разработчиков
  componentDidCatch(error, errorInfo) {
    console.error("🚨 Ошибка поймана:", error, errorInfo);
    // Отправляем в Sentry/New Relic
    // logErrorToService(error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return (
        <div
          style={{
            padding: 40,
            textAlign: "center",
            background: "#fee",
            border: "2px solid #fcc",
            borderRadius: 8,
            maxWidth: 500,
            margin: "50px auto",
          }}
        >
          <h1>😵 Что-то сломалось</h1>
          <p>{this.state.error?.message}</p>
          <button
            onClick={() => window.location.reload()}
            style={{
              padding: "10px 20px",
              background: "#ff4444",
              color: "white",
              border: "none",
              borderRadius: 4,
              cursor: "pointer",
            }}
          >
            🔄 Перезагрузить
          </button>
        </div>
      );
    }

    // Нет ошибки — показываем детей
    return this.props.children;
  }
}

// Использование
function App() {
  return (
    <ErrorBoundary>
      <Header />
      <BuggyButton /> {/* Упадёт — но только внутри boundary */}
      <Footer />
    </ErrorBoundary>
  );
}
```

## Что поймает / НЕ поймает

| ✅ Поймает             | ❌ НЕ поймает                           |
| :--------------------- | :-------------------------------------- |
| Ошибки в render()      | Обработчики событий (`onClick`)         |
| Ошибки в lifecycle     | Асинхронный код (`setTimeout`, `fetch`) |
| Ошибки в конструкторах | Ошибки в самом Error Boundary           |
| Хуки внутри дочерних   | Серверный рендеринг                     |

## Многоуровневая защита (реальный проект)

```jsx
function App() {
  return (
    <ErrorBoundary>
      {" "}
      {/* Глобальная ошибка */}
      <Header />
      <ErrorBoundary>
        {" "}
        {/* Ошибка в контенте */}
        <PostsList />
      </ErrorBoundary>
      <ErrorBoundary>
        {" "}
        {/* Ошибка в футере */}
        <Footer />
      </ErrorBoundary>
    </ErrorBoundary>
  );
}
```

```
BuggyButton в PostsList упадёт →
✅ Header работает
❌ PostsList показывает ошибку
✅ Footer работает
```

## Почему классовые компоненты?

```jsx
// ❌ НЕ РАБОТАЕТ с хуками!
function BadErrorBoundary({ children }) {
  const [hasError, setHasError] = useState(false);

  // Нет lifecycle-хука для ошибок!
  return children;
}
```

## Современный подход (React 18+)

```bash
npm install react-error-boundary
```

```jsx
import { ErrorBoundary } from "react-error-boundary";

function App() {
  return (
    <ErrorBoundary
      fallback={
        <div>
          <h2>😵 Ошибка!</h2>
          <button onClick={() => window.location.reload()}>
            Перезагрузить
          </button>
        </div>
      }
      onError={(error, errorInfo) => {
        console.error(error);
        // Sentry.captureException(error);
      }}
    >
      <YourApp />
    </ErrorBoundary>
  );
}
```

## Практические кейсы

```
✅ API вернул null → map() сломался
✅ Библиотека упала (Chart.js, Quill)
✅ Плохой prop drilling (undefined.name)
✅ Баг в стороннем компоненте
```

## Результат в продакшене

```
👉 Пользователь видит "Что-то сломалось" + кнопку
👉 Ты в Sentry видишь stack trace
👉 Фиксишь баг
👉 Пользователь перезагружает → всё работает
```

**Без Error Boundary** = **белый экран + 404 в аналитике** 😱

**TL;DR**: Error Boundary = `try-catch` для **дерева React-компонентов**.[^1]
