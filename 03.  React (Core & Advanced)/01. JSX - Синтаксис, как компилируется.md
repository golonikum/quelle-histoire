# JSX: Синтаксис, как компилируется

JSX — это синтаксическое расширение JavaScript для описания UI в React, похожее на HTML, но компилируемое в нативный JS. Оно упрощает создание элементов через React.createElement и поддерживает динамические выражения в фигурных скобках.[^1][^4]

## Основной синтаксис

```
JSX использует теги как функции: `<h1 className="title">Привет</h1>` вместо строк HTML. Атрибуты camelCase (className вместо class, htmlFor вместо for), style как объект `{color: 'red'}`. Вставка JS: `{name}` или `{2 + 2}`; компоненты: `<MyComponent prop={value} />`. Всегда закрывайте теги: `<img />`, многострочный JSX в скобках `()`.[^2][^1]
```

## Компиляция

Babel или esbuild транслирует JSX в вызовы `React.createElement(type, props, ...children)`:

| JSX                             | После компиляции [^4][^6]                                                   |
| :------------------------------ | :-------------------------------------------------------------------------- |
| `<h1 className="title">Hi</h1>` | `React.createElement('h1', {className: 'title'}, 'Hi')`                     |
| `<div><p>Child</p></div>`       | `React.createElement('div', null, React.createElement('p', null, 'Child'))` |

Это создаёт React-элементы (объекты), которые React рендерит в DOM. JSX — выражение, работает в if/тернарниках, map(), но требует одного корня или `<></>` (Fragment).

# Распространенные ошибки и подводные камни JSX

Распространённые ошибки и подводные камни JSX часто связаны с его отличиями от HTML и особенностями React. Они приводят к ошибкам компиляции, неожиданному рендерингу или проблемам производительности.[^1][^2]

## Синтаксические ошибки

```
Не закрывайте самозакрывающиеся теги: используйте `<img />` или `<br />`, а не `<img>` — иначе Babel выдаст ошибку. Атрибуты camelCase: `className`, `htmlFor`, `onClick`, а не `class`, `for`, `onclick`. Style только как объект: `{color: 'red'}`, не строка `"color: red"`.[^2][^7]
```

## Проблемы рендеринга

```
Всегда один корневой элемент: JSX-выражение не может возвращать несколько соседних узлов без обёртки `<div>` или `<React.Fragment>` (сокращённо `<>...</>`). Ошибка "Adjacent JSX elements must be wrapped". Логические операторы: `{false}` не рендерится, но `{count && <Comp />}` сломается при `count === 0`, так как 0 — truthy в JSX; используйте `Boolean(count)` или `count > 0`.[^7][^1]
```

## Производительность и ключи

```
Индексы в `key` при map(): `{items.map((item, i) => <Item key={i} />}` вызывает лишние ререндеры при сортировке/фильтрации; используйте уникальный `item.id`. Inline-функции в props: `<Button onClick={() => handler()}>` создаёт новую функцию при каждом рендере, ломая `React.memo`; выносите в `useCallback`.[^1]
```

| Ошибка          | Неправильно                       | Правильно [^1]                  |
| :-------------- | :-------------------------------- | :------------------------------ |
| Ключи           | `key={index}`                     | `key={item.id}`                 |
| Условный рендер | `{count && <Comp />}` при count=0 | `{count > 0 && <Comp />}`       |
| Фрагменты       | Два div без обёртки               | `<><div>1</div><div>2</div></>` |

# Самые популярные вопросы на собеседовании на эту тему.

На собеседованиях по React вопросы про JSX проверяют понимание его синтаксиса, отличий от HTML и компиляции. Чаще всего спрашивают базовые концепции и типичные ошибки.[^1][^2]

## Базовые вопросы

- Что такое JSX и чем он отличается от HTML? (Ответ: синтаксис для React.createElement, camelCase атрибуты, JS-выражения в {}).
- Как компилируется JSX? (Babel → React.createElement('tag', props, children)).
- Можно ли использовать JSX без React? (Да, как extension с @babel/plugin-transform-react-jsx).
- Зачем нужен React.Fragment или <>?</> (Обёртка для нескольких элементов без лишнего DOM-узла).[^1]

## Синтаксис и ошибки

- Почему className вместо class, а htmlFor вместо for? (JSX компилируется в JS-объекты).
- Что рендерит `{false}` или `{null}` в JSX? (Ничего — falsy значения игнорируются).
- Как правильно рендерить условно: `{condition && <Comp />}`? (Проблема при condition=0; фикс: `condition > 0`).
- Зачем ключи (key) в списках и почему не индекс? (Реакт использует для diffing; индекс ломается при мутациях).[^2][^5]

## Продвинутые

| Вопрос                   | Краткий ответ                                 |
| :----------------------- | :-------------------------------------------- |
| Разница JSX в React 17+? | Новый JSX Transform без импорта React [^1]    |
| JSX в TypeScript (.tsx)? | Типизированные props: `<Comp str={string} />` |
| Динамические теги?       | `const Tag = 'h1'; <Tag>Hi</Tag>`             |

Типичная задача: "Напишите JSX для списка с ключом" или "Пофиксите ошибку 'Adjacent JSX elements'".

# Кто еще может понимать JSX кроме babel?

Кроме Babel, JSX понимают современные быстрые транспайлеры и сборщики, ориентированные на производительность. Они часто заменяют Babel в проектах React/Next.js.[^1][^3]

## Основные альтернативы

**SWC** (Speedy Web Compiler) — основной конкурент, написан на Rust, в 20+ раз быстрее Babel. Поддерживает JSX → React.createElement из коробки, используется в Next.js 12+ по умолчанию.[^3][^5]
**esbuild** — ультрабыстрый бандлер на Go, транспилирует JSX без плагинов. Идеален для билдов, но слабее в сложных пресетах.[^3]

**TypeScript (tsc)** — встроенная поддержка JSX в .tsx файлах с опциями "react" или "preserve".[^5]

## Сравнение транспайлеров

| Инструмент | Скорость  | JSX поддержка | Использование         |
| :--------- | :-------- | :------------ | :-------------------- |
| SWC        | 20x Babel | Полная        | Next.js, Rust-плагины |
| esbuild    | 100x+     | Базовая       | Бандлинг              |
| TypeScript | Средняя   | .tsx          | TS-проекты            |
| Sucrase    | Быстрая   | Ограниченная  | Лёгкий CLI            |

## Как настроить

### SWC в Next.js (авто)

```json
// next.config.js
module.exports = { swcMinify: true };
```

### esbuild standalone

```bash
npx esbuild app.jsx --bundle --outfile=out.js --jsx-factory=React.createElement
```

### Браузерный вариант

Маленькие библиотеки вроде **jsxLoader** (5KB) или **acorn-jsx** парсят JSX на клиенте без Node.js. Для продакшена компилируйте заранее.[^4][^6]

React 17+ с JSX Transform вообще не требует импорта React в каждом файле — работает с любым транспайлером.
