# Context API: Как передавать данные без пропсов

Context API позволяет передавать данные через дерево компонентов **без пропсов** (prop drilling), создавая "глобальное" состояние для глубоко вложенных компонентов. Идеально для тем, аутентификации, локализации.

## Как работает

1. **Создаём контекст**: `const ThemeContext = React.createContext()`
2. **Provider** оборачивает дерево и предоставляет `value`
3. **useContext** или `<Context.Consumer>` получают значение из ближайшего Provider

## Полный пример

```jsx
import React, { createContext, useContext, useState } from 'react';

// 1. Создаём контекст
const ThemeContext = createContext();

// 2. Provider-компонент (обычно сверху приложения)
function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');

  // Мемоизируем value для стабильности ссылки
  const value = useMemo(() => {
    theme,
    setTheme,
    isDark: theme === 'dark'
  }, [theme]);

  return (
    <ThemeContext.Provider value={value}>
      {children}
    </ThemeContext.Provider>
  );
}

// 3. Custom hook для удобства
function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) throw new Error('useTheme must be used within ThemeProvider');
  return context;
}

// 4. Используем в любом компоненте (даже глубоко вложенном!)
function Button() {
  const { theme, setTheme, isDark } = useTheme();

  return (
    <button
      style={{ background: isDark ? 'black' : 'white', color: isDark ? 'white' : 'black' }}
      onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}
    >
      Current theme: {theme}
    </button>
  );
}

// 5. Оборачиваем App
function App() {
  return (
    <ThemeProvider>
      <div>
        <Button />
        <Toolbar /> {/* Может быть на 10 уровней глубже */}
      </div>
    </ThemeProvider>
  );
}
```

## Сравнение подходов

| С props drilling                                                      | С Context                                           |
| :-------------------------------------------------------------------- | :-------------------------------------------------- |
| `<Parent><Middle name={name}><Child name={name} /></Middle></Parent>` | `<ThemeProvider><любые компоненты></ThemeProvider>` |
| Изменение → обновить все промежуточные                                | Изменение → подписчики ререндерятся автоматически   |
| Легко отследить источник                                              | Provider может быть далеко                          |

## Важные нюансы

**✅ Стабильный `value`**:

```jsx
// Плохо — новый объект каждый рендер
<ThemeContext.Provider value={{ theme, setTheme }}>

// Хорошо — useMemo
const value = useMemo(() => ({ theme, setTheme }), [theme]);
```

**✅ Несколько контекстов**: Theme, Auth, Language — отдельные Provider'ы.

**✅ Custom hooks**: `useTheme()`, `useAuth()` — инкапсуляция + ошибки без контекста.

**Когда НЕ использовать**: Много мелких состояний (используйте Zustand/Jotai/Redux). Context для **глобальных** данных.
