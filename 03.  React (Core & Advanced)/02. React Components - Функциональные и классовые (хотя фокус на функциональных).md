# React Components - Функциональные и классовые (хотя фокус на функциональных)

React компоненты делятся на функциональные и классовые, но фокус современного React — на функциональных с хуками (useState, useEffect). Функциональные проще, быстрее и покрывают 99% случаев, классовые используются редко в новых проектах.

## Функциональные компоненты

Простая функция, принимает props и возвращает JSX. Состояние и эффекты — через хуки.

```jsx
import { useState, useEffect } from "react";

const Counter = ({ initial = 0 }) => {
  const [count, setCount] = useState(initial);

  useEffect(() => {
    document.title = `Count: ${count}`;
  }, [count]);

  return <button onClick={() => setCount(count + 1)}>Count: {count}</button>;
};
```

**Плюсы**: нет `this`, меньше кода, перерендер только при изменении зависимостей, легко тестировать.

## Классовые компоненты

ES6-класс, extends React.Component с обязательным `render()`.

```jsx
import { Component } from "react";

class Counter extends Component {
  constructor(props) {
    super(props);
    this.state = { count: props.initial || 0 };
  }

  componentDidMount() {
    document.title = `Count: ${this.state.count}`;
  }

  componentDidUpdate(prevProps, prevState) {
    if (prevState.count !== this.state.count) {
      document.title = `Count: ${this.state.count}`;
    }
  }

  render() {
    return (
      <button onClick={() => this.setState({ count: this.state.count + 1 })}>
        Count: {this.state.count}
      </button>
    );
  }
}
```

## Сравнение

| Аспект      | Функциональные | Классовые                  |
| :---------- | :------------- | :------------------------- |
| Синтаксис   | Функция → JSX  | class + render()           |
| Состояние   | `useState()`   | `this.state`               |
| Эффекты     | `useEffect()`  | `componentDidMount/Update` |
| `this`      | Нет            | Обязателен (bind/стрелки)  |
| Ререндер    | Только deps    | Всегда при setState        |
| Хуки        | ✅             | ❌                         |
| Bundle size | Меньше         | Больше                     |

## Почему функциональные предпочтительны

- **Hooks** заменили lifecycle методы: `useEffect` = componentDidMount + Update + Unmount.
- Нет проблем с `this` (bind, стрелочные функции).
- Лучшая производительность и tree-shaking.
- Легче делить логику на custom hooks.[^3]

**Рекомендация**: Новые компоненты — только функциональные. Классовые оставьте для legacy-кода или error boundaries.

# Что такое ререндер, происходит ли он при вызове хука useState?

Ререндер в React — это повторный вызов функции компонента для генерации нового Virtual DOM при изменении состояния или пропсов. Сам вызов `useState` (или его setter `setState`) **не вызывает ререндер немедленно**, но **ставит компонент в очередь на ререндер**.

## Как это работает

1. **setCount(value)** — асинхронный планировщик: React батчит обновления, ререндер происходит **на следующем тике** (не в том же стеке вызовов).
2. React вызывает функцию компонента **с новыми значениями** state.
3. Генерируется новый JSX → diffing → минимальные правки реального DOM.

```jsx
const [count, setCount] = useState(0);

console.log("Рендер!"); // Выполнится ПРИ setCount

const handleClick = () => {
  setCount(1); // ❌ НЕ рендерит СЕЙЧАС
  console.log(count); // ❌ Все еще 0 (state не обновился)
  setCount(2); // ✅ Батчится, count станет 2
};

return <button onClick={handleClick}>Count: {count}</button>;
```

## Ключевые особенности

**Батчинг**: множественные `setState` в одном обработчике сливаются в один ререндер (React 18+ батчит даже асинхронно).

**Старая value в setter**: `setCount(count + 1)` видит **предыдущее** значение, т.к. ререндер ещё не произошёл.

**Только при изменении**: если `setCount(текущее_значение)`, ререндера не будет (Object.is сравнение).

## Дерево ререндеров

```jsx
// Родитель
function Parent() {
  const [count, setCount] = useState(0);
  console.log("Parent render"); // 🔥 Срабатывает

  return <Child count={count} />; // ❌ Child тоже ререндерится!
}

// Дитя
function Child({ count }) {
  console.log("Child render"); // 🔥 Тоже!
  return <div>{count}</div>;
}
```

**Решение**: `React.memo(Child)` + стабильные пропсы.

## Проверка ререндеров

```jsx
const Comp = ({ value }) => {
  console.trace("Render:", value); // Стек вызовов покажет, кто виноват
  return <div>{value}</div>;
};
```

Ререндер = новая функция компонента. Оптимизируйте через `memo`, `useCallback`, `useMemo`.
