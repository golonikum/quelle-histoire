# Lifecycle Methods / Hooks: useEffect, useState, useContext, useReducer, useMemo, useCallback, useRef

Lifecycle методы управляют жизненным циклом классовых компонентов, а хуки — функциональные аналоги для современных компонентов React. Фокус на хуках: useEffect заменяет большинство lifecycle, остальные оптимизируют производительность и логику.

## Основные хуки

**useState** — состояние: `const [value, setValue] = useState(0);`.
**useEffect** — сайд-эффекты (fetch, подписки): `useEffect(() => { fetchData(); }, [deps]);`.
**useContext** — глобальное состояние: `const theme = useContext(ThemeContext);`.
**useReducer** — сложная логика state (альтернатива Redux): `const [state, dispatch] = useReducer(reducer, initial);`.
**useMemo** — мемоизированные вычисления: `const expensive = useMemo(() => calc(a, b), [a, b]);`.
**useCallback** — стабильные функции: `const handleClick = useCallback(() => {}, [deps]);`.
**useRef** — мутируемая ссылка (DOM, значения): `const inputRef = useRef();`.

## Lifecycle: Классы vs Хуки

| Фаза    | Классовый метод        | Хук                                  |
| :------ | :--------------------- | :----------------------------------- |
| Mount   | `componentDidMount`    | `useEffect(fn, [])`                  |
| Update  | `componentDidUpdate`   | `useEffect(fn, [deps])`              |
| Unmount | `componentWillUnmount` | `useEffect(() => () => cleanup, [])` |
| All     | -                      | `useEffect(fn)`                      |

```jsx
// Класс
class Example extends Component {
  componentDidMount() {
    fetchData();
  }
  componentDidUpdate(prevProps) {
    if (prevProps.id !== this.props.id) fetchData();
  }
  componentWillUnmount() {
    cleanup();
  }
}

// Хук (1 вызов = все фазы)
function Example({ id }) {
  useEffect(() => {
    const data = fetchData();
    return () => cleanup(data); // cleanup при unmount/update
  }, [id]);
}
```

## Практическое использование

```jsx
function TodoList() {
  const [todos, setTodos] = useState([]); // Состояние
  const [filter, setFilter] = useState("all"); // Локальное состояние
  const theme = useContext(ThemeContext); // Глобальное
  const listRef = useRef(); // DOM

  // Сайд-эффекты
  useEffect(() => {
    const load = async () => setTodos(await fetchTodos());
    load();
  }, []);

  // useReducer для сложной логики
  const [state, dispatch] = useReducer(todoReducer, { todos, filter });

  // Оптимизация
  const visibleTodos = useMemo(
    () => todos.filter((todo) => filter === "all" || todo.status === filter),
    [todos, filter],
  );
  const addTodo = useCallback((text) => {
    setTodos((prev) => [...prev, { id: Date.now(), text, status: "pending" }]);
  }, []);

  return (
    <div ref={listRef} className={theme}>
      <TodoList visibleTodos={visibleTodos} onAdd={addTodo} />
    </div>
  );
}
```

## Когда какой хук

- **useState**: простое состояние (число, строка, объект).
- **useReducer**: сложная логика, много setState, как Redux.
- **useMemo**: тяжелые вычисления в рендере.
- **useCallback**: функции-пропсы для React.memo.
- **useRef**: DOM-узлы, таймеры, предыдущее значение (`prevRef.current = value`).

**Правило**: хуки всегда в том же порядке, сверху вниз, только в корне компонента.

# Какие каверзные вопросы на собеседовании для Senior React могут быть по этой теме?

Для Senior React-разработчика по хукам и lifecycle задают вопросы, проверяющие глубокое понимание механик, оптимизации и архитектуры. Вот самые каверзные:

## Механика хуков (подводные камни)

**1. Почему `useEffect` с пустыми `[]` может не сработать дважды в StrictMode?**
React 18+ в development дважды монтирует/размонтирует компоненты для выявления side-effects. Cleanup `useEffect(() => () => ws.close(), [])` выполняется между вызовами.[^1]

## Оптимизация (ловушки)

**3. Зачем `useCallback` если есть `React.memo`?**

```jsx
const Child = React.memo(({ onClick }) => (
  <button onClick={onClick}>Click</button>
));
const Parent = () => {
  const handleClick = () => {}; // Новый объект каждый рендер!
  return <Child onClick={handleClick} />; // ❌ memo не спасает
};
```

**Ответ**: `memo` сравнивает **props по ссылке**. Новый `onClick={() => {}}` каждый раз → Child ререндерится. `useCallback` стабилизирует ссылку.

**4. Когда `useMemo` ухудшает производительность?**
При частых ререндерах с малыми вычислениями overhead мемо больше пользы. Или когда deps меняются каждый рендер:

```jsx
const expensive = useMemo(() => calc(items, Date.now()), [items, Date.now()]); // ❌ Бесполезно
```

## Архитектура и паттерны

**5. useReducer vs Redux — когда какой?**
`useReducer` — локальная сложная логика в компоненте. Redux — глобальное состояние, middleware (thunk/saga), DevTools, коллаборация.

**6. Как реализуете `useEffect` с race conditions?**

```jsx
useEffect(() => {
  let mounted = true;
  fetchData().then((data) => {
    if (mounted) setData(data); // ❌ Race condition
  });
  return () => {
    mounted = false;
  };
}, []);
```

**Senior ответ**: AbortController + isCurrentId:

```jsx
const [currentAbort, setCurrentAbort] = useState();
useEffect(() => {
  const abort = new AbortController();
  setCurrentAbort(abort);

  fetchData(abort.signal).then((data) => {
    if (abort.signal.aborted) return;
    setData(data);
  });

  return () => abort.abort();
}, []);
```

## Теория (очень senior)

**7. Как React internally хранит хуки между ререндерами?**
Linked list в fiber node: `hook.queue = { pending: null, dispatch }`. Каждый вызов хука идёт по списку `currentHook = currentHook.next`.[^1]

**8. useLayoutEffect vs useEffect — когда какой?**
`useLayoutEffect` синхронно после мутаций DOM, перед paint. Для измерений/scroll. `useEffect` асинхронно после paint — для fetch/подписок.

**Задача live-coding**: "Напишите debounce-хук с отменой и cleanup":

```jsx
function useDebounce(callback, delay) {
  const timeout = useRef();

  return useCallback(
    (...args) => {
      clearTimeout(timeout.current);
      timeout.current = setTimeout(() => callback(...args), delay);
    },
    [callback, delay],
  );
}
```

**Красный флаг**: кандидат не знает про stale closures, race conditions или говорит "просто добавлю useCallback везде".
