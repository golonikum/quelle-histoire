# Props \& State: Разница, как обновлять state

Props передаются от родителя к дочерним компонентам и являются **read-only** — их нельзя менять внутри принимающего компонента. State — это **внутреннее изменяемое состояние** компонента, управляемое только им самим.

## Разница между Props и State

| Аспект        | Props                   | State             |
| :------------ | :---------------------- | :---------------- |
| **Источник**  | Родительский компонент  | Сам компонент     |
| **Изменение** | ❌ Только родитель      | ✅ `setState()`   |
| **Область**   | Между компонентами      | Локально          |
| **Ререндер**  | При изменении родителем | При `setState()`  |
| **Пример**    | `color`, `items`        | `isOpen`, `count` |

```jsx
// Родитель
function Parent() {
  const [count, setCount] = useState(0);
  return <Child count={count} onIncrement={() => setCount(count + 1)} />;
}

// Дитя (НЕ меняет props!)
function Child({ count, onIncrement }) {
  return <button onClick={onIncrement}>Count: {count}</button>;
}
```

## Как обновлять State

**Только через setter** из `useState`, никогда напрямую:

```jsx
const [count, setCount] = useState(0);

// ✅ Правильно
const increment = () => setCount(count + 1); // Функциональное обновление
const incrementFn = () => setCount((prev) => prev + 1); // Лучше (стабильно)

// ❌ Неправильно
count = count + 1; // Мутация! Не сработает
setCount(count + 1); // Может работать случайно (stale closure)
```

### Особенности обновления

1. **Асинхронно**: `setCount(1); console.log(count);` → `count` всё ещё старое.
2. **Батчинг**: несколько `setState` в одном обработчике → один ререндер.
3. **Функциональный вариант** (`prev => prev + 1`) — всегда видит актуальное значение.

### Правило "один источник правды"

```
isOpen     → state в модалке
userData   → props от родителя
filteredItems → state (фильтр локальный)
```

**Принцип**: если данные могут измениться в компоненте → state. Если только читаются → props.

# Какие вопросы на собеседовании могут быть по этой теме?

На собеседованиях по React вопросы про компоненты, props и state проверяют понимание архитектуры и принципов unidirectional data flow. Чаще спрашивают про функциональные компоненты, но сравнение с классовыми тоже популярно.

## Базовые вопросы

- **Разница между функциональными и классовыми компонентами?**
  Функциональные — простые функции с хуками (useState/useEffect), классовые — ES6-классы с this.state и lifecycle методами. Функциональные проще, быстрее, рекомендуются.
- **Props vs State — в чём разница?**
  Props — read-only данные от родителя, State — изменяемое внутреннее состояние через setState.
- **Как обновлять state? Почему не напрямую?**
  Только через setter: `setCount(prev => prev + 1)`. Прямое `count++` не триггерит ререндер.

## Практические задачи

```
Реализуйте счётчик:
const Counter = () => {

```

## Типичные ошибки для обсуждения

- **Мутация state**: `state.push(item)` вместо `setState([...state, item])`.
- **Обновление props**: попытка `this.props.count++`.
- **Синхронность**: `setCount(1); console.log(count)` → count всё ещё старое.[^1]

**Задача на код**: "Пофиксите: дочерний компонент меняет props родителя напрямую" → передать setter как prop.
