# Области видимости в JS: var, let, const, scope, hoisting

JavaScript имеет три способа объявления переменных: `var`, `let` и `const`. Они различаются по области видимости (scope) и поведению hoisting (поднятия объявлений).

## Области видимости (Scope)

JavaScript поддерживает **глобальную**, **функциональную** и **блочную** области видимости.

**var** — функциональная область видимости (Function Scope).
**let/const** — блочная область видимости (Block Scope) + функциональная.

```js
{
  var a = 1; // Функциональная scope
  let b = 2; // Блочная scope
  const c = 3; // Блочная scope
}

console.log(a); // 1 (доступно!)
console.log(b); // ReferenceError
console.log(c); // ReferenceError
```

## Hoisting (Поднятие)

Объявления переменных поднимаются в начало своей области видимости, но **инициализация** — нет.

### var — поднимается с `undefined`

```js
console.log(x); // undefined (поднято!)
var x = 5;

console.log(y); // ReferenceError (TDZ!)
let y = 10;
```

**Под капотом**:

```js
// var
var x; // hoisted: undefined
console.log(x);
x = 5;

// let/const — Temporal Dead Zone (TDZ)
let y; // hoisted, но недоступно до объявления
```

## Сравнение в таблице

| Характеристика        | `var`          | `let`         | `const`         |
| :-------------------- | :------------- | :------------ | :-------------- |
| **Область видимости** | Функциональная | Блочная       | Блочная         |
| **Hoisting**          | `undefined`    | TDZ           | TDZ             |
| **Переобъявление**    | ✅             | ❌            | ❌              |
| **Переприсваивание**  | ✅             | ✅            | ❌              |
| **Инициализация**     | Необязательна  | Необязательна | **Обязательна** |

## Практические примеры

### Цикл for (классическая проблема var)

```js
// ❌ var — общая переменная для всех итераций
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0); // 3, 3, 3
}

// ✅ let — новая i в каждой итерации
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0); // 0, 1, 2
}
```

### Функциональная область видимости

```js
function test() {
  var a = 1;
  let b = 2;

  if (true) {
    var a = 10; // Та же переменная!
    let b = 20; // Новая переменная
    console.log(a); // 10
    console.log(b); // 20
  }

  console.log(a); // 10
  console.log(b); // 2
}
```

### const — константы и объекты

```js
const num = 42;
num = 100; // ❌ TypeError

const obj = { a: 1 };
obj.a = 2; // ✅ Работает (ссылка неизменна)
obj = {}; // ❌ TypeError
```

## Когда что использовать

- **`const`** — значения, которые не переприсваиваются (90% случаев)
- **`let`** — изменяемые значения в блоке
- **`var`** — **никогда** (устарело, вызывает баги)

```js
// ✅ Правильно
const users = [];
const config = { theme: "dark" };
let counter = 0;
```

**Правило**: "const по умолчанию, let когда нужно менять значение".[^1]

# Пример вопроса на собеседовании по поводу var, let, const

**Типичный вопрос на собеседовании (middle+ уровень):**

````
Что выведет этот код? Объясните разницу между var, let и const.

```javascript
console.log(a);  // ?
var a = 1;

console.log(b);  // ?
let b = 2;

console.log(c);  // ?
const c = 3;

if (true) {
  var x = 10;
  let y = 20;
  const z = 30;
  console.log(x, y, z);  // ?
}

console.log(x);  // ?
console.log(y);  // ?
console.log(z);  // ?
````

```

**Ожидаемый ответ (что покажет кандидат):**

```

undefined
ReferenceError: Cannot access 'b' before initialization
ReferenceError: Cannot access 'c' before initialization
10 20 30
10
ReferenceError: y is not defined
ReferenceError: z is not defined

````

**Разбор для интервьюера:**

1. **`var a`** — hoisting поднимает объявление с `undefined`
2. **`let/const b, c`** — Temporal Dead Zone (TDZ), ошибка до инициализации
3. **`var x`** — функциональная область видимости, доступно после `if`
4. **`let/const y, z`** — блочная область видимости `{}` внутри `if`

**Следующий вопрос (продвинутый):**
```javascript
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
// Что выведет? Как исправить с let/const?
````

**Правильный ответ:** `3 3 3` → исправить: `for (let i = 0; i < 3; i++)`

**Для middle React dev это must-have** — показывает понимание closure, scope и типичных багов с циклами в колбэках.
