# Замыкания (Closures): Принцип работы, примеры использования

Замыкания — это способность функции "запоминать" лексическое окружение, в котором она была создана. Внутренняя функция сохраняет доступ к переменным внешней даже после завершения внешней.

## Принцип работы

1. **Лексическое окружение** — цепочка областей видимости (scope chain)
2. При создании функции сохраняется **ссылка** на внешнее окружение
3. Внутренняя функция "замыкает" (захватывает) переменные внешней

```js
function outer() {
  const secret = "Я приватная!"; // Лексическое окружение

  function inner() {
    console.log(secret); // Доступ через замыкание
  }

  return inner; // Возвращаем функцию
}

const fn = outer();
fn(); // 'Я приватная!' — secret жива!
```

## Проблема с var в циклах

```js
// ❌ var — одна переменная на все итерации
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0); // 3, 3, 3
}
```

```js
// ✅ let — новая i для каждой итерации (благодаря блочной scope)
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0); // 0, 1, 2
}
```

## Практические примеры

### 1. Счетчик (инкапсуляция)

```js
function createCounter() {
  let count = 0; // Приватная переменная

  return {
    increment: () => ++count,
    decrement: () => --count,
    getCount: () => count,
  };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.getCount()); // 1
console.log(count); // ReferenceError! Приватно
```

### 2. Функция с параметрами

```js
function createMultiplier(factor) {
  return function (x) {
    return x * factor;
  };
}

const double = createMultiplier(2);
const triple = createMultiplier(3);

console.log(double(5)); // 10
console.log(triple(5)); // 15
```

### 3. Обработчики событий (React паттерн)

```js
function createButtons() {
  for (let i = 0; i < 3; i++) {
    const button = document.createElement("button");
    button.textContent = `Button ${i}`;
    button.onclick = () => alert(`Clicked button ${i}`); // ✅ Правильно
    document.body.appendChild(button);
  }
}
```

## React контекст (middle+ уровень)

```js
function useCounter(initial = 0) {
  const [count, setCount] = useState(initial);

  // Замыкание на count через useCallback
  const increment = useCallback(() => {
    setCount((c) => c + 1);
  }, []); // [] — захватывает только внешние deps

  return { count, increment };
}
```

**Ключевое правило**: Замыкание = функция + её лексическое окружение. Внешние переменные живут, пока жива возвращенная функция.

**Собеседование**: "Что выведет `for (let i=0; i<3; i++) setTimeout(() => console.log(i), 0)`?" → `0, 1, 2` (блочная scope).

## Типичные вопросы по замыканиям

### 1. Классический цикл (middle уровень)

````
Что выведет код?
```javascript
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
````

**Ответ**: `3 3 3` (var — функциональная scope).
**Исправление**: `for (let i = 0; i < 3; i++)` → `0 1 2`.

### 2. Создание счетчика (middle+)

```
Напишите функцию `createCounter()`, которая возвращает объект с методами `increment()`, `decrement()`, `get()`. Счетчик должен быть приватным.
```

**Правильный ответ**:

```javascript
function createCounter() {
  let count = 0;
  return {
    increment: () => ++count,
    decrement: () => --count,
    get: () => count,
  };
}
```

### 3. Функция с N вызовами (senior)

```
Напишите `createThrottler(n)`, которая первые N раз возвращает `'yes'`, потом `'no'`.
```

```javascript
function createThrottler(n) {
  let calls = 0;
  return () => (calls++ < n ? "yes" : "no");
}

const fn = createThrottler(3);
console.log(fn()); // 'yes'
console.log(fn()); // 'yes'
console.log(fn()); // 'no'
```

**Проблема**: Создается новая функция для каждого `i` при каждом рендере.
**Решение**: `useCallback` + замыкание или `let` в цикле.

# Практические применения замыканий в React

Замыкания в React — основа функциональных компонентов. Все хуки (`useState`, `useEffect`, `useCallback`) работают именно через замыкания, захватывая состояние и пропсы.

## 1. Кастомные хуки (приватное состояние)

```jsx
function useCounter(initial = 0) {
  const [count, setCount] = useState(initial);

  // Замыкание на count через useCallback
  const increment = useCallback(() => {
    setCount((c) => c + 1);
  }, []);

  const decrement = useCallback(() => {
    setCount((c) => c - 1);
  }, []);

  return { count, increment, decrement };
}

// Использование
function Counter() {
  const { count, increment, decrement } = useCounter(0);
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={increment}>+</button>
      <button onClick={decrement}>-</button>
    </div>
  );
}
```

## 2. Debounce/Throttle хук

```jsx
function useDebounce(callback, delay) {
  const timeoutRef = useRef();

  return useCallback(
    (...args) => {
      clearTimeout(timeoutRef.current);
      timeoutRef.current = setTimeout(() => callback(...args), delay);
    },
    [callback, delay],
  );
}

// Использование для поиска
function Search() {
  const [query, setQuery] = useState("");
  const [results, setResults] = useState([]);

  const debouncedSearch = useDebounce(async (searchTerm) => {
    const data = await fetch(`/api/search?q=${searchTerm}`).then((r) =>
      r.json(),
    );
    setResults(data);
  }, 300);

  const handleSearch = (e) => {
    const term = e.target.value;
    setQuery(term);
    debouncedSearch(term);
  };

  return (
    <div>
      <input value={query} onChange={handleSearch} />
      <ul>
        {results.map((r) => (
          <li key={r.id}>{r.name}</li>
        ))}
      </ul>
    </div>
  );
}
```

## 4. Фикс проблемы с циклами в рендере

```jsx
// ❌ Плохо — новая функция каждый рендер
function BadList({ items }) {
  return items.map((item, index) => (
    <button key={item.id} onClick={() => console.log(index)}>
      {item.name}
    </button>
  ));
}

// ✅ Хорошо — замыкание + useCallback
function GoodList({ items }) {
  const handleClick = useCallback((index) => {
    console.log(index);
  }, []);

  return items.map((item, index) => (
    <button key={item.id} onClick={() => handleClick(index)}>
      {item.name}
    </button>
  ));
}
```

## 5. Мемоизация с зависимостями

```jsx
function useAsyncData(fetchFn, deps) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(false);

  useEffect(() => {
    setLoading(true);
    fetchFn()
      .then(setData)
      .finally(() => setLoading(false));
  }, deps); // Замыкание захватывает deps

  return { data, loading };
}
```

## Проблемы и решения

| Проблема                                 | Решение                                |
| :--------------------------------------- | :------------------------------------- |
| **Устаревшие замыкания** (stale closure) | `useCallback` с deps или `useRef`      |
| **Лишние ререндеры**                     | Правильные зависимости в `useCallback` |
| **Память**                               | `useEffect` cleanup для таймеров       |

```jsx
// ❌ Устаревшее замыкание
const increment = useCallback(() => {
  setCount(count + 1); // count — замыкание на старое!
}, []);

// ✅ Функциональное обновление
const increment = useCallback(() => {
  setCount((c) => c + 1); // Всегда актуально
}, []);
```

**Правило**: Замыкания — сердце React хуков. `useCallback`, `useEffect`, `useMemo` всегда работают через closure на состояние компонента.

**Красные флаги**

    ❌ "Замыкания не используются в React"

    ❌ Игнорирует useCallback для массивов

    ❌ Пишет useEffect(() => {}, []) без объяснения

    ❌ Не знает про функциональные обновления setState(prev => ...)

**Идеальный ответ**: "Замыкания — основа хуков. `useState` захватывает состояние через closure, поэтому нужен `useCallback` для оптимизации и функциональные обновления для актуальности."
