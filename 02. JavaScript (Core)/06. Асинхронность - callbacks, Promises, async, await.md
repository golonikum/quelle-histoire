# Асинхронность: callbacks, Promises, async/await

JavaScript — однопоточный, но **неблокирующий**. Асинхронность решает "callback hell" и позволяет работать с сетью, таймерами, файлами без заморозки UI.

## 1. Callbacks (Пирамида смерти)

Функция передается как аргумент и вызывается после завершения операции.

```js
// ✅ Простой пример
setTimeout(() => {
  console.log('Через 1с');
}, 1000);

// ❌ Callback hell
getUser(1, (user) => {
  getPosts(user.id, (posts) => {
    getComments(posts[^0].id, (comments) => {
      processData(comments);  // Глубина 4+
    });
  });
});
```

## 2. Promises (ES6)

Объект с состояниями: `pending` → `fulfilled`/`rejected`.

```js
const promise = new Promise((resolve, reject) => {
  setTimeout(() => {
    const success = true;
    success ? resolve("Успех!") : reject("Ошибка!");
  }, 1000);
});

// then/catch/finally
promise
  .then((result) => console.log(result)) // 'Успех!'
  .catch((error) => console.error(error))
  .finally(() => console.log("Завершено"));
```

**Цепочки**:

```js
fetchUser(1)
  .then(user => fetchPosts(user.id))
  .then(posts => fetchComments(posts[^0].id))
  .then(comments => processData(comments));
```

## 3. async/await (ES8)

Синтаксический сахар над Promise. `await` **приостанавливает** выполнение async функции.

```js
async function fetchUserData() {
  try {
    const user = await fetchUser(1);
    const posts = await fetchPosts(user.id);
    const comments = await fetchComments(posts[^0].id);
    return processData(comments);
  } catch (error) {
    console.error('Ошибка:', error);
  }
}
```

## Сравнение подходов

| Аспект               | Callback           | Promise         | async/await     |
| :------------------- | :----------------- | :-------------- | :-------------- |
| **Читаемость**       | ❌ Пирамида        | ✅ Цепочка      | ✅ Линейный     |
| **Ошибки**           | ❌ Ручная проверка | `.catch()`      | `try/catch`     |
| **Параллельность**   | ❌                 | `Promise.all()` | `Promise.all()` |
| **Композабельность** | ❌                 | ✅              | ✅              |

## Параллельное выполнение

```js
// Promise.all — все завершатся успешно или первый reject
const results = await Promise.all([fetchUser(1), fetchPosts(10), fetchStats()]);

// Promise.allSettled — все до конца
const results = await Promise.allSettled([
  fetchUser(1),
  fetchUser(999), // Ошибка, но остальные выполнятся
]);

// Promise.race — первый завершившийся
const fastest = await Promise.race([
  fetchUser(1),
  new Promise((resolve) => setTimeout(() => resolve("timeout"), 100)),
]);
```

## React примеры (ваш стек)

## Типичные ошибки

```js
// ❌ Неправильно — await заблокирует
async function bad() {
  const data1 = await fetch("/api/1");
  const data2 = await fetch("/api/2"); // Ждет data1!
}

// ✅ Promise.all для параллельности
async function good() {
  const [data1, data2] = await Promise.all([fetch("/api/1"), fetch("/api/2")]);
}
```

## Event Loop (упрощенно)

```
1. Call stack пуст → берём из Task Queue
2. Microtasks (.then, await) → перед Macrotasks (setTimeout)
3. Выполняем render после microtasks
```

**Правило**: `async/await` для читаемости, `Promise.all` для скорости, **никогда** не используйте callback hell.

## Типичные вопросы по асинхронности на собеседовании

### 1. Event Loop порядок (Middle must-have)

````
Что выведет код?
```javascript
console.log('1');

setTimeout(() => console.log('2'), 0);

Promise.resolve().then(() => console.log('3'));

console.log('4');
````

**Ответ**: `1 4 3 2`
**Объяснение**:

1. Синхронный код (`console.log`)
2. Microtasks (Promises `.then`)
3. Macrotasks (`setTimeout`)

### 2. Promise цепочка (Middle)

````
Какой порядок вывода?
```javascript
console.log('start');

setTimeout(() => console.log('timeout'), 0);

Promise.resolve()
  .then(() => {
    console.log('promise1');
    return Promise.resolve('promise2');
  })
  .then(console.log);

console.log('end');
````

**Ответ**: `start end promise1 promise2 timeout`

### 3. async/await vs Promise.all (Middle+)

````
Что быстрее выполнится?
```javascript
// Вариант 1
async function sequential() {
  const user = await fetchUser(1);
  const posts = await fetchPosts(user.id);
  return posts;
}

// Вариант 2
async function parallel() {
  const [user, posts] = await Promise.all([
    fetchUser(1),
    fetchPosts(1)
  ]);
  return posts;
}
````

**Ответ**: `parallel` — выполняет запросы **параллельно**.

### 4. React useEffect ловушка

````
Сколько раз выполнится эффект?
```jsx
function Component() {
  const [count, setCount] = useState(0);

  useEffect(async () => {
    const data = await fetch('/api/data');
    setCount(data.value);
  }, []);  // Пустой массив

  return <div>{count}</div>;
}
````

**Ответ**: **1 раз**, но `async` функция возвращает Promise. React игнорирует результат.

**Правильно**:

```jsx
useEffect(() => {
  fetch("/api/data").then((data) => {
    setCount(data.value);
  });
}, []);
```

### 5. Promise.allSettled vs Promise.all

```
В чем разница? Покажите пример.
```

**Ответ**:

```javascript
const promises = [
  Promise.resolve(1),
  Promise.reject("Ошибка"),
  fetch("/api/slow"),
];

// Promise.all — остановится на первой ошибке
Promise.all(promises).catch((err) => console.log("Ошибка в любом"));

// Promise.allSettled — дождется всех
Promise.allSettled(promises).then((results) => {
  results.forEach((result, i) => {
    if (result.status === "fulfilled") {
      console.log(`Promise ${i}:`, result.value);
    } else {
      console.log(`Promise ${i} rejected:`, result.reason);
    }
  });
});
```

### 6. TanStack Query контекст (ваш стек)

```
Зачем нужен `enabled: false` в useQuery?
```

```javascript
const { data } = useQuery({
  queryKey: ["user", userId],
  queryFn: () => fetchUser(userId),
  enabled: !!userId, // Не запрашивать при пустом userId
});
```

**Ответ**: Предотвращает запросы до готовности зависимостей (userId).

### 7. Callback Hell → Promise → async/await

````
Перепишите callback в async/await:
```javascript
getUser(1, (err, user) => {
  if (err) return handleError(err);
  getPosts(user.id, (err, posts) => {
    if (err) return handleError(err);
    process(posts);
  });
});
````

````
**Ответ**:
```javascript
try {
  const user = await getUser(1);
  const posts = await getPosts(user.id);
  process(posts);
} catch (err) {
  handleError(err);
}
````

## Event Loop приоритеты

```
Синхронный код → Microtasks (.then, await, queueMicrotask) → Macrotasks (setTimeout, fetch)
```

```
console.log('script start');
setTimeout(() => console.log('timeout'), 0);
queueMicrotask(() => console.log('microtask'));
Promise.resolve().then(() => console.log('promise'));
console.log('script end');
// script start → script end → promise → microtask → timeout
```

## Что проверяет интервьюер

| Уровень | Тема                                      | Красный флаг                  |
| :------ | :---------------------------------------- | :---------------------------- |
| Junior  | `setTimeout(..., 0)`                      | Думает, что "немедленно"      |
| Middle  | Microtasks vs Macrotasks                  | Путает `.then` и `setTimeout` |
| Middle+ | `Promise.all` vs последовательный `await` | Не знает про параллельность   |
| Senior  | React + async pitfalls                    | `useEffect(async () => {})`   |

**Золотое правило**: "Сначала синхронный стек, потом микрозадачи, потом макрозадачи. `await` = `.then()` под капотом."
