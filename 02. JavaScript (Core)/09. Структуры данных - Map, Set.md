## 🗃️ **Map/Set vs Object/Array + O-нотация (Middle+/Senior)**

### **🎯 Когда использовать что**

| Задача                   | ✅ **Object** | ✅ **Map** | ✅ **Set** | ✅ **Array** |
| :----------------------- | :------------ | :--------- | :--------- | :----------- |
| **Конфиг** `config.port` | ✅ O(1)       | ❌         | ❌         | ❌           |
| **Кэш** `userId → data`  | ❌            | ✅ O(1)    | ❌         | ❌           |
| **Уникальные ID**        | ❌            | ❌         | ✅ O(1)    | ❌ O(n)      |
| **Список** `[1,2,3]`     | ❌            | ❌         | ❌         | ✅ O(1)      |
| **Объект как ключ**      | ❌            | ✅         | ❌         | ❌           |

### **🔥 Ключевые отличия**

#### **1. Object vs Map — ключи**

```javascript
const obj = {};
obj[^1] = 'число';
obj['1'] = 'строка';
console.log(obj[^1]); // "строка" ❌ ключи → string

const map = new Map();
map.set(1, 'число');
map.set('1', 'строка');
console.log(map.get(1)); // "число" ✅ разные ключи
console.log(map.size); // 2
```

#### **2. Array vs Set — уникальность**

```javascript
const arr = [1, 1, 2, 2];
console.log(arr.includes(1)); // O(n) ❌ медленно

const set = new Set(arr);
console.log(set.has(1)); // O(1) ✅ быстро
console.log([...set]); // [1, 2] дедупликация
```

### **⚡ Сложность операций (O-нотация)**

| Операция            | **Object** | **Map**  | **Set**  | **Array** |
| :------------------ | :--------- | :------- | :------- | :-------- |
| **get/поиск**       | O(1)       | **O(1)** | **O(1)** | O(n)      |
| **set/добавление**  | O(1)       | **O(1)** | **O(1)** | O(1)      |
| **delete/удаление** | O(n)❌     | **O(1)** | **O(1)** | O(n)❌    |
| **size**            | O(n)❌     | **O(1)** | **O(1)** | O(1)      |
| **Итерация**        | O(n)       | **O(n)** | **O(n)** | **O(n)**  |

### **🎯 Практические сценарии**

#### **✅ Map — кэш пользователей**

```javascript
const userCache = new Map(); // userId → {name, email}

function getUser(userId) {
  if (!userCache.has(userId)) {
    userCache.set(userId, fetchUser(userId)); // O(1)
  }
  return userCache.get(userId); // O(1)
}
```

#### **✅ Set — проверка просмотренных**

```javascript
const viewedPosts = new Set();

function markAsViewed(postId) {
  viewedPosts.add(postId); // O(1), дубли игнор
}

function isViewed(postId) {
  return viewedPosts.has(postId); // O(1) vs array.includes O(n)
}
```

#### **✅ Object — конфигурация**

```javascript
const config = {
  port: 3000,
  env: "production",
};
// config.port → O(1), JSON.stringify работает
```

### **🚨 Типичные собес-задачи**

**1. "Дедуплицировать массив"**

```javascript
// ❌ O(n²)
const unique = arr.filter((item, i) => arr.indexOf(item) === i);

// ✅ O(n)
const unique = [...new Set(arr)];
```

**2. "Подсчёт частоты слов"**

```javascript
// ✅ Map
const wordCount = new Map();
text.split(" ").forEach((word) => {
  wordCount.set(word, (wordCount.get(word) || 0) + 1);
});
```

**3. "LRU Cache" (Senior)**

```javascript
class LRUCache {
  constructor(size) {
    this.max = size;
    this.cache = new Map();
  }

  get(key) {
    if (this.cache.has(key)) {
      const value = this.cache.get(key);
      this.cache.delete(key); // O(1)
      this.cache.set(key, value); // recent
      return value;
    }
  }
}
```

### **🔥 Вопросы собеседования**

**Q: "Map vs Object производительность?"**

```
Map: добавление/удаление O(1), size O(1)
Object: delete O(n), Object.keys().length O(n)
```

**Q: "Зачем Set если есть includes()?"**

```
Set.has() = O(1), array.includes() = O(n)
10k элементов: Set = 0.1ms, Array = 50ms
```

**Q: "Object.keys() порядок?"**

```
ES2015+: порядок вставки строковых ключей
НЕ РЕЛИАБИЛЬНО! Используй Map для порядка
```

### **🎯 Правило выбора**

```
Строковые ключи + JSON = Object
Любые ключи + частые mutating = Map
Уникальные значения = Set
Индексы 0..n = Array
```

**Формула**: **O(1) поиск = Map/Set, O(n) = Array.includes()** 🚀
