# Прототипы и Прототипное наследование: Как работают объекты в JS.

JavaScript использует **прототипное наследование** — каждый объект имеет ссылку на прототип (другой объект), от которого наследует свойства и методы. Нет классов как в Java/C\#, только объекты и их цепочки прототипов.

## Как работают прототипы

1. **У каждого объекта есть `[[Prototype]]`** (внутреннее свойство)
2. **Поиск свойства**: объект → прототип → прототип прототипа → `Object.prototype` → `null`
3. **Запись**: всегда в **собственные** свойства объекта (не в прототип)

```js
const animal = {
  eats: true,
  walk() {
    console.log(`${this.name} гуляет`);
  },
};

const rabbit = {
  name: "Белый кролик",
  __proto__: animal, // устаревший синтаксис (для примера)
};

console.log(rabbit.eats); // true (найдено в прототипе)
rabbit.jump = () => console.log("Прыжок!"); // Собственное свойство
```

## Цепочка прототипов

```
rabbit → animal → Object.prototype → null
```

```js
console.log(rabbit.hasOwnProperty("eats")); // false (наследовано)
console.log(rabbit.hasOwnProperty("jump")); // true (собственное)
console.log(rabbit.toString()); // "[object Object]" (Object.prototype)
```

## Современные методы

### Object.create()

```js
const animal = { eats: true };
const rabbit = Object.create(animal);
rabbit.name = "Кролик";

console.log(rabbit.eats); // true (прототип)
console.log(Object.getPrototypeOf(rabbit) === animal); // true
```

### Конструкторы + prototype

```js
function Animal(name) {
  this.name = name;
}

Animal.prototype.walk = function () {
  console.log(`${this.name} идет`);
};

function Rabbit(name) {
  Animal.call(this, name); // Наследуем свойства
}

Rabbit.prototype = Object.create(Animal.prototype); // Наследуем методы
Rabbit.prototype.constructor = Rabbit;

const rabbit = new Rabbit("Кролик");
rabbit.walk(); // 'Кролик идет'
```

## ES6 class (синтаксический сахар)

```js
class Animal {
  constructor(name) {
    this.name = name;
  }

  walk() {
    console.log(`${this.name} идет`);
  }
}

class Rabbit extends Animal {
  jump() {
    console.log(`${this.name} прыгает`);
  }
}

const rabbit = new Rabbit("Кролик");
rabbit.walk(); // 'Кролик идет'
rabbit.jump(); // 'Кролик прыгает'
```

**Под капотом**:

```js
Rabbit.prototype.__proto__ === Animal.prototype; // true
Rabbit.prototype.__proto__.__proto__ === Object.prototype; // true
```

## Практика для React

### 1. Расширение встроенных типов

```js
// Добавляем метод ко всем массивам
Array.prototype.last = function () {
  return this[this.length - 1];
};

console.log([1, 2, 3].last()); // 3 (работает везде!)
```

### 2. Mixins (паттерн)

```js
const Flyable = {
  fly() {
    console.log(`${this.name} летит`);
  },
};

const Swimmable = {
  swim() {
    console.log(`${this.name} плывет`);
  },
};

const Duck = Object.create(Flyable);
Object.assign(Duck, Swimmable);
Duck.name = "Утка";

Duck.fly(); // 'Утка летит'
Duck.swim(); // 'Утка плывет'
```

## Таблица методов прототипов

| Метод                               | Что делает                          |
| :---------------------------------- | :---------------------------------- |
| `Object.create(proto)`              | Создает объект с прототипом `proto` |
| `Object.getPrototypeOf(obj)`        | Возвращает `[[Prototype]]`          |
| `Object.setPrototypeOf(obj, proto)` | Меняет прототип                     |
| `obj.hasOwnProperty(prop)`          | Собственное свойство?               |
| `Object.prototype.toString`         | `[object Object]`                   |

## Типичные ошибки

```js
// ❌ Запись в прототип (не работает)
function User() {}
User.prototype.name = "Иван";
const user = new User();
user.name = "Петр"; // Собственное свойство!
console.log(user.name); // 'Петр'

// ✅ Чтение из прототипа
delete user.name; // Удаляем собственное
console.log(user.name); // 'Иван' (прототип)
```

**Ключевое правило**: **Поиск вверх по цепочке прототипов, запись только в собственные свойства**.[^1]

## Типичные вопросы по прототипам на собеседовании

### 1. Базовый поиск свойства (Middle)

````
Что выведет код? Объясните поиск свойства.
```javascript
const animal = { eats: true };
const rabbit = Object.create(animal);
rabbit.jumps = false;

console.log(rabbit.eats);     // ?
console.log(rabbit.jumps);    // ?
console.log(rabbit.fly);      // ?
console.log(rabbit.hasOwnProperty('eats'));  // ?
````

**Ответ**:

```
true           // Прототип
false          // Собственное свойство
undefined      // Не найдено в цепочке
false          // hasOwnProperty проверяет только собственные
```

### 2. Запись vs чтение (Middle)

````
Что произойдет после выполнения?
```javascript
function Animal() {}
Animal.prototype.eats = true;

const cat = new Animal();
cat.eats = false;  // Собственное свойство

console.log(cat.eats);  // ?
delete cat.eats;
console.log(cat.eats);  // ?
````

**Ответ**:

```
false  // Собственное свойство
true   // Прототип (после delete)
```

### 3. Цепочка прототипов (Middle+)

````
Какова цепочка прототипов? Проверьте в консоли.
```javascript
class Animal {}
class Rabbit extends Animal {}

const rabbit = new Rabbit();
````

**Ответ**: `rabbit → Rabbit.prototype → Animal.prototype → Object.prototype → null`

```javascript
console.log(Object.getPrototypeOf(rabbit) === Rabbit.prototype); // true
console.log(Object.getPrototypeOf(Rabbit.prototype) === Animal.prototype); // true
```

### 4. Наследование конструкторов (Senior)

````
Что не так с этим кодом? Исправьте.
```javascript
function Animal(name) {
  this.name = name;
}
Animal.prototype.walk = function() { console.log(this.name + ' идет'); };

function Rabbit(name) {
  Animal.call(this, name);
}
Rabbit.prototype = Animal.prototype;  // ❌ Ошибка!

const rabbit = new Rabbit('Кролик');
rabbit.walk();  // ?
````

**Проблема**: `Rabbit.prototype` и `Animal.prototype` — **один объект**. Изменения Rabbit сломают Animal.
**Исправление**:

```javascript
Rabbit.prototype = Object.create(Animal.prototype);
Rabbit.prototype.constructor = Rabbit;
```

### 5. ES6 class под капотом (Senior)

````
Какой код генерирует этот class?
```javascript
class Rabbit extends Animal {
  constructor(name) {
    super(name);
  }
}
````

**Ответ**:

```javascript
Rabbit.prototype = Object.create(Animal.prototype);
Rabbit.prototype.constructor = Rabbit;
Rabbit.prototype.__proto__ = Animal.prototype;
```

### 6. Практика: добавление метода (React Middle)

````
Добавьте метод `last()` ко всем массивам.
```javascript
const arr = [1, 2, 3];
console.log(arr.last());  // 3
````

**Ответ**:

```javascript
Array.prototype.last = function () {
  return this[this.length - 1];
};
```

## React контекст

````
Почему этот паттерн работает в React?
```javascript
const withLogging = (Component) => {
  const originalPrototype = Object.getPrototypeOf(Component.prototype);
  const LoggedComponent = function(props) {
    console.log('Render:', props);
    return Component.call(this, props);
  };
  LoggedComponent.prototype = Object.create(originalPrototype);
  LoggedComponent.prototype.constructor = LoggedComponent;
  return LoggedComponent;
};
````

**Ответ**: Создает HOC с сохранением прототипа (методов lifecycle).
