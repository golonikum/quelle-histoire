# Event Loop: Цикл событий, очередь задач, микрозадачи

Event Loop — бесконечный цикл, который управляет выполнением синхронного и асинхронного кода в однопоточном JavaScript. Он координирует **Call Stack**, **Task Queue** (макрозадачи) и **Microtask Queue** (микрозадачи).

## Архитектура Event Loop

```
┌─────────────────┐     ┌──────────────────┐    ┌─────────────────┐
│   Call Stack    │ ←── │  Event Loop      │──→ │ Render (Browser)│
│ (синхронный)    │     │ (координатор)    │    │ (перерисовка)   │
└─────────────────┘     └──────────────────┘    └─────────────────┘
                                  │
                    ┌─────────────┼──────────────┐
                    │             │              │
        ┌─────────────────┐ ┌───────────────┐ ┌──────────────────┐
        │ Microtask Queue │ │  Task Queue   │ │   Web APIs       │
        │  (.then, await) │ │ (setTimeout)  │ │ (fetch, DOM)     │
        └─────────────────┘ └───────────────┘ └──────────────────┘
```

## Принцип работы (алгоритм)

```
while (true) {
  // 1. Выполнить ВСЕ синхронные задачи из Call Stack
  while (callStack.length > 0) {
    executeNextTask();
  }

  // 2. Выполнить ВСЕ микрозадачи
  while (microtasks.length > 0) {
    executeMicrotask();
  }

  // 3. Взять ОДНУ макрозадачу из Task Queue
  const macrotask = taskQueue.shift();
  if (macrotask) {
    callStack.push(macrotask);
  }

  // 4. Render (если браузер)
  if (needRender()) {
    render();
  }
}
```

## Типы очередей

| Очередь        | Что попадает                                   | Пример                     | Приоритет |
| :------------- | :--------------------------------------------- | :------------------------- | :-------- |
| **Call Stack** | Синхронный код                                 | `console.log()`            | Высший    |
| **Microtasks** | Promise `.then()`, `await`, `queueMicrotask()` | `Promise.resolve().then()` | Высокий   |
| **Macrotasks** | `setTimeout`, `setInterval`, I/O               | `setTimeout(fn, 0)`        | Низкий    |

## Классический пример

```javascript
console.log("1"); // 1. Синхронный

setTimeout(() => console.log("2"), 0); // 4. Макрозадача

Promise.resolve()
  .then(() => console.log("3")) // 2. Микрозадача
  .then(() => console.log("4")); // 3. Микрозадача (после 3)

queueMicrotask(() => console.log("5")); // 5. Микрозадача

console.log("6"); // 6. Синхронный

// Вывод: 1 6 3 4 5 2
```

## Пошаговый разбор

```
1️⃣ Call Stack: console.log('1') → вывод "1"
2️⃣ Call Stack: setTimeout → в Web APIs → позже в Task Queue
3️⃣ Call Stack: Promise → .then в Microtask Queue
4️⃣ Call Stack: queueMicrotask → в Microtask Queue
5️⃣ Call Stack: console.log('6') → вывод "6"
6️⃣ Call Stack пуст → Microtasks: вывод "3"
7️⃣ Microtasks: вывод "4"
8️⃣ Microtasks: вывод "5"
9️⃣ Microtasks пусты → Macrotasks: вывод "2"
🔄 Повторяется...
```

## React контекст

### useEffect порядок

```jsx
useEffect(() => {
  console.log("effect"); // Макрозадача (после render)
});

Promise.resolve().then(() => {
  console.log("promise"); // Микрозадача (до effect)
});

console.log("render"); // Синхронный (сначала)
// render → promise → effect
```

### Баг с состоянием

```jsx
const [count, setCount] = useState(0);

setTimeout(() => console.log(count), 0); // 0 (макрозадача)
Promise.resolve().then(() => console.log(count)); // 0 (микрозадача)

setCount(1); // Батчинг — состояние обновится позже
// Оба покажут 0!
```

## Node.js vs Browser

**Node.js** имеет 6 фаз Event Loop:

```
timers → pending callbacks → idle/prepare → poll → check → close callbacks
```

**Browser**: timers → microtasks → render → repeat

## Практические примеры

### 1. Почему React Suspense работает

```javascript
// Микрозадачи выполняются ДО render
await promise; // Suspense "ждет" микрозадачу
return <div>Данные готовы</div>; // Render после await
```

### 2. Оптимизация батчинга

```javascript
// ❌ 3 макрозадачи
setState1();
setState2();
setState3();

// ✅ 1 микрозадача (React 18 auto-batch)
flushSync(setState1);
queueMicrotask(() => {
  setState2();
  setState3(); // Батчатся вместе
});
```

## Таблица приоритетов

```
1. Call Stack (синхронный код)
   ↓
2. Microtask Queue (.then, await, queueMicrotask)
   ↓ (ВСЕ микрозадачи)
3. Task Queue (setTimeout, UI события, fetch)
   ↓ (ОДНА макрозадача идет в Call Stack)
4. Render (browser)
   ↓
Повтор
```

**Золотое правило**: **"Микрозадачи > Макрозадачи. Синхронный код всегда первый"**.

**Собеседование**: "Что выведет `console.log('1'); Promise.resolve().then(() => console.log('2')); setTimeout(() => console.log('3'), 0); console.log('4')`?" → `1 4 2 3`.

# Event Loop: разница в браузере и Node.js

Event Loop в JavaScript обеспечивает асинхронное выполнение кода в однопоточном окружении, но его реализация отличается между браузером и Node.js. В браузере он проще и ориентирован на UI, а в Node.js — более сложный из-за серверных задач. Вот ключевые различия.

## Основная архитектура

В браузере Event Loop встроен в HTML Host и использует Web APIs (DOM, таймеры, Fetch) для асинхронных операций, с фокусом на микрозадачи (Promises, MutationObserver) и макрозадачи (setTimeout, UI-события). Node.js полагается на библиотеку libuv, которая добавляет многопоточность для I/O (файлы, сеть) через пул потоков, сохраняя JS однопоточным. Браузерный поток отвечает за рендеринг, поэтому макрозадачи не блокируют UI.

## Фазы обработки

Браузер следует простой модели: выполнение Call Stack → микрозадачи → макрозадачи → рендер (если нужно). Node.js имеет 6+ фаз: timers (setTimeout), pending callbacks (I/O), idle/prepare, poll (новые I/O), check (setImmediate), close callbacks, с микрозадачами после каждой фазы. Это позволяет точнее контролировать порядок, например setImmediate может выполниться раньше setTimeout(0) в poll-фазе.

## API и задачи

| Аспект           | Браузер                        | Node.js                                   |
| :--------------- | :----------------------------- | :---------------------------------------- |
| Асинхронные API  | DOM-события, XHR/Fetch, Canvas | Файлы, DNS, TCP/UDP, child processes [^4] |
| Особые методы    | requestAnimationFrame          | setImmediate, process.nextTick            |
| Worker-поддержка | Web Workers                    | Worker Threads [^2]                       |

В Node.js nextTick создаёт отдельную очередь с приоритетом выше микрозадач, чего нет в браузере. Эти различия важны для отладки: в браузере setTimeout(0) всегда после микрозадач, в Node.js — зависит от фазы.
