# Типы данных

JavaScript имеет 8 типов данных: 7 примитивов и объекты. Примитивы неизменяемы и передаются по значению, а объекты — по ссылке, что влияет на сравнения.

## Примитивы

Примитивы — это базовые атомарные значения без методов (хотя временно "оборачиваются" в объекты при вызове свойств).

- `string`: `'hello'`.[^1]
- `number`: `42`, `3.14`, `NaN`, `Infinity`.
- `bigint`: `123n` (для больших чисел).
- `boolean`: `true`, `false`.
- `null`: намеренное пустое значение.
- `undefined`: значение по умолчанию.
- `symbol`: уникальный идентификатор, `Symbol('id')`.[^4]

Пример: `typeof 'hello' // "string"`, `typeof null // "object"` (историческая ошибка).[^2]

## Объекты

Объекты — коллекции свойств (ключ-значение), изменяемы, передаются по ссылке.
Включают: `{a: 1}`, `[1,2]`, `() => {}`, `new Date()`.
Пример:

```js
let obj = { name: "Иван" };
let copy = obj; // ссылка, не копия
obj.name = "Петр";
console.log(copy.name); // 'Петр'
```

````
Примитивы копируются: `let a = 5; let b = a; b = 10; console.log(a);  // 5` [web:32].

## Сравнение (== vs ===)
`===` — строгое, сравнивает тип и значение, без преобразований.
`==` — нестрогое, преобразует операнды (абсурдные правила Abstract Equality) [web:30].

| Оператор | `0 == false` | `'' == 0` | `' ' == 0` | `[] == 0` | `null == undefined` |
|----------|--------------|-----------|------------|-----------|---------------------|
| `==`     | `true`       | `true`    | `false`    | `true`    |  `true`             |
| `===`    | `false`      | `false`   | `false`    | `false`   |  `false`            |[web:31]

Примеры:
```js
console.log(1 == '1');     // true (string → number)
console.log(1 === '1');    // false (типы разные)
console.log(true == 1);    // true (boolean → number)
console.log(null == 0);    // false
console.log([] == 0);      // true ([] → '' → 0)
````

Всегда используйте `===` — `==` приводит к багам [web:36].

# Как правильно сравнивать объекты по содержимому

В JavaScript объекты сравниваются по ссылке через `===`, поэтому для сравнения по содержимому нужны специальные методы. Стандартного глубокого равенства нет, но есть проверенные подходы от простых до продвинутых.

## Поверхностное сравнение

Сравнивает только enumerable свойства первого уровня, без вложенных объектов.

```js
function shallowEqual(obj1, obj2) {
  const keys1 = Object.keys(obj1);
  const keys2 = Object.keys(obj2);

  if (keys1.length !== keys2.length) return false;

  for (let key of keys1) {
    if (obj1[key] !== obj2[key]) return false; // === для примитивов
  }
  return true;
}

// Работает для плоских объектов
const user1 = { name: "Иван", age: 30 };
const user2 = { name: "Иван", age: 30 };
console.log(shallowEqual(user1, user2)); // true
```

**Проблема**: `{a: {b: 1}} !== {a: {b: 1}}` (вложенные объекты разные).

## Глубокое сравнение (рекурсия)

Рекурсивно проходит по всем свойствам, включая массивы и вложенные объекты.

```js
function deepEqual(obj1, obj2) {
  // Проверка на null/undefined
  if (obj1 === null || obj2 === null) return obj1 === obj2;
  if (typeof obj1 !== "object" || typeof obj2 !== "object") {
    return obj1 === obj2;
  }

  // Массивы
  if (Array.isArray(obj1) !== Array.isArray(obj2)) return false;
  const keys1 = Object.keys(obj1);
  const keys2 = Object.keys(obj2);

  if (keys1.length !== keys2.length) return false;

  for (let key of keys1) {
    if (!keys2.includes(key) || !deepEqual(obj1[key], obj2[key])) {
      return false;
    }
  }
  return true;
}

// ✅ Работает с вложенностью
const nested1 = { user: { info: { age: 30 } }, tags: [1, 2] };
const nested2 = { user: { info: { age: 30 } }, tags: [1, 2] };
console.log(deepEqual(nested1, nested2)); // true
```

**Ограничения**: циклические ссылки (нужен WeakMap), функции/даты/RegExp требуют доп. логики.

## JSON.stringify (быстрый хак)

Самый простой способ для JSON-совместимых данных.

```js
console.log(JSON.stringify({a: 1, b: [^2]}) === JSON.stringify({a: 1, b: [^2]}));  // true
```

**НЕ работает для**: функций, `undefined`, `Date`, порядка ключей в ES2015+, циклических ссылок.

```js
console.log(JSON.stringify({ a: 1, b: 2 }) === JSON.stringify({ b: 2, a: 1 })); // false
```

## Библиотеки

**Lodash `_.isEqual`** — золотой стандарт, обрабатывает все edge-кейсы.

```js
import { isEqual } from "lodash";
isEqual({ a: { b: 1 } }, { a: { b: 1 } }); // true
```

**fast-deep-equal** — быстрее Lodash для React (используется в React.memo).

```bash
npm i fast-deep-equal
```

```js
import equal from "fast-deep-equal";
equal(state1, state2); // оптимизировано для React
```

## React контекст

В React для `React.memo` и `useMemo` сравниваются пропсы shallow (по умолчанию).

```js
const MyComponent = React.memo(({ data }) => (
  <div>{JSON.stringify(data)}</div> // ❌ Дорого!
));
```

**Лучше**: нормализуйте данные или используйте `fast-deep-equal` в `areEqual`.

```js
const arePropsEqual = (prev, next) => fastDeepEqual(prev.data, next.data);
const MyComponent = React.memo(Component, arePropsEqual);
```

## Практические советы

| Сценарий        | Метод                             |
| :-------------- | :-------------------------------- |
| Простые объекты | `shallowEqual`                    |
| JSON-данные     | `JSON.stringify`                  |
| React пропсы    | `fast-deep-equal`                 |
| Production      | Lodash/Ramda                      |
| Edge-кейсы      | Собственная `deepEqual` с WeakMap |

**Правило**: 90% задач — `JSON.stringify`. Сложные случаи — библиотеки. Самописное реже.

# Примеры автобоксинга примитивов в методы объектов

В JavaScript **нет автобоксинга** как в Java. Примитивы не преобразуются автоматически в объекты при вызове методов. Вместо этого JS использует **временные объекты-обертки** (String, Number, Boolean), которые создаются только на время вызова метода и сразу уничтожаются.

## Как это работает

Когда вы вызываете метод у примитива, JS:

1. Создает объект-обертку с этим значением
2. Вызывает метод на объекте
3. Уничтожает объект-обертку
4. Возвращает результат

## Примеры

### String

```js
let str = "hello";
console.log(str.toUpperCase()); // 'HELLO'
console.log(str.length); // 5

// Под капотом:
console.log(String(str).toUpperCase()); // Временная обертка
```

### Number

```js
let num = 42;
console.log(num.toFixed(2)); // '42.00'
console.log(num.toString(16)); // '2a' (шестнадцатеричное)

// Под капотом:
console.log(Number(num).toFixed(2));
```

### Boolean

```js
let bool = true;
console.log(bool.valueOf()); // true
console.log(bool.toString()); // 'true'

// Под капотой:
console.log(Boolean(bool).valueOf());
```

## Особенности и подводные камни

### 1. Методы доступны только у примитивов с методами

```js
let num = 42;
num.myCustomMethod(); // TypeError! У Number нет такого метода
```

### 2. Автоматическое приведение типов

```js
let str = "42";
console.log(str * 2); // 84 (строка → число)
console.log(str + 2); // '422' (число → строка)
```

### 3. Явная работа с обертками

```js
let str = new String("hello"); // Объект String
let num = new Number(42); // Объект Number

typeof str; // 'object'
typeof num; // 'object'
```

**Ключевое**: В JS примитивы **имитируют** объекты только при вызове методов. После выполнения примитив остается примитивом.
