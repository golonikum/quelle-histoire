# this: Как определяется this в разных контекстах

`this` в JavaScript определяется **контекстом вызова**, а не места объявления функции. Значение устанавливается динамически по 4 правилам приоритета.

## Правила определения `this`

| Правило           | Приоритет | Пример          | `this`                        |
| :---------------- | :-------- | :-------------- | :---------------------------- |
| `new`             | 1         | `new MyClass()` | Новый экземпляр               |
| `call/apply/bind` | 2         | `fn.call(obj)`  | `obj`                         |
| Метод объекта     | 3         | `obj.method()`  | `obj`                         |
| Обычный вызов     | 4         | `fn()`          | `window`/`undefined` (strict) |

## 1. Глобальный контекст

```js
console.log(this); // window (браузер) / global (Node.js)

function globalFn() {
  console.log(this); // window / undefined ('use strict')
}

globalFn();
```

## 2. Метод объекта

```js
const user = {
  name: "Иван",
  greet() {
    console.log(`Привет, ${this.name}!`); // this = user
  },
};

user.greet(); // 'Привет, Иван!'
```

**Проблема** — потеря контекста:

```js
const fn = user.greet; // Ссылка на функцию
fn(); // this = window → TypeError или undefined.name
```

## 3. Конструктор + `new`

```js
function User(name) {
  this.name = name; // this = новый объект
}

const ivan = new User("Иван");
console.log(ivan.name); // 'Иван'
```

## 4. Стрелочные функции (лексический this)

Стрелки **НЕ** имеют собственного `this` — захватывают из внешнего контекста.

```js
const user = {
  name: "Иван",
  greetRegular() {
    setTimeout(function () {
      console.log(this.name); // window.name = undefined
    }, 0);
  },
  greetArrow() {
    setTimeout(() => {
      console.log(this.name); // 'Иван' — лексический this
    }, 0);
  },
};
```

## 5. `call`, `apply`, `bind`

**Принудительно устанавливают** `this`:

```js
function greet(greeting) {
  console.log(`${greeting}, ${this.name}`);
}

const user = { name: "Иван" };

// call — аргументы по очереди
greet.call(user, "Привет"); // 'Привет, Иван'

// apply — аргументы массивом
greet.apply(user, ["Здравствуй"]); // 'Здравствуй, Иван'

// bind — возвращает новую функцию
const boundGreet = greet.bind(user, "Привет");
boundGreet(); // 'Привет, Иван'
```

## React примеры

### ❌ Плохо — потеря this в колбэках

```jsx
class MyComponent extends React.Component {
  constructor() {
    this.state = { count: 0 };
    // ❌ Забыли bind
  }

  increment() {
    this.setState({ count: this.state.count + 1 });
  }

  render() {
    return <button onClick={this.increment}>+</button>; // this = undefined
  }
}
```

### ✅ Правильно

```jsx
// 1. Constructor bind
constructor() {
  this.increment = this.increment.bind(this);
}

// 2. Arrow function
increment = () => {
  this.setState({ count: this.state.count + 1 });
}

// 3. Arrow в render (функциональные компоненты)
const increment = () => {
  setCount(c => c + 1);  // Лексический this
};
```

## Таблица для собеседования

| Контекст            | Обычная функция | Стрелочная     |
| :------------------ | :-------------- | :------------- |
| `obj.method()`      | `obj`           | Лексический    |
| `method()`          | `window`/undef  | Лексический    |
| `new Fn()`          | Новый объект    | ❌ Ошибка      |
| `setTimeout(fn, 0)` | `window`/undef  | Лексический    |
| `fn.call(obj)`      | `obj`           | ❌ Не меняется |

**Правило**: "this смотрит на точку слева от скобок вызова `()`".

## Типичные вопросы по `this` на собеседовании

### 1. Базовый (Junior/Middle)

````
Что выведет код?
```javascript
const obj = {
  name: 'Иван',
  greet: function() { console.log(this.name); }
};

const fn = obj.greet;
fn();  // ?
````

**Ответ**: `undefined` (обычный вызов → `this = window/undefined`)
**Правильно**: `obj.greet()` → `'Иван'`

### 2. setTimeout ловушка (Middle)

````
Что выведет?
```javascript
const obj = {
  value: 42,
  getValue: function() {
    setTimeout(function() {
      console.log(this.value);
    }, 100);
  }
};

obj.getValue();  // ?
````

**Ответ**: `undefined` (`setTimeout` вызывает колбэк глобально)
**Исправления**:

```javascript
// 1. Arrow
setTimeout(() => console.log(this.value), 100); // 42

// 2. bind
setTimeout(
  function () {
    console.log(this.value);
  }.bind(this),
  100,
);

// 3. self
const self = this;
setTimeout(function () {
  console.log(self.value);
}, 100);
```

### 3. call/apply/bind (Middle+)

```
Разница между `call`, `apply`, `bind`? Покажите пример.
```

**Ответ**:

```javascript
function greet(greeting, punctuation) {
  console.log(`${greeting}, ${this.name}${punctuation}`);
}

const user = { name: "Иван" };

// call — аргументы последовательно
greet.call(user, "Привет", "!"); // 'Привет, Иван!'

// apply — аргументы массивом
greet.apply(user, ["Здравствуй", "?"]); // 'Здравствуй, Иван?'

// bind — новая функция
const bound = greet.bind(user, "Привет");
bound("!"); // 'Привет, Иван!'
```

### 4. Arrow vs Regular (Middle)

````
Почему этот код работает по-разному?
```javascript
const obj = {
  name: 'Иван',
  regular: function() {
    setTimeout(function() {
      console.log(this.name);
    }, 0);
  },
  arrow: function() {
    setTimeout(() => {
      console.log(this.name);
    }, 0);
  }
};
````

**Ответ**:

- `regular`: `undefined` (глобальный `this`)
- `arrow`: `'Иван'` (лексический `this` из `arrow`)

### 5. React классовый компонент (React Middle)

````
Почему кнопка не работает в классе?
```jsx
class Counter extends React.Component {
  state = { count: 0 };

  increment() {
    this.setState({ count: this.state.count + 1 });
  }

  render() {
    return (
      <button onClick={this.increment}>  // ❌ this = undefined
        {this.state.count}
      </button>
    );
  }
}
````

**Решения**:

```javascript
// 1. Constructor bind
constructor() {
  this.increment = this.increment.bind(this);
}

// 2. Arrow property
increment = () => {
  this.setState({ count: this.state.count + 1 });
}

// 3. Arrow в render
<button onClick={() => this.increment()}>  // ✅ Но лишние ререндеры
```

## React функциональные компоненты

```
Почему в функциональных компонентах нет проблем с this?
```

**Ответ**: Лексический `this` + хуки. Стрелочные функции захватывают состояние из scope:

```jsx
const increment = useCallback(() => {
  setCount((c) => c + 1); // Лексический захват useState
}, []);
```

## Что проверяет интервьюер

| Вопрос                    | Уровень      | Знание                |
| :------------------------ | :----------- | :-------------------- |
| `fn = obj.method(); fn()` | Junior       | Потеря контекста      |
| `setTimeout` в методе     | Middle       | Колбэки + стрелки     |
| `call/apply/bind`         | Middle+      | Управление контекстом |
| React классы              | React Middle | Практика              |

**Красные флаги**:

- ❌ "this всегда указывает на объект класса"
- ❌ Не знает про стрелочные функции
- ❌ Путает `call` и `bind`

**Идеальный ответ**: "`this` определяется точкой слева от `()`. Стрелки используют лексический контекст. В React — `useCallback` вместо `bind`."
